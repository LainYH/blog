# 第1章 绪论

## 1.1 数据结构的基本概念

### 1.1.1 数据

数据是信息的载体，是描述客观事物的数、字符以及所有能输入到计算机中并被计算机程序识别和处理的符号的集合。

### 1.1.2 数据元素

数据元素是数据的基本单位，在计算机中通常作为一个整体进行考虑和处理。一个数据元素可由若干**数据项**组成，数据项是构成数据元素的不可分割的最小单位。

### 1.1.3 数据对象

数据对象是具有相同性质的数据元素的集合，是数据的一个子集。

### 1.1.4 数据类型

数据类型是一个值的集合和定义在此集合上的一组操作的总称：
- **原子类型**：其值不可再分的数据类型，如 int、char 等。
- **结构类型**：其值可以再分解为若干成分的数据类型，如结构体。
- **抽象数据类型（ADT）**：一个数学模型及定义在该模型上的一组操作。抽象数据类型的定义仅取决于它的一组逻辑特性，而与其在计算机内部如何表示和实现无关。

⚠️ **重难点：抽象数据类型（ADT）**

ADT 的定义格式：
```
ADT 抽象数据类型名 {
    数据对象：<数据对象的定义>
    数据关系：<数据关系的定义>
    基本操作：<基本操作的定义>
}
```

### 1.1.5 数据结构

数据结构是相互之间存在一种或多种特定关系的数据元素的集合。数据结构包括三方面的内容：**逻辑结构**、**存储结构（物理结构）** 和 **数据的运算**。

⚠️ **重难点：逻辑结构 vs 存储结构**

| 维度 | 逻辑结构 | 存储结构 |
|------|---------|---------|
| 定义 | 数据元素之间的逻辑关系 | 数据结构在计算机中的表示（映像） |
| 关注点 | "是什么关系" | "怎么存放" |
| 独立性 | 独立于计算机 | 依赖计算机 |

#### （1）逻辑结构

数据的逻辑结构分为以下四类：

| 类型 | 特征 | 典型例子 |
|------|------|---------|
| **集合** | 数据元素之间除"同属一个集合"外，无其他关系 | 集合 |
| **线性结构** | 一对一关系 | 线性表、栈、队列 |
| **树形结构** | 一对多关系 | 二叉树、B树 |
| **图状结构（网状结构）** | 多对多关系 | 图 |

⚠️ 注意：线性表和非线性表的区别——线性表有且仅有一个开始结点和一个终端结点，所有结点最多只有一个直接前驱和一个直接后继。

#### （2）存储结构

四种基本的存储结构：

| 存储结构 | 说明 | 优缺点 |
|---------|------|--------|
| **顺序存储** | 把逻辑上相邻的元素存储在物理位置上也相邻的存储单元中 | 优点：可实现随机存取；缺点：可能产生较多外部碎片 |
| **链式存储** | 借助指示元素存储地址的指针来表示元素之间的逻辑关系 | 优点：不会产生碎片；缺点：只能顺序存取，额外指针占用空间 |
| **索引存储** | 在存储元素信息的同时，还建立附加的索引表 | 优点：检索速度快；缺点：增加空间开销 |
| **哈希存储（散列存储）** | 根据元素的关键字直接计算出该元素的存储地址 | 优点：检索、增加、删除都很快；缺点：只适用于关键字与地址存在对应关系的场景 |

⚠️ **重难点：同一逻辑结构可以有多种存储结构**

例如线性表既可以用顺序表（顺序存储）实现，也可以用链表（链式存储）实现。不同的存储结构各有优劣。

## 1.2 算法和算法评价

### 1.2.1 算法的定义

算法（Algorithm）是对特定问题求解步骤的一种描述，它是指令的有限序列，其中每一条指令表示一个或多个操作。

### 1.2.2 算法的五个特性

| 特性 | 含义 |
|------|------|
| **有穷性** | 一个算法必须总在执行有穷步之后结束，且每一步都可在有穷时间内完成 |
| **确定性** | 算法中每一条指令必须有确切的含义，不会产生二义性 |
| **可行性** | 算法中描述的操作都是可以通过已经实现的基本运算执行有限次来实现的 |
| **输入** | 一个算法有零个或多个输入 |
| **输出** | 一个算法有一个或多个输出 |

### 1.2.3 "好"算法的五个标准

| 标准 | 含义 |
|------|------|
| **正确性** | 算法应当能够正确地解决求解问题 |
| **可读性** | 算法应具有良好的可读性，以帮助人们理解 |
| **健壮性** | 输入非法数据时，算法能适当地做出反应或进行处理 |
| **高效率与低存储量需求** | 效率指算法执行的时间，存储量指算法执行过程中所需的最大存储空间 |

## 1.3 算法的时间复杂度 ⚠️（核心考点）

### 1.3.1 时间复杂度的定义

一般情况下，算法中基本操作重复执行的次数是问题规模 n 的某个函数 f(n)，算法的时间量度记作：

$$T(n) = O(f(n))$$

它表示随问题规模 n 的增大，算法执行时间的增长率和 f(n) 的增长率相同，称为算法的**渐近时间复杂度**，简称**时间复杂度**。

### 1.3.2 求时间复杂度的方法

1. 找出算法中的**基本语句**（执行次数最多的语句，通常是循环的最内层循环体）
2. 计算基本语句的执行次数（频度）f(n)
3. 用大 O 表示法表示，只保留最高阶项，去掉最高阶项的常数系数

### 1.3.3 常见的时间复杂度

按增长率从低到高排列：

$$O(1) < O(\log_2 n) < O(n) < O(n\log_2 n) < O(n^2) < O(n^3) < O(2^n) < O(n!) < O(n^n)$$

⚠️ **高频考点：常见时间复杂度的判断**

| 时间复杂度 | 典型场景 |
|-----------|---------|
| O(1) | 直接计算、哈希表查找 |
| O(log₂n) | 折半查找、二叉树的操作 |
| O(n) | 线性查找、遍历数组 |
| O(nlog₂n) | 快速排序、归并排序、堆排序的平均/最坏时间复杂度 |
| O(n²) | 冒泡排序、简单选择排序、直接插入排序 |
| O(2^n) | 汉诺塔问题、递归的斐波那契 |
| O(n!) | 旅行商问题的暴力解法 |

### 1.3.4 递归算法的时间复杂度

对于递归算法，通常用**递推方程**求解时间复杂度。

例：分析以下递归函数的时间复杂度：
```c
void func(int n) {
    if (n <= 1) return;
    func(n-1);
    func(n-1);
}
```
递推方程：T(n) = 2T(n-1) + O(1)，解得 T(n) = O(2ⁿ)

## 1.4 算法的空间复杂度

### 1.4.1 定义

算法的空间复杂度 S(n) 定义为该算法所耗费的存储空间，它是问题规模 n 的函数。

$$S(n) = O(g(n))$$

### 1.4.2 说明

- 算法原地工作是指算法所需的辅助空间为常量，即 O(1)
- 递归算法的空间复杂度 = 递归调用的深度 × 每层所需的辅助空间

⚠️ **易混淆点：时间复杂度 vs 空间复杂度**

以归并排序为例：时间复杂度 O(nlog₂n)，空间复杂度 O(n)（需要辅助数组）。

## 1.5 本章例题

### 例题1：时间复杂度分析

分析以下代码段的时间复杂度：
```c
int count = 0;
for (int i = 1; i <= n; i *= 2) {
    for (int j = 1; j <= i; j++) {
        count++;
    }
}
```

**解析：**

外层循环：i 的取值为 1, 2, 4, 8, ..., 直到 i > n，共 log₂n + 1 次。

对于每次外层循环（设 i = 2^k），内层循环执行 2^k 次。

总执行次数 = 1 + 2 + 4 + 8 + ... + 2^⌊log₂n⌋ = 2^(⌊log₂n⌋+1) - 1 ≈ 2n - 1

**答案：O(n)**

### 例题2：递归时间复杂度

以下算法的时间复杂度是多少？
```c
void fun(int n) {
    int i = 0;
    while (i * i <= n) {
        i++;
    }
}
```

**解析：**

while 循环在 i² > n 时结束，即 i > √n 时结束。循环执行 ⌊√n⌋ + 1 次。

**答案：O(√n)**

### 例题3：多重循环时间复杂度

分析以下代码段的时间复杂度：
```c
for (int i = 0; i < n; i++) {
    for (int j = i; j < n; j++) {
        // 基本操作
    }
}
```

**解析：**

外层 i = 0 时，内层执行 n 次；
外层 i = 1 时，内层执行 n-1 次；
...
外层 i = n-1 时，内层执行 1 次。

总次数 = n + (n-1) + ... + 1 = n(n+1)/2

**答案：O(n²)**

### 例题4：概念辨析

下列关于数据结构的说法中，正确的是（  ）。

A. 数据的逻辑结构独立于其存储结构
B. 数据的存储结构是数据逻辑结构的计算机实现
C. 数据的逻辑结构唯一决定了其存储结构
D. 以上说法都正确

**答案：A、B**

**解析：** 逻辑结构独立于存储结构（A 正确）；存储结构是逻辑结构在计算机中的表示（B 正确）；同一逻辑结构可以有不同的存储结构实现，如线性表既可以用顺序表也可以用链表（C 错误）。D 自然错误。

### 例题5：时间复杂度计算

设 n 是描述问题规模的非负整数，下列程序段的时间复杂度是（  ）。（2019年408真题）
```c
int x = 2;
while (x < n / 2) {
    x = 2 * x;
}
```

**解析：**

x 的取值序列：2, 4, 8, 16, ..., 2^(k+1)

循环条件：2^(k+1) < n/2，即 k+1 < log₂(n/2)，k < log₂n - 2

循环次数 ≈ log₂n - 2

**答案：O(log₂n)**
