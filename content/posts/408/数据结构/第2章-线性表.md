# 第2章 线性表

## 2.1 线性表的定义和基本操作

### 2.1.1 线性表的定义

线性表是具有**相同数据类型**的 n (n ≥ 0) 个数据元素的**有限序列**，其中 n 为表长，当 n = 0 时线性表是一个**空表**。

记为：L = (a₁, a₂, ..., aᵢ, ..., aₙ)

- a₁ 是唯一的"第一个"数据元素，又称**表头元素**
- aₙ 是唯一的"最后一个"数据元素，又称**表尾元素**
- 除第一个元素外，每个元素有且仅有一个**直接前驱**
- 除最后一个元素外，每个元素有且仅有一个**直接后继**

⚠️ **重点：线性表的特点**
- 元素个数有限
- 元素具有逻辑上的顺序性，表中元素有其先后次序
- 表中元素都是数据元素，每个元素都是单个元素
- 表中元素的数据类型都相同，意味着每个元素占有相同大小的存储空间
- 表中元素具有抽象性，仅讨论元素间的逻辑关系，而不考虑元素究竟表示什么内容

### 2.1.2 线性表的基本操作

| 操作 | 说明 |
|------|------|
| `InitList(&L)` | 初始化表，构造一个空的线性表 |
| `Length(L)` | 求表长，返回线性表 L 的长度 |
| `LocateElem(L, e)` | 按值查找，在表 L 中查找具有给定关键字值的元素 |
| `GetElem(L, i)` | 按位查找，获取表 L 中第 i 个位置的元素的值 |
| `ListInsert(&L, i, e)` | 插入操作，在表 L 中的第 i 个位置上插入指定元素 e |
| `ListDelete(&L, i, &e)` | 删除操作，删除表 L 中第 i 个位置的元素，并用 e 返回删除元素的值 |
| `PrintList(L)` | 输出操作，按前后顺序输出线性表 L 的所有元素值 |
| `Empty(L)` | 判空操作，若 L 为空表，则返回 true，否则返回 false |
| `DestroyList(&L)` | 销毁操作，销毁线性表，并释放线性表 L 所占用的内存空间 |

## 2.2 线性表的顺序表示（顺序表）

### 2.2.1 顺序表的定义

线性表的顺序存储又称**顺序表**。它是用一组地址连续的存储单元依次存储线性表中的数据元素，从而使得逻辑上相邻的两个元素在物理位置上也相邻。

⚠️ **重要特征**：顺序表中元素的**物理顺序**和**逻辑顺序**一致。

### 2.2.2 顺序表的实现

#### 静态分配

```c
#define MaxSize 50
typedef struct {
    ElemType data[MaxSize];  // 顺序表的元素
    int length;              // 顺序表的当前长度
} SqList;
```

⚠️ 注意：静态分配时，数组的大小和空间是固定的，一旦空间占满，再加入新的数据就会产生溢出。

#### 动态分配

```c
#define InitSize 100
typedef struct {
    ElemType *data;      // 指示动态分配数组的指针
    int MaxSize, length; // 数组的最大容量和当前个数
} SeqList;

// C语言动态分配语句
L.data = (ElemType *)malloc(sizeof(ElemType) * InitSize);
```

⚠️ **易混淆点：动态分配 ≠ 链式存储**

动态分配依然是顺序存储结构，只是分配的空间大小可以在运行时决定。逻辑结构不变，仍然是"物理位置相邻"。

### 2.2.3 顺序表上基本操作的实现

#### （1）插入操作

```c
bool ListInsert(SqList &L, int i, ElemType e) {
    if (i < 1 || i > L.length + 1)  // 判断i的范围是否有效
        return false;
    if (L.length >= MaxSize)         // 当前存储空间已满
        return false;
    for (int j = L.length; j >= i; j--)  // 将第i个元素及之后的元素后移
        L.data[j] = L.data[j-1];
    L.data[i-1] = e;                 // 在位置i处放入e
    L.length++;                      // 线性表长度加1
    return true;
}
```

⚠️ **时间复杂度分析：**
- 最好情况：在表尾插入（i = length + 1），时间复杂度 O(1)
- 最坏情况：在表头插入（i = 1），需要移动 n 个元素，时间复杂度 O(n)
- **平均情况：** 在位置 i 插入需要移动 n - i + 1 个元素，平均移动次数 = n/2，**平均时间复杂度 O(n)**

#### （2）删除操作

```c
bool ListDelete(SqList &L, int i, ElemType &e) {
    if (i < 1 || i > L.length)      // 判断i的范围是否有效
        return false;
    e = L.data[i-1];                 // 将被删除的元素赋值给e
    for (int j = i; j < L.length; j++)  // 将第i个位置后的元素前移
        L.data[j-1] = L.data[j];
    L.length--;                      // 线性表长度减1
    return true;
}
```

⚠️ **时间复杂度分析：**
- 最好情况：删除表尾元素，时间复杂度 O(1)
- 最坏情况：删除表头元素，时间复杂度 O(n)
- **平均情况：** 平均移动次数 = (n-1)/2，**平均时间复杂度 O(n)**

#### （3）按值查找

```c
int LocateElem(SqList L, ElemType e) {
    for (int i = 0; i < L.length; i++)
        if (L.data[i] == e)
            return i + 1;  // 返回位序（从1开始）
    return 0;              // 查找失败
}
```

**时间复杂度：O(n)**

### 2.2.4 顺序表的特点

- **优点：** 支持随机存取，存储密度高（只存数据，不存指针）
- **缺点：** 插入和删除需要移动大量元素；静态分配需要预先确定大小；动态分配虽然可以扩展，但需要移动数据

## 2.3 线性表的链式表示（链表）

### 2.3.1 单链表

#### 定义

线性表的链式存储又称**链表**。它通过一组任意的存储单元来存储线性表中的数据元素。对于每个数据元素 aᵢ，除了存储本身的信息外，还需存放一个指示其直接后继的信息——即**后继指针**。

```c
typedef struct LNode {
    ElemType data;           // 数据域
    struct LNode *next;      // 指针域
} LNode, *LinkList;
```

⚠️ **重点：区分 LNode 和 LinkList**
- `LNode` 是结点类型
- `LinkList` 是指向 LNode 的指针类型，强调这是一个链表

#### 头结点 vs 头指针

| 概念 | 说明 |
|------|------|
| **头指针** | 指向链表中第一个结点的指针，链表可以没有头结点但一定有头指针 |
| **头结点** | 链表第一个结点之前附设的一个结点，其数据域可以不存储任何信息（也可以存储表长等信息） |

⚠️ **引入头结点的好处：**
1. 由于第一个元素结点的位置被存放在头结点的指针域中，因此在链表的第一个位置上的操作和在表的其他位置上的操作一致，无需特殊处理
2. 无论链表是否为空，其头指针都是指向头结点的非空指针（空表中头结点的指针域为空），因此空表和非空表的处理也就得到了统一

#### 单链表的基本操作

**① 采用头插法建立单链表**

```c
LinkList List_HeadInsert(LinkList &L) {
    LNode *s;
    int x;
    L = (LinkList)malloc(sizeof(LNode));  // 创建头结点
    L->next = NULL;                        // 初始为空链表
    scanf("%d", &x);                      // 输入结点的值
    while (x != 9999) {                    // 输入9999表示结束
        s = (LNode*)malloc(sizeof(LNode)); // 创建新结点
        s->data = x;
        s->next = L->next;                 // 新结点的next指向原首元结点
        L->next = s;                       // 头结点的next指向新结点
        scanf("%d", &x);
    }
    return L;
}
```

⚠️ 头插法建立的链表中，元素的顺序与输入顺序**相反**。

**时间复杂度：O(n)**，每个结点只遍历一次。

**② 采用尾插法建立单链表**

```c
LinkList List_TailInsert(LinkList &L) {
    int x;
    L = (LinkList)malloc(sizeof(LNode));  // 创建头结点
    LNode *s, *r = L;                     // r为表尾指针
    scanf("%d", &x);
    while (x != 9999) {
        s = (LNode*)malloc(sizeof(LNode));
        s->data = x;
        r->next = s;
        r = s;                             // r指向新的表尾结点
        scanf("%d", &x);
    }
    r->next = NULL;                        // 尾结点指针置空
    return L;
}
```

尾插法中需要一个**尾指针** r，使得插入操作的时间复杂度保持在 O(n)（整体），每个结点插入为 O(1)。

**③ 按序号查找**

```c
LNode *GetElem(LinkList L, int i) {
    int j = 1;
    LNode *p = L->next;  // p指向第一个结点
    if (i == 0) return L; // 若i等于0，则返回头结点
    if (i < 1) return NULL;
    while (p && j < i) {
        p = p->next;
        j++;
    }
    return p;  // 返回第i个结点的指针，若i大于表长则返回NULL
}
```

**时间复杂度：O(n)**

**④ 按值查找**

```c
LNode *LocateElem(LinkList L, ElemType e) {
    LNode *p = L->next;
    while (p != NULL && p->data != e)
        p = p->next;
    return p;  // 找到后返回该结点指针，否则返回NULL
}
```

**时间复杂度：O(n)**

**⑤ 插入结点（在第 i 个位置插入）**

```c
bool ListInsert(LinkList &L, int i, ElemType e) {
    if (i < 1) return false;
    LNode *p = GetElem(L, i - 1);  // 找到第i-1个结点
    if (p == NULL) return false;
    LNode *s = (LNode*)malloc(sizeof(LNode));
    s->data = e;
    s->next = p->next;
    p->next = s;
    return true;
}
```

⚠️ **核心操作：** `s->next = p->next; p->next = s;` 两步顺序不能颠倒！如果先执行 `p->next = s`，则原后继结点的地址丢失。

**时间复杂度：O(n)**（主要花在查找第 i-1 个结点上）

**⑥ 删除结点**

```c
bool ListDelete(LinkList &L, int i, ElemType &e) {
    if (i < 1) return false;
    LNode *p = GetElem(L, i - 1);  // 找到第i-1个结点
    if (p == NULL || p->next == NULL) return false;
    LNode *q = p->next;             // q指向被删除结点
    e = q->data;
    p->next = q->next;              // 将*q结点从链中"断开"
    free(q);                        // 释放结点的存储空间
    return true;
}
```

**时间复杂度：O(n)**

### 2.3.2 双链表

```c
typedef struct DNode {
    ElemType data;
    struct DNode *prior, *next;  // 前驱和后继指针
} DNode, *DLinkList;
```

#### 双链表的插入

```c
bool DListInsert(DLinkList &L, int i, ElemType e) {
    DNode *p = GetDElem(L, i - 1);  // 找到第i-1个结点
    if (p == NULL) return false;
    DNode *s = (DNode*)malloc(sizeof(DNode));
    s->data = e;
    s->prior = p;
    s->next = p->next;
    if (p->next != NULL)             // ⚠️ 判断p是否有后继
        p->next->prior = s;
    p->next = s;
    return true;
}
```

⚠️ **注意：** 如果 p 是最后一个结点，则 `p->next = NULL`，不能执行 `p->next->prior = s`。

#### 双链表的删除

```c
bool DListDelete(DLinkList &L, int i, ElemType &e) {
    DNode *p = GetDElem(L, i - 1);
    if (p == NULL || p->next == NULL) return false;
    DNode *q = p->next;
    e = q->data;
    p->next = q->next;
    if (q->next != NULL)             // ⚠️ 判断q是否有后继
        q->next->prior = p;
    free(q);
    return true;
}
```

⚠️ **双链表 vs 单链表：**
- 双链表可以方便地找到前驱结点，因此在某些操作（如删除指定结点、在指定结点前插入）上有优势
- 但每个结点多一个指针域，空间开销更大

### 2.3.3 循环链表

#### 循环单链表

最后一个结点的指针不是 NULL，而是指向头结点，形成一个环。

```c
// 判空条件
L->next == L;

// 判断是否为表尾结点
p->next == L;
```

⚠️ **循环单链表的特点：**
- 从表中任一结点出发均可找到表中其他结点
- 如果操作主要在表头和表尾进行，可以设**尾指针**而非头指针，这样表头和表尾的操作均为 O(1)

#### 循环双链表

头结点的 prior 指向尾结点，尾结点的 next 指向头结点。

```c
// 判空条件
L->next == L && L->prior == L;
// 或
L->next == L;  // 因为循环双链表中，L->next == L 必然有 L->prior == L
```

⚠️ **循环双链表的插入操作无需判断后继是否为 NULL**，因为循环特性保证了任何结点都有后继和前驱。

### 2.3.4 静态链表

静态链表借助数组来描述线性表的链式存储结构，结点也有数据域 data 和指针域 next。

⚠️ **注意：** 这里的指针是结点的相对地址（数组下标），又称**游标**。

```c
#define MaxSize 50
typedef struct {
    ElemType data;
    int next;  // 下一个元素的数组下标
} SLinkList[MaxSize];
```

⚠️ **静态链表的应用场景：**
- 不支持指针的高级语言中模拟链表
- 某些特定场景下（如文件系统的 FAT 表）

## 2.4 顺序表和链表的比较

| 对比维度 | 顺序表 | 链表 |
|---------|--------|------|
| **存取方式** | 随机存取（O(1)按位访问） | 顺序存取（O(n)按位访问） |
| **逻辑与物理存储** | 逻辑上相邻的元素物理上也相邻 | 逻辑上相邻的元素物理上不一定相邻 |
| **查找** | 按值 O(n)，按位 O(1) | 按值 O(n)，按位 O(n) |
| **插入/删除** | O(n)（需要移动元素） | O(n)（需要查找，但修改指针 O(1)） |
| **空间** | 需要预分配空间，可能浪费或不够 | 按需分配，每个结点有指针域的额外开销 |
| **存储密度** | 高（只存数据） | 低（数据 + 指针） |

⚠️ **如何选择？**
- 当线性表需要频繁查找、很少进行插入和删除操作时，宜采用**顺序表**
- 当线性表需要频繁插入和删除时，宜采用**链表**
- 当线性表的元素个数变化较大或难以估计其最大规模时，宜采用**链表**

## 2.5 本章例题

### 例题1：顺序表插入算法（2010年真题改编）

已知一个顺序表 L，其中的元素递增有序排列。设计一个算法，插入元素 e 后仍保持递增有序。

**答案：**

```c
bool InsertOrderList(SqList &L, ElemType e) {
    if (L.length >= MaxSize) return false;
    int i = 0;
    // 查找插入位置
    while (i < L.length && L.data[i] < e)
        i++;
    // 将第i个元素及之后的元素后移
    for (int j = L.length; j > i; j--)
        L.data[j] = L.data[j-1];
    L.data[i] = e;
    L.length++;
    return true;
}
```

**时间复杂度：** O(n)，最坏情况插入在表头需要移动所有元素。

### 例题2：逆置单链表

将一个带头结点的单链表就地逆置。要求空间复杂度 O(1)。

**答案：**

```c
void ReverseList(LinkList &L) {
    LNode *p = L->next;  // p指向第一个结点
    L->next = NULL;       // 断开头结点
    LNode *q;
    while (p != NULL) {
        q = p->next;      // 保存后继
        p->next = L->next; // 头插法：p指向原首元结点
        L->next = p;       // 头结点指向p
        p = q;             // p后移
    }
}
```

**解析：** 核心思想是头插法。依次摘下原链表的每个结点，用头插法插入到头结点之后，得到的链表顺序正好与原链表相反。空间复杂度 O(1)，时间复杂度 O(n)。

### 例题3：删除链表中所有值为 x 的结点

在一个带头结点的单链表中，删除所有值为 x 的结点。

**答案：**

```c
void DeleteX(LinkList &L, ElemType x) {
    LNode *p = L->next, *pre = L, *q;
    while (p != NULL) {
        if (p->data == x) {
            q = p;
            p = p->next;
            pre->next = p;
            free(q);
        } else {
            pre = p;
            p = p->next;
        }
    }
}
```

### 例题4：找链表倒数第k个结点

已知一个带头结点的单链表，设计一个算法找出链表中倒数第 k 个结点（k ≥ 1）。

**答案：**

```c
LNode *FindKth(LinkList L, int k) {
    LNode *p = L->next, *q = L->next;
    // p先走k步
    for (int i = 0; i < k; i++) {
        if (p == NULL) return NULL;  // k大于表长
        p = p->next;
    }
    // p和q同时后移，当p到达末尾时，q指向倒数第k个
    while (p != NULL) {
        p = p->next;
        q = q->next;
    }
    return q;
}
```

**解析：** 利用双指针（快慢指针）技巧，p 先走 k 步，然后 p、q 同步前进。时间复杂度 O(n)，空间复杂度 O(1)。

### 例题5：判断链表是否有环

设计一个算法判断带头结点的单链表是否有环。

**答案：**

```c
bool HasCycle(LinkList L) {
    LNode *slow = L->next, *fast = L->next;
    while (fast != NULL && fast->next != NULL) {
        slow = slow->next;        // 慢指针走一步
        fast = fast->next->next;  // 快指针走两步
        if (slow == fast)          // 相遇则有环
            return true;
    }
    return false;  // fast到达链表尾部则无环
}
```

**解析：** Floyd 判圈算法（龟兔赛跑）。时间复杂度 O(n)，空间复杂度 O(1)。

### 例题6：两个链表的第一个公共结点

已知两个单链表可能有公共结点，找出它们的第一个公共结点。

**答案：**

```c
LNode *FindCommon(LinkList L1, LinkList L2) {
    int len1 = 0, len2 = 0;
    LNode *p = L1->next, *q = L2->next;
    // 分别求两个链表的长度
    while (p) { len1++; p = p->next; }
    while (q) { len2++; q = q->next; }
    // p指向较长的链表
    p = (len1 > len2) ? L1->next : L2->next;
    q = (len1 > len2) ? L2->next : L1->next;
    int diff = abs(len1 - len2);
    // 长链表先走diff步
    for (int i = 0; i < diff; i++)
        p = p->next;
    // 同步前进，直到相交
    while (p != q) {
        p = p->next;
        q = q->next;
    }
    return p;  // 可能为NULL（无公共结点）
}
```

**解析：** 如果两个链表有公共结点，则公共结点之后的结点全部相同。让较长链表先走长度差步，然后同步前进，第一个相同的结点即为公共结点。

### 例题7：有序链表合并（2016年真题改编）

将两个递增有序的单链表合并为一个递增有序的单链表。

**答案：**

```c
LinkList MergeList(LinkList L1, LinkList L2) {
    LNode *p = L1->next, *q = L2->next;
    LNode *r = L1;  // r指向结果链表的尾部
    L1->next = NULL;
    while (p && q) {
        if (p->data <= q->data) {
            LNode *t = p->next;
            r->next = p;
            r = p;
            p = t;
        } else {
            LNode *t = q->next;
            r->next = q;
            r = q;
            q = t;
        }
    }
    r->next = p ? p : q;  // 链接剩余部分
    free(L2);
    return L1;
}
```

**解析：** 利用尾插法，每次比较两个链表的当前结点，将较小者摘下插入结果链表尾部。时间复杂度 O(m+n)，空间复杂度 O(1)。
