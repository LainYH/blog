# 第3章 栈和队列

## 3.1 栈

### 3.1.1 栈的定义

栈（Stack）是只允许在一端进行插入或删除操作的线性表。允许插入和删除的一端称为**栈顶（Top）**，另一端称为**栈底（Bottom）**。不含任何数据元素的栈称为**空栈**。

栈的特点：**后进先出（LIFO, Last In First Out）**

⚠️ **重要：** 栈是一种受限的线性表，受限之处在于插入和删除只能在栈顶进行。

### 3.1.2 栈的基本操作

| 操作 | 说明 |
|------|------|
| `InitStack(&S)` | 初始化栈，构造一个空栈 |
| `StackEmpty(S)` | 判断栈是否为空，若为空返回 true |
| `Push(&S, x)` | 进栈，若栈未满，将 x 加入使之成为新栈顶 |
| `Pop(&S, &x)` | 出栈，若栈非空，弹出栈顶元素并用 x 返回 |
| `GetTop(S, &x)` | 读栈顶元素，若栈非空，用 x 返回栈顶元素 |
| `DestroyStack(&S)` | 销毁栈，并释放栈 S 占用的存储空间 |

⚠️ **易错点：** 进栈和出栈并不一定交替进行，可以连续进栈再连续出栈。

### 3.1.3 栈的顺序存储结构

#### 顺序栈的实现

```c
#define MaxSize 50
typedef struct {
    ElemType data[MaxSize];  // 存放栈中元素
    int top;                 // 栈顶指针
} SqStack;
```

**初始化：**
```c
void InitStack(SqStack &S) {
    S.top = -1;  // 初始化栈顶指针
}
```

⚠️ **注意区分两种方式：**
- `top = -1`：栈顶指针初始指向 -1，入栈时先 `top++` 再赋值
- `top = 0`：栈顶指针初始指向 0，入栈时先赋值再 `top++`

**判空：**
```c
bool StackEmpty(SqStack S) {
    return (S.top == -1);
}
```

**进栈：**
```c
bool Push(SqStack &S, ElemType x) {
    if (S.top == MaxSize - 1)  // 栈满，报错
        return false;
    S.data[++S.top] = x;       // 指针先加1，再入栈
    return true;
}
```

**出栈：**
```c
bool Pop(SqStack &S, ElemType &x) {
    if (S.top == -1)           // 栈空，报错
        return false;
    x = S.data[S.top--];       // 先出栈，指针再减1
    return true;
}
```

**读栈顶元素：**
```c
bool GetTop(SqStack S, ElemType &x) {
    if (S.top == -1) return false;
    x = S.data[S.top];
    return true;
}
```

#### ⚠️ 共享栈

利用栈底位置相对不变的特性，可以让两个顺序栈共享一个一维数组空间，将两个栈的栈底分别设置在共享空间的两端，两个栈顶向共享空间的中间延伸。

```c
#define MaxSize 100
typedef struct {
    ElemType data[MaxSize];
    int top0;  // 0号栈栈顶指针
    int top1;  // 1号栈栈顶指针
} ShStack;

void InitShStack(ShStack &S) {
    S.top0 = -1;
    S.top1 = MaxSize;
}

// 0号栈进栈
bool Push0(ShStack &S, ElemType x) {
    if (S.top0 + 1 == S.top1) return false;  // 栈满
    S.data[++S.top0] = x;
    return true;
}

// 1号栈进栈
bool Push1(ShStack &S, ElemType x) {
    if (S.top1 - 1 == S.top0) return false;  // 栈满
    S.data[--S.top1] = x;
    return true;
}
```

⚠️ **共享栈的栈满条件：** `top0 + 1 == top1`

共享栈的优点：存取数据的时间复杂度为 O(1)，且更有效地利用了存储空间。

### 3.1.4 栈的链式存储结构（链栈）

```c
typedef struct LinkNode {
    ElemType data;
    struct LinkNode *next;
} *LiStack;
```

⚠️ **链栈的优点：** 便于多个栈共享存储空间和提高其效率，且不存在栈满上溢的情况。

⚠️ **链栈通常采用带头结点的单链表实现，所有操作均在表头进行。**

## 3.2 队列

### 3.2.1 队列的定义

队列（Queue）是只允许在**一端**进行插入，而在**另一端**进行删除的线性表。允许插入的一端称为**队尾（Rear）**，允许删除的一端称为**队头（Front）**。

队列的特点：**先进先出（FIFO, First In First Out）**

### 3.2.2 队列的基本操作

| 操作 | 说明 |
|------|------|
| `InitQueue(&Q)` | 初始化队列，构造一个空队列 |
| `QueueEmpty(Q)` | 判断队列是否为空 |
| `EnQueue(&Q, x)` | 入队，若队列未满，将 x 加入，使之成为新的队尾 |
| `DeQueue(&Q, &x)` | 出队，若队列非空，删除队头元素，并用 x 返回 |
| `GetHead(Q, &x)` | 读队头元素，若队列非空，将队头元素赋值给 x |

### 3.2.3 队列的顺序存储结构

#### 一般顺序队列

```c
#define MaxSize 50
typedef struct {
    ElemType data[MaxSize];
    int front, rear;  // 队头指针和队尾指针
} SqQueue;
```

⚠️ **假溢出问题：** 在顺序队列中，当队尾指针 rear 到达数组上界时，即使前面有空闲位置，也不能再做入队操作，这种现象称为"假溢出"。

**解决方法：循环队列（或链队列）**

#### 循环队列 ⚠️（高频考点）

将顺序队列臆造为一个环状的空间，即把存储队列元素的表从逻辑上视为一个环，称为循环队列。

```c
#define MaxSize 50
typedef struct {
    ElemType data[MaxSize];
    int front, rear;
} SqQueue;
```

**初始化：**
```c
void InitQueue(SqQueue &Q) {
    Q.rear = Q.front = 0;
}
```

**判空：**
```c
bool QueueEmpty(SqQueue Q) {
    return Q.rear == Q.front;
}
```

⚠️ **循环队列的几个关键问题（必考）：**

**① 牺牲一个存储单元**

队满条件：`(Q.rear + 1) % MaxSize == Q.front`

队列元素个数：`(Q.rear - Q.front + MaxSize) % MaxSize`

**② 增设 size 成员**

```c
typedef struct {
    ElemType data[MaxSize];
    int front, rear;
    int size;  // 记录元素个数
} SqQueue;
```
队满：`Q.size == MaxSize`
队空：`Q.size == 0`

**③ 增设 tag 成员**

```c
typedef struct {
    ElemType data[MaxSize];
    int front, rear;
    int tag;  // 最近一次操作：0为删除，1为插入
} SqQueue;
```
队满：`Q.front == Q.rear && Q.tag == 1`
队空：`Q.front == Q.rear && Q.tag == 0`

**入队操作（牺牲一个存储单元方案）：**
```c
bool EnQueue(SqQueue &Q, ElemType x) {
    if ((Q.rear + 1) % MaxSize == Q.front)
        return false;  // 队满
    Q.data[Q.rear] = x;
    Q.rear = (Q.rear + 1) % MaxSize;
    return true;
}
```

**出队操作：**
```c
bool DeQueue(SqQueue &Q, ElemType &x) {
    if (Q.rear == Q.front)
        return false;  // 队空
    x = Q.data[Q.front];
    Q.front = (Q.front + 1) % MaxSize;
    return true;
}
```

### 3.2.4 队列的链式存储结构（链队列）

```c
typedef struct LinkNode {
    ElemType data;
    struct LinkNode *next;
} LinkNode;

typedef struct {
    LinkNode *front, *rear;  // 队头和队尾指针
} LinkQueue;
```

**初始化（带头结点）：**
```c
void InitQueue(LinkQueue &Q) {
    Q.front = Q.rear = (LinkNode*)malloc(sizeof(LinkNode));
    Q.front->next = NULL;
}
```

**判空：**
```c
bool QueueEmpty(LinkQueue Q) {
    return Q.front == Q.rear;
}
```

**入队：**
```c
void EnQueue(LinkQueue &Q, ElemType x) {
    LinkNode *s = (LinkNode*)malloc(sizeof(LinkNode));
    s->data = x;
    s->next = NULL;
    Q.rear->next = s;
    Q.rear = s;
}
```

**出队：**
```c
bool DeQueue(LinkQueue &Q, ElemType &x) {
    if (Q.front == Q.rear) return false;  // 空队
    LinkNode *p = Q.front->next;
    x = p->data;
    Q.front->next = p->next;
    if (Q.rear == p)  // ⚠️ 若原队列中只有一个结点，删除后队列变空
        Q.rear = Q.front;
    free(p);
    return true;
}
```

⚠️ **易错点：** 出队操作时，如果删除的是最后一个结点，需要同时修改尾指针 rear，使其重新指向头结点。

## 3.3 栈和队列的应用

### 3.3.1 栈在括号匹配中的应用

算法思想：遇到左括号入栈，遇到右括号则与栈顶的左括号匹配。若匹配则弹出栈顶，否则匹配失败。最后若栈为空则匹配成功。

```c
bool BracketCheck(char *str) {
    SqStack S;
    InitStack(S);
    int i = 0;
    while (str[i] != '\0') {
        if (str[i] == '(' || str[i] == '[' || str[i] == '{')
            Push(S, str[i]);
        else if (str[i] == ')' || str[i] == ']' || str[i] == '}') {
            char top;
            if (StackEmpty(S)) return false;
            Pop(S, top);
            if ((str[i] == ')' && top != '(') ||
                (str[i] == ']' && top != '[') ||
                (str[i] == '}' && top != '{'))
                return false;
        }
        i++;
    }
    return StackEmpty(S);
}
```

### 3.3.2 栈在表达式求值中的应用 ⚠️（高频考点）

#### 中缀表达式 → 后缀表达式

**算法思想：**
1. 从左到右扫描中缀表达式
2. 遇到操作数直接输出
3. 遇到操作符：
   - 若栈空或栈顶为 `(`，直接入栈
   - 若当前操作符优先级 **>** 栈顶操作符优先级，入栈
   - 若当前操作符优先级 **≤** 栈顶操作符优先级，弹出栈顶输出，直到满足入栈条件
   - 遇到 `(` 直接入栈
   - 遇到 `)` 弹出栈顶输出，直到遇到 `(`（`(` 弹出但不输出）
4. 扫描结束后，弹出栈中所有操作符

**操作符优先级：**

| 操作符 | 优先级 |
|--------|--------|
| (       | 1（最低） |
| +, -   | 2 |
| *, /   | 3 |

**例：** 将 `a+b*c+(d*e+f)*g` 转换为后缀表达式

| 步骤 | 扫描字符 | 栈 | 输出 |
|------|---------|-----|------|
| 1 | a | | a |
| 2 | + | + | a |
| 3 | b | + | ab |
| 4 | * | +* | ab |
| 5 | c | +* | abc |
| 6 | + | + | abc*+ |
| 7 | ( | +( | abc*+ |
| 8 | d | +( | abc*+d |
| 9 | * | +(* | abc*+d |
| 10 | e | +(* | abc*+de |
| 11 | + | +(+ | abc*+de* |
| 12 | f | +(+ | abc*+de*f |
| 13 | ) | + | abc*+de*f+ |
| 14 | * | +* | abc*+de*f+ |
| 15 | g | +* | abc*+de*f+g |
| 结束 | | | abc*+de*f+g*+ |

**答案：** 后缀表达式为 `abc*+de*f+g*+`

#### 后缀表达式求值

**算法思想：**
1. 从左到右扫描后缀表达式
2. 遇到操作数则入栈
3. 遇到操作符则弹出栈顶两个元素（先弹出的是右操作数），进行运算后将结果入栈
4. 最后栈中的唯一元素即为表达式的值

**例：** 求 `123*+45*6+*+` 的值

设表达式中各数字均为一位数字。

| 步骤 | 扫描字符 | 栈 |
|------|---------|-----|
| 1 | 1 | 1 |
| 2 | 2 | 1, 2 |
| 3 | 3 | 1, 2, 3 |
| 4 | * | 1, 6 |
| 5 | + | 7 |
| 6 | 4 | 7, 4 |
| 7 | 5 | 7, 4, 5 |
| 8 | * | 7, 20 |
| 9 | 6 | 7, 20, 6 |
| 10 | + | 7, 26 |
| 11 | * | 182 |
| 12 | + | 189 |

**答案：** 189

#### 前缀表达式求值

与后缀表达式类似，但从**右到左**扫描。遇到操作数入栈，遇到操作符弹出两个操作数（先弹出的是左操作数）。

### 3.3.3 栈在递归中的应用

递归是一种重要的编程技术。在递归调用过程中，系统自动用栈来保存调用现场信息：
- 每次调用时，系统将实参、局部变量及返回地址入栈
- 每次返回时，系统从栈顶弹出相应信息，恢复调用现场

⚠️ **递归转非递归：** 所有递归算法都可以借助栈转换为等价的非递归算法。

### 3.3.4 队列在层次遍历中的应用

在树的层次遍历和图的广度优先搜索（BFS）中，队列是核心数据结构：

```c
// 二叉树层次遍历伪代码
void LevelOrder(BiTree T) {
    LinkQueue Q;
    InitQueue(Q);
    BiTree p;
    EnQueue(Q, T);
    while (!QueueEmpty(Q)) {
        DeQueue(Q, p);
        visit(p);
        if (p->lchild != NULL) EnQueue(Q, p->lchild);
        if (p->rchild != NULL) EnQueue(Q, p->rchild);
    }
}
```

### 3.3.5 队列在计算机系统中的应用

- 解决主机与外部设备之间速度不匹配的问题（如打印机缓冲区）
- 解决多用户引起的资源竞争问题（如CPU调度）

## 3.4 特殊矩阵的压缩存储 ⚠️（高频考点）

### 3.4.1 数组的定义与存储

数组是由 n (n ≥ 1) 个相同类型的数据元素构成的有限序列。

**一维数组**的存储地址计算：
$$LOC(a_i) = LOC(a_0) + i \times L$$

其中 L 是每个数据元素占用的存储单元。

**二维数组**的存储（以行优先为例）：
$$LOC(a_{i,j}) = LOC(a_{0,0}) + (i \times n + j) \times L$$

其中 n 为每行的元素个数。

### 3.4.2 对称矩阵

若 n 阶矩阵 A 中的元素满足 aᵢⱼ = aⱼᵢ (1 ≤ i, j ≤ n)，则称为对称矩阵。

⚠️ **压缩存储方法：** 只存储下三角（或上三角）部分，共 n(n+1)/2 个元素。

下标对应关系（下三角存储，行优先）：

对于 aᵢⱼ (i ≥ j)，在一维数组中的位置为：

$$k = \frac{i(i-1)}{2} + j - 1 \quad (i \geq j, 从1开始)$$

若从0开始编号：
$$k = \frac{i(i-1)}{2} + j - 1$$

对于 aᵢⱼ (i < j)，利用对称性 aᵢⱼ = aⱼᵢ，等价于找 aⱼᵢ 的位置。

### 3.4.3 三角矩阵

**下三角矩阵：** 上三角（不含对角线）元素全为常数 c。
- 存储方法：将下三角部分逐行存入一维数组，最后一个位置存放常数 c。
- 下标对应：k = i(i-1)/2 + j - 1（i ≥ j）；i < j 时 k = n(n+1)/2

**上三角矩阵：** 下三角（不含对角线）元素全为常数 c。
- 存储方法：将上三角部分逐列（或逐行）存入一维数组。
- 逐列存储时 k = (j-1)(2n-j+2)/2 + (i-j)（i ≤ j）

### 3.4.4 三对角矩阵（带状矩阵）

n 阶三对角矩阵满足：当 |i - j| > 1 时，aᵢⱼ = 0。

⚠️ **压缩存储：** 将三条对角线上的元素按行优先存入一维数组。

下标对应关系（从0开始编号）：
$$k = 2i + j \quad (|i-j| \leq 1)$$

反推：给定 k，i = (k+1)/3（向下取整），j = k - 2i

### 3.4.5 稀疏矩阵

非零元素个数远小于矩阵元素总个数的矩阵。

**存储方法：**

**① 三元组表**
```c
typedef struct {
    int i, j;      // 行下标和列下标
    ElemType v;    // 元素值
} Triple;

typedef struct {
    Triple data[MaxSize];  // 非零元素三元组表
    int mu, nu, tu;        // 矩阵的行数、列数和非零元素个数
} TSMatrix;
```

**② 十字链表法**

每个非零元素用一个结点表示，同时有行指针和列指针。

⚠️ **注意：** 稀疏矩阵的压缩存储会失去随机存取的特性。

## 3.5 本章例题

### 例题1：循环队列判断队满（2011年真题改编）

已知循环队列存储在一维数组 A[0..n-1] 中，且队列非空时 front 和 rear 分别指向队头元素和队尾元素的下一个位置。初始时 front = rear = 0。设队列的最大长度为 n，问：
（1）队列中元素个数？
（2）队满条件？

**答案：**
（1）元素个数 = `(rear - front + n) % n`
（2）队满条件：`(rear + 1) % n == front`（牺牲一个存储单元区分队空和队满）

### 例题2：中缀转后缀并求值

将中缀表达式 `(a+b)*c-d/e` 转换为后缀表达式，并写出求值过程（假设 a=1, b=2, c=3, d=8, e=4）。

**答案：**

后缀表达式：`ab+c*de/-`

求值过程：
| 步骤 | 扫描 | 栈 |
|------|------|-----|
| 1 | a=1 | 1 |
| 2 | b=2 | 1, 2 |
| 3 | + | 3 |
| 4 | c=3 | 3, 3 |
| 5 | * | 9 |
| 6 | d=8 | 9, 8 |
| 7 | e=4 | 9, 8, 4 |
| 8 | / | 9, 2 |
| 9 | - | 7 |

**结果：7**

### 例题3：栈的出栈序列（经典高频考题）

设一个栈的输入序列为 1, 2, 3, 4, 5，则下列序列中，不可能是栈的输出序列的是（  ）。

A. 5, 4, 3, 2, 1  B. 4, 5, 3, 2, 1  C. 4, 3, 5, 1, 2  D. 1, 2, 3, 4, 5

**答案：C**

**解析：** 对于选项 C，若输出 4，说明 1,2,3 已入栈（栈中从底到顶为 1,2,3,4），弹出 4 后栈中为 1,2,3。接着输出 3,5 意味着弹出 3，然后 5 入栈再弹出。此时栈中为 1,2。接下来应输出 2，但选项给的是 1，这是不可能的（栈顶是 2，必须先弹出 2 才能弹出 1）。

**判断出栈序列的技巧：** 对于入栈序列 a₁, a₂, ..., aₙ，输出中若出现 aⱼ 之后仍出现 aᵢ（i < j），则 aᵢ 的入栈和出栈之间不能有任何比 aᵢ 小且比 aⱼ 大的元素在 aᵢ 前面出栈。

### 例题4：用两个栈模拟队列（2018年真题改编）

用两个栈 S1 和 S2 模拟一个队列，写出入队和出队算法。

**答案：**

```c
void EnQueue(SqStack &S1, SqStack &S2, ElemType x) {
    if (!StackFull(S1))
        Push(S1, x);
}

bool DeQueue(SqStack &S1, SqStack &S2, ElemType &x) {
    if (StackEmpty(S2)) {
        if (StackEmpty(S1)) return false;
        // 将S1中所有元素倒入S2
        while (!StackEmpty(S1)) {
            ElemType e;
            Pop(S1, e);
            Push(S2, e);
        }
    }
    Pop(S2, x);
    return true;
}
```

**解析：** S1 作为入队栈，S2 作为出队栈。出队时若 S2 为空，将 S1 中所有元素倒入 S2（此时顺序正好变为先进先出）。只有 S2 为空时才进行倒入操作，均摊时间复杂度为 O(1)。

### 例题5：循环队列元素个数（2015年真题改编）

循环队列放在一维数组 A[0..M-1] 中，end1 指向队头元素，end2 指向队尾元素的后一个位置。假设队列两端均可进行入队和出队操作。队列中最多能容纳 M-1 个元素。初始时队列为空。

写出队列元素个数的计算公式。

**答案：**

队列元素个数 = `(end2 - end1 + M) % M`

### 例题6：对称矩阵压缩存储计算

已知 n 阶对称矩阵 A 以行优先方式将下三角部分存储在一维数组 B[0..n(n+1)/2-1] 中。对于元素 aᵢⱼ（i, j 从 1 开始），写出在 B 中的下标 k 的表达式。

**答案：**

$$k = \frac{i(i-1)}{2} + j - 1 \quad (i \geq j)$$

当 i < j 时，利用对称性 aᵢⱼ = aⱼᵢ：

$$k = \frac{j(j-1)}{2} + i - 1 \quad (i < j)$$
