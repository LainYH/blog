# 第8章 排序

## 8.1 排序的基本概念

### 8.1.1 排序的定义

排序（Sorting）就是重新排列表中的元素，使表中的元素满足按关键字有序的过程。

### 8.1.2 基本术语

| 术语 | 定义 |
|------|------|
| **稳定性** | 若待排序表中有两个元素 Rᵢ 和 Rⱼ，其对应的关键字相同即 keyᵢ = keyⱼ，且在排序前 Rᵢ 在 Rⱼ 前面，若使用某一排序算法排序后，Rᵢ 仍在 Rⱼ 的前面，则称这个排序算法是**稳定的**，否则称排序算法是**不稳定的** |
| **就地排序** | 排序过程中，算法所需的辅助空间为 O(1)，则称为就地排序（或原地排序） |
| **内部排序** | 排序期间元素全部存放在内存中的排序 |
| **外部排序** | 排序期间元素无法全部同时存放在内存中，必须在排序过程中根据要求不断地在内、外存之间移动的排序 |

### 8.1.3 排序算法的分类

| 类别 | 算法 |
|------|------|
| **插入排序** | 直接插入排序、折半插入排序、希尔排序 |
| **交换排序** | 冒泡排序、快速排序 |
| **选择排序** | 简单选择排序、堆排序 |
| **归并排序** | 二路归并排序 |
| **基数排序** | 基数排序 |

## 8.2 插入排序

### 8.2.1 直接插入排序 ⭐

**基本思想：** 每次将一个待排序的元素按其关键字大小插入到前面已排好序的子序列中。

```c
void InsertSort(int A[], int n) {
    int i, j, temp;
    for (i = 1; i < n; i++) {  // 从第二个元素开始
        if (A[i] < A[i - 1]) {
            temp = A[i];        // 暂存待插入元素
            for (j = i - 1; j >= 0 && A[j] > temp; j--)
                A[j + 1] = A[j];  // 后移
            A[j + 1] = temp;      // 插入
        }
    }
}
```

⚠️ **性能分析：**

| 情况 | 时间复杂度 | 说明 |
|------|-----------|------|
| 最好 | **O(n)** | 表已有序，每趟只需比较1次 |
| 最坏 | **O(n²)** | 表逆序，每趟需比较 i 次 |
| 平均 | **O(n²)** | |
| 空间 | **O(1)** | 就地排序 |
| 稳定性 | **稳定** | |

⚠️ **稳定性分析：** 直接插入排序是稳定的，因为只有当 A[i] < A[j] 时才移动（严格小于才移动，等于时不移动）。

### 8.2.2 折半插入排序

在直接插入排序中，插入时用**折半查找**来确定插入位置。

```c
void BinaryInsertSort(int A[], int n) {
    int i, j, low, high, mid, temp;
    for (i = 1; i < n; i++) {
        temp = A[i];
        low = 0; high = i - 1;
        // 折半查找插入位置
        while (low <= high) {
            mid = (low + high) / 2;
            if (A[mid] > temp)
                high = mid - 1;
            else
                low = mid + 1;
        }
        // 移动元素
        for (j = i - 1; j >= low; j--)
            A[j + 1] = A[j];
        A[low] = temp;
    }
}
```

⚠️ **折半插入排序减少了比较次数，但移动次数不变。时间复杂度仍为 O(n²)。**

⚠️ **稳定性：稳定。**

### 8.2.3 希尔排序（Shell Sort）⭐

**基本思想：** 先将待排序表分割为若干形如 L[i, i+d, i+2d, ..., i+kd] 的"特殊"子表，分别进行直接插入排序，当整个表已基本有序时，再对全体记录进行一次直接插入排序。

**增量（步长）序列：** d₁, d₂, ..., dₜ，其中 d₁ = n/2，dᵢ₊₁ = dᵢ/2（向下取整），最后一个增量为 1。

```c
void ShellSort(int A[], int n) {
    int d, i, j, temp;
    for (d = n / 2; d >= 1; d = d / 2) {  // 步长变化
        for (i = d; i < n; i++) {
            if (A[i] < A[i - d]) {
                temp = A[i];
                for (j = i - d; j >= 0 && A[j] > temp; j -= d)
                    A[j + d] = A[j];
                A[j + d] = temp;
            }
        }
    }
}
```

⚠️ **性能分析：**

| 维度 | 说明 |
|------|------|
| 时间复杂度 | 与增量序列有关，**最坏 O(n²)**，当增量为 d = 2^k - 1 时可达 O(n^1.5) |
| 空间复杂度 | **O(1)** |
| 稳定性 | **不稳定**（相同关键字的元素可能被划分到不同子表中） |
| 适用性 | 适用于中等规模的数据 |

## 8.3 交换排序

### 8.3.1 冒泡排序 ⭐

**基本思想：** 从前往后两两比较相邻元素的值，若为逆序则交换，直到序列比较完。每趟将最大的元素"冒泡"到末尾。

```c
void BubbleSort(int A[], int n) {
    for (int i = 0; i < n - 1; i++) {
        bool flag = false;  // 本趟是否发生交换的标志
        for (int j = n - 1; j > i; j--)
            if (A[j - 1] > A[j]) {
                swap(A[j - 1], A[j]);
                flag = true;
            }
        if (!flag) return;  // 本趟未发生交换，说明已有序
    }
}
```

⚠️ **性能分析：**

| 情况 | 时间复杂度 | 说明 |
|------|-----------|------|
| 最好 | **O(n)** | 表已有序，一趟就结束（flag检测） |
| 最坏 | **O(n²)** | 表逆序 |
| 平均 | **O(n²)** | |
| 空间 | **O(1)** | 就地排序 |
| 稳定性 | **稳定** | 相等元素不会交换 |

### 8.3.2 快速排序（Quick Sort）⭐⭐（最重要的排序算法）

**基本思想：** 基于分治法。在待排序表中任取一个元素 pivot 作为**基准（枢轴）**，通过一趟排序将待排序表划分为独立的两部分 L[1..k-1] 和 L[k+1..n]，使得 L[1..k-1] 中所有元素小于 pivot，L[k+1..n] 中所有元素大于等于 pivot，则 pivot 放在了其最终位置 L[k] 上。然后分别递归地对两个子表重复上述过程。

```c
int Partition(int A[], int low, int high) {
    int pivot = A[low];  // 以第一个元素为基准
    while (low < high) {
        while (low < high && A[high] >= pivot) high--;
        A[low] = A[high];       // 将比基准小的元素移到左端
        while (low < high && A[low] <= pivot) low++;
        A[high] = A[low];       // 将比基准大的元素移到右端
    }
    A[low] = pivot;             // 基准元素放到最终位置
    return low;
}

void QuickSort(int A[], int low, int high) {
    if (low < high) {
        int pivotpos = Partition(A, low, high);  // 划分
        QuickSort(A, low, pivotpos - 1);         // 递归处理左子表
        QuickSort(A, pivotpos + 1, high);         // 递归处理右子表
    }
}
```

⚠️ **性能分析：**

| 情况 | 时间复杂度 | 说明 |
|------|-----------|------|
| 最好 | **O(nlog₂n)** | 每次划分均匀，基准在中间 |
| 最坏 | **O(n²)** | 每次划分极端不均匀（如表已有序，选第一个为基准） |
| 平均 | **O(nlog₂n)** | |
| 空间 | 最好 **O(log₂n)**，最坏 **O(n)** | 递归栈空间 |
| 稳定性 | **不稳定** | |

⚠️ **快排的优化：**
- 随机选取基准（Randomized QuickSort）
- 三数取中法：取 low、mid、high 三个位置的元素的中位数作为基准
- 小区间改用插入排序

⚠️ **快排是所有内部排序算法中平均性能最好的！**

⚠️ **快排的递归深度：** 最好 O(log₂n)，最坏 O(n)。

⚠️ **快排 vs 其他排序：**
- 快排一趟排序后，**基准元素在最终位置上**
- 快排每一趟都能确定一个元素的最终位置
- n 个元素的快排至少需要 ⌈log₂n⌉ 趟（最好情况）

## 8.4 选择排序

### 8.4.1 简单选择排序 ⭐

**基本思想：** 每一趟从待排序的元素中选出关键字最小的元素，与当前未排序部分的第一个元素交换。

```c
void SelectSort(int A[], int n) {
    for (int i = 0; i < n - 1; i++) {
        int min = i;
        for (int j = i + 1; j < n; j++)
            if (A[j] < A[min])
                min = j;
        if (min != i)
            swap(A[i], A[min]);
    }
}
```

⚠️ **性能分析：**

| 维度 | 说明 |
|------|------|
| 时间复杂度 | **最好、最坏、平均都是 O(n²)**（比较次数始终为 n(n-1)/2） |
| 空间复杂度 | **O(1)** |
| 稳定性 | **不稳定**（交换可能改变相同关键字的相对顺序） |

例：序列 {5, 5, 3}，第一趟选择 3 与第一个 5 交换，两个 5 的相对顺序改变。

### 8.4.2 堆排序 ⭐⭐（高频考点）

#### 堆的定义

n 个元素的序列 {k₁, k₂, ..., kₙ}，当且仅当满足以下关系时，称为**堆**：

**大根堆（大顶堆）：** kᵢ ≥ k₂ᵢ 且 kᵢ ≥ k₂ᵢ₊₁（对于所有 i）
**小根堆（小顶堆）：** kᵢ ≤ k₂ᵢ 且 kᵢ ≤ k₂ᵢ₊₁（对于所有 i）

⚠️ **将堆看作一棵完全二叉树，则：**
- **大根堆：** 每个结点的值 ≥ 其左右孩子的值
- **小根堆：** 每个结点的值 ≤ 其左右孩子的值

#### 建立大根堆 ⭐

```c
void BuildMaxHeap(int A[], int len) {
    for (int i = len / 2; i >= 1; i--)  // 从最后一个非叶子结点开始
        HeadAdjust(A, i, len);
}

void HeadAdjust(int A[], int k, int len) {
    A[0] = A[k];  // 暂存子树的根结点
    for (int i = 2 * k; i <= len; i *= 2) {
        if (i < len && A[i] < A[i + 1])  // 取较大孩子的下标
            i++;
        if (A[0] >= A[i]) break;         // 根结点已经大于较大孩子
        else {
            A[k] = A[i];  // 将较大孩子上移
            k = i;         // 修改k值，继续向下筛选
        }
    }
    A[k] = A[0];  // 将被筛选结点放入最终位置
}
```

⚠️ **建堆的时间复杂度：O(n)**

#### 堆排序过程

```c
void HeapSort(int A[], int len) {
    BuildMaxHeap(A, len);           // 建立初始大根堆
    for (int i = len; i > 1; i--) {
        swap(A[i], A[1]);          // 将堆顶元素（最大值）与末尾交换
        HeadAdjust(A, 1, i - 1);   // 调整剩余元素为大根堆
    }
}
```

⚠️ **堆排序性能分析：**

| 维度 | 说明 |
|------|------|
| 建堆时间 | **O(n)** |
| 排序时间 | 每次调整 O(log₂n)，共 n-1 次，总计 **O(nlog₂n)** |
| 最好/最坏/平均 | 都是 **O(nlog₂n)** |
| 空间复杂度 | **O(1)** |
| 稳定性 | **不稳定** |

⚠️ **堆排序的特点：**
- 适合**大数据量**排序（时间复杂度稳定在 O(nlog₂n)）
- 适合**Top-K 问题**（用小根堆求最大的 K 个元素）
- 不适合小数据量（常数因子较大）

⚠️ **重要性质：**
- 小根堆的堆顶元素是最小值
- 大根堆的堆顶元素是最大值
- 一个完全二叉树有 ⌊n/2⌋ 个叶子结点（不需要调整）

## 8.5 归并排序 ⭐

**基本思想：** 将两个或两个以上的有序表组合成一个新的有序表。

### 8.5.1 二路归并

```c
int *B = (int *)malloc(n * sizeof(int));  // 辅助数组

void Merge(int A[], int low, int mid, int high) {
    int i, j, k;
    // 将A中元素复制到B
    for (k = low; k <= high; k++)
        B[k] = A[k];
    // 归并
    for (i = low, j = mid + 1, k = low; i <= mid && j <= high; k++) {
        if (B[i] <= B[j])  // ⚠️ <=保证稳定性
            A[k] = B[i++];
        else
            A[k] = B[j++];
    }
    // 处理剩余元素
    while (i <= mid) A[k++] = B[i++];
    while (j <= high) A[k++] = B[j++];
}
```

### 8.5.2 归并排序

```c
void MergeSort(int A[], int low, int high) {
    if (low < high) {
        int mid = (low + high) / 2;
        MergeSort(A, low, mid);       // 对左半部分归并排序
        MergeSort(A, mid + 1, high);  // 对右半部分归并排序
        Merge(A, low, mid, high);     // 归并
    }
}
```

⚠️ **归并排序性能分析：**

| 维度 | 说明 |
|------|------|
| 时间复杂度 | **最好、最坏、平均都是 O(nlog₂n)** |
| 空间复杂度 | **O(n)**（需要辅助数组） |
| 稳定性 | **稳定**（`B[i] <= B[j]` 用的是 `<=`） |

⚠️ **归并排序的递归深度：O(log₂n)**

⚠️ **注意：** 2-路归并排序的趟数 = ⌈log₂n⌉

## 8.6 基数排序 ⭐

**基本思想：** 采用多关键字排序思想，借助"分配"和"收集"两种操作对单逻辑关键字进行排序。

### 8.6.1 最低位优先（LSD）法

按关键字权重递增依次进行排序，先按最低位排序，再按次低位排序，直到最高位。

### 8.6.2 基数排序过程

```c
#define RADIX 10  // 基数（十进制）

// 基数排序（链式队列实现）
void RadixSort(int A[], int n) {
    // 初始化 r 个链队列
    LinkQueue Q[RADIX];
    for (int i = 0; i < RADIX; i++)
        InitQueue(Q[i]);

    int d = GetDigitCount(A, n);  // 获取最大关键字的位数

    for (int i = 0; i < d; i++) {  // 进行 d 趟分配和收集
        // 分配：按第 i 位数字入队
        for (int j = 0; j < n; j++) {
            int digit = GetDigit(A[j], i);
            EnQueue(Q[digit], A[j]);
        }
        // 收集：按顺序出队
        int k = 0;
        for (int j = 0; j < RADIX; j++)
            while (!QueueEmpty(Q[j]))
                DeQueue(Q[j], A[k++]);
    }
}
```

⚠️ **基数排序性能分析：**

| 维度 | 说明 |
|------|------|
| 时间复杂度 | **O(d(n+r))**，d 为关键字位数，r 为基数，n 为元素个数 |
| 空间复杂度 | **O(r)**，需要 r 个辅助队列 |
| 稳定性 | **稳定** |

⚠️ **基数排序的特点：**
- 是一种**不基于比较**的排序算法
- 适用于关键字可以分解为多个位/字段的场景
- 不适用于实数排序（因为无法确定"位数"）

## 8.7 各种排序算法的比较 ⚠️（高频考点）

### 8.7.1 时间复杂度汇总

| 排序算法 | 最好 | 平均 | 最坏 | 空间 | 稳定性 |
|---------|------|------|------|------|--------|
| **直接插入排序** | O(n) | O(n²) | O(n²) | O(1) | 稳定 |
| **冒泡排序** | O(n) | O(n²) | O(n²) | O(1) | 稳定 |
| **简单选择排序** | O(n²) | O(n²) | O(n²) | O(1) | 不稳定 |
| **希尔排序** | O(n) | O(n^1.3) | O(n²) | O(1) | 不稳定 |
| **快速排序** | O(nlog₂n) | O(nlog₂n) | O(n²) | O(log₂n) | 不稳定 |
| **堆排序** | O(nlog₂n) | O(nlog₂n) | O(nlog₂n) | O(1) | 不稳定 |
| **归并排序** | O(nlog₂n) | O(nlog₂n) | O(nlog₂n) | O(n) | 稳定 |
| **基数排序** | O(d(n+r)) | O(d(n+r)) | O(d(n+r)) | O(r) | 稳定 |

### 8.7.2 选择排序算法的原则

| 场景 | 推荐算法 | 原因 |
|------|---------|------|
| n 较小 | 直接插入排序或简单选择排序 | 实现简单，常数因子小 |
| n 较大 | 快速排序 | 平均性能最好 |
| n 较大且要求稳定 | 归并排序 | 时间 O(nlog₂n) 且稳定 |
| n 较大且空间受限 | 堆排序 | 时间 O(nlog₂n) 且空间 O(1) |
| 基本有序 | 直接插入排序或冒泡排序 | 接近 O(n) |
| 关键字位数少且可分解 | 基数排序 | O(d(n+r)) |

### 8.7.3 重要结论 ⚠️

1. **基于比较的排序算法的下界是 O(nlog₂n)**，即不存在时间复杂度低于 O(nlog₂n) 的基于比较的排序算法
2. **快排的最坏情况：** 每次选的基准都是最大或最小元素（如表已有序时选第一个元素为基准），退化为 O(n²)
3. **稳定性判断技巧：**
   - 稳定的：直接插入排序、冒泡排序、归并排序、基数排序
   - 不稳定的：简单选择排序、希尔排序、快速排序、堆排序
   - **记忆口诀：** "快些选堆"（快排、希尔、选择、堆排序——不稳定）
4. **一趟排序后，** 快排能确定基准的最终位置；冒泡排序能确定最大/最小元素的位置；简单选择排序能确定最大/最小元素的位置
5. **归并排序是外部排序的基础算法**

## 8.8 外部排序

### 8.8.1 基本概念

当待排序的记录数量太大，无法一次全部调入内存时，需要利用外存进行多次内部排序，然后将这些有序子文件合并得到最终的有序文件。

### 8.8.2 外部排序的过程

1. 将待排序文件分成若干个能一次装入内存的子文件
2. 对每个子文件进行内部排序，得到初始归并段（顺串）
3. 将这些初始归并段逐趟合并成更大的归并段

### 8.8.3 多路平衡归并 ⭐

**k 路平衡归并：** 每次从 k 个归并段中选择一个关键字最小的记录输出。

⚠️ **败者树（Loser Tree）：** 用于 k 路归并中快速选择最小关键字。时间复杂度从 O(k) 降低到 O(log₂k)。

### 8.8.4 败者树

败者树是一棵完全二叉树，叶子结点存储各归并段当前参加比较的记录，内部结点存储其两个子结点中"败者"（关键字较大者）的段号。根结点上方还有一个额外的结点存储最终的胜者。

⚠️ **败者树的时间复杂度：** 每次选择最小元素需要 **O(log₂k)**（只需沿败者树的一条路径向上比较）。

### 8.8.5 置换-选择排序

用于生成更长的初始归并段。基本思想：在内存中选择一个最小的关键字写入输出归并段，然后从输入文件中读入一个新记录，如果新记录的关键字 ≥ 刚输出的关键字，则将新记录加入候选集（可以归入当前归并段），否则将其冻结（留到下一个归并段）。

⚠️ **使用置换-选择排序生成的初始归并段长度不等，平均长度为内存工作区大小的 2 倍。**

### 8.8.6 最佳归并树

将初始归并段组织成一棵**哈夫曼树**，使 I/O 次数最少。

⚠️ **对于 k 路归并，若初始归并段数目 m 不满足 m-1 是 k-1 的倍数，需要补充长度为 0 的"虚段"。**

用快速排序对序列 {49, 38, 65, 97, 76, 13, 27, 49'} 进行递增排序（以第一个元素为基准），写出每一趟的结果。

### 例题2：堆排序（2015年真题改编）

将序列 {5, 3, 1, 7, 2, 4, 6} 调整为大根堆。

**解析：**

从最后一个非叶子结点（⌊7/2⌋ = 3）开始向下调整：

```
初始完全二叉树：
        5
       / \
      3    1
     / \  / \
    7  2 4   6
```

- 结点 3（值1）：孩子是4和6，6>1，交换 → 不对，结点3（下标3）的孩子是下标6和7，值为4和6。等等，让我重新排列。

用数组下标从1开始：A[1]=5, A[2]=3, A[3]=1, A[4]=7, A[5]=2, A[6]=4, A[7]=6

```
        5(1)
       /     \
     3(2)     1(3)
    /   \    /   \
  7(4) 2(5) 4(6) 6(7)
```

i=3: 调整结点 A[3]=1，孩子 A[6]=4, A[7]=6，最大是6，交换 → A[3]=6, A[7]=1
```
        5
       / \
      3    6
     / \  / \
    7  2 4   1
```

i=2: 调整结点 A[2]=3，孩子 A[4]=7, A[5]=2，最大是7，交换 → A[2]=7, A[4]=3
```
        5
       / \
      7    6
     / \  / \
    3  2 4   1
```

i=1: 调整结点 A[1]=5，孩子 A[2]=7, A[3]=6，最大是7，交换 → A[1]=7, A[2]=5

```
        7
       / \
      5    6
     / \  / \
    3  2 4   1
```

### 例题3：排序稳定性判断（高频考题）

下列排序算法中，稳定的是（  ）。

A. 快速排序  B. 希尔排序  C. 堆排序  D. 归并排序

**答案：D**

**解析：** 快速排序、希尔排序、堆排序都是不稳定的。归并排序是稳定的。记忆口诀"快些选堆"对应不稳定的四种排序。

### 例题4：排序算法的时间复杂度（2011年真题改编）

下列排序算法中，每一趟排序结束后至少能确定一个元素最终位置的是（  ）。

A. 归并排序  B. 希尔排序  C. 冒泡排序  D. 直接插入排序

**答案：C**

**解析：**
- 快速排序：每一趟确定基准元素的最终位置 ✓
- 冒泡排序：每一趟确定最大（或最小）元素的最终位置 ✓
- 简单选择排序：每一趟确定最小元素的最终位置 ✓
- 直接插入排序：不能保证每一趟确定一个元素的最终位置 ✗
- 归并排序：不能保证每一趟确定一个元素的最终位置 ✗

（若选项中有快排，也是正确答案。这道题的关键是 C 和可能有的快排。）

### 例题5：n个元素排序的最少比较次数

对 n 个元素进行排序，在最坏情况下，基于比较的排序算法最少需要进行多少次比较？

**答案：** ⌈log₂(n!)⌉

**解析：** 基于比较的排序可以看作一棵决策树，n 个元素有 n! 种排列，决策树至少有 n! 个叶子结点。高度为 h 的二叉树最多有 2^h 个叶子结点，所以 2^h ≥ n!，h ≥ log₂(n!)。根据 Stirling 公式，log₂(n!) ≈ nlog₂n。

### 例题6：快排的最坏情况优化

对于已基本有序的序列，如何优化快排使其避免最坏情况？

**答案：**

1. **随机选取基准：** `swap(A[low], A[rand()%(high-low+1)+low])`
2. **三数取中：** 取 A[low]、A[mid]、A[high] 三者的中位数作为基准
3. **当子序列长度小于某个阈值时改用插入排序**

### 例题7：外部排序的归并趟数

设有 10000 个初始归并段，做 5 路归并，需要几趟？

**答案：**

需要 ⌈log₅10000⌉ = ⌈log₁₀10000/log₁₀5⌉ = ⌈4/0.699⌉ = ⌈5.72⌉ = **6 趟**

⚠️ **一般公式：k 路归并 m 个初始归并段，需要 ⌈logₖm⌉ 趟。**
