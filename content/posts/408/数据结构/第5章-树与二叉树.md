# 第5章 树与二叉树

## 5.1 树的基本概念

### 5.1.1 树的定义

树是 n (n ≥ 0) 个结点的有限集。当 n = 0 时，称为**空树**。在任意一棵非空树中：
- 有且仅有一个特定的称为**根（Root）**的结点
- 当 n > 1 时，其余结点可分为 m (m > 0) 个互不相交的有限集 T₁, T₂, ..., Tₘ，其中每个集合本身又是一棵树，并且称为根的**子树（SubTree）**

⚠️ **树的定义是递归的**——树的定义中又用到了树的概念。

### 5.1.2 基本术语

| 术语 | 定义 |
|------|------|
| **结点的度** | 结点拥有的子树个数 |
| **树的度** | 树中所有结点的度的最大值 |
| **叶子结点（终端结点）** | 度为 0 的结点 |
| **分支结点（非终端结点）** | 度不为 0 的结点 |
| **孩子/双亲** | 结点的子树的根称为该结点的孩子，该结点称为孩子的双亲 |
| **兄弟** | 同一双亲的孩子之间互称兄弟 |
| **祖先** | 从根到该结点所经分支上的所有结点 |
| **子孙** | 以某结点为根的子树中的任一结点 |
| **层次** | 根为第一层，根的孩子为第二层，依此类推 |
| **深度（高度）** | 树中结点的最大层次 |
| **堂兄弟** | 双亲在同一层的结点互为堂兄弟 |
| **有序树/无序树** | 如果将树中结点的各子树看成从左至右是有次序的，则为有序树，否则为无序树 |
| **森林** | m (m ≥ 0) 棵互不相交的树的集合 |

### 5.1.3 树的性质 ⚠️（常考）

**性质1：** 树中所有结点数等于所有结点的度数之和 + 1。

**性质2：** 度为 m 的树中，第 i 层上至多有 m^(i-1) 个结点（i ≥ 1）。

**性质3：** 高度为 h 的 m 叉树至多有 (m^h - 1)/(m - 1) 个结点。

**性质4：** 具有 n 个结点的 m 叉树的最小高度为 ⌈logₘ(n(m-1)+1)⌉。

⚠️ **注意区分：** "度为 m 的树"与"m 叉树"的区别：
- 度为 m 的树：至少有一个结点的度为 m，且树中所有结点的度不超过 m
- m 叉树：每个结点最多有 m 个孩子，但可以为空

## 5.2 二叉树的概念

### 5.2.1 二叉树的定义

二叉树是 n (n ≥ 0) 个结点的有限集合：
- 或者为空二叉树 (n = 0)
- 或者由一个根结点和两个互不相交的被称为根的**左子树**和**右子树**组成

⚠️ **二叉树与度为2的有序树的区别：**
- 二叉树可以为空，度为2的树至少有3个结点
- 二叉树的子树有左右之分，不能随意颠倒

### 5.2.2 几种特殊的二叉树

#### （1）满二叉树

一棵高度为 h 且有 2^h - 1 个结点的二叉树称为满二叉树。

特点：
- 每一层上的结点数都是最大结点数
- 叶子结点全部在最底层
- 编号：从根结点开始，自上而下、自左而右依次编号为 1, 2, ..., 2^h-1

#### （2）完全二叉树 ⚠️

高度为 h、有 n 个结点的二叉树，当且仅当其每个结点都与高度为 h 的满二叉树中编号为 1~n 的结点一一对应时，称为完全二叉树。

特点：
- 若 i ≤ ⌊n/2⌋，则结点 i 为分支结点，否则为叶子结点
- 叶子结点只可能在层次最大的两层上出现
- 若有度为 1 的结点，最多只有一个，且该结点只有左孩子而无右孩子
- 按层序编号后，一旦出现某结点为叶子结点或只有左孩子，则后面所有结点都是叶子结点
- **编号为 i 的结点：**
  - 双亲编号为 ⌊i/2⌋（i > 1）
  - 左孩子编号为 2i（若 2i ≤ n）
  - 右孩子编号为 2i+1（若 2i+1 ≤ n）

#### （3）二叉排序树（BST）

左子树上所有结点的关键字均小于根结点的关键字；右子树上所有结点的关键字均大于根结点的关键字；左子树和右子树又各是一棵二叉排序树。

#### （4）平衡二叉树（AVL树）

树上任一结点的左子树和右子树的深度之差不超过 1。

### 5.2.3 二叉树的性质 ⚠️（高频考点）

**性质1：** 非空二叉树上第 i 层至多有 2^(i-1) 个结点（i ≥ 1）。

**性质2：** 高度为 h 的二叉树至多有 2^h - 1 个结点（满二叉树）。

**性质3：** 对任何非空二叉树 T，若叶子结点数为 n₀，度为 2 的结点数为 n₂，则 **n₀ = n₂ + 1**。

⚠️ **推导：**
- 设总结点数为 n，度为 1 的结点数为 n₁
- n = n₀ + n₁ + n₂（按度分类）
- n = n₁ + 2n₂ + 1（分支数 = 结点数 - 1）
- 两式联立得：n₀ = n₂ + 1

**性质4：** 具有 n 个结点的完全二叉树的高度为 ⌊log₂n⌋ + 1 或 ⌈log₂(n+1)⌉。

**性质5：** 对完全二叉树按层序编号（从 1 开始）：
- 结点 i 的双亲为 ⌊i/2⌋（i=1 时为根，无双亲）
- 结点 i 的左孩子为 2i（2i ≤ n 时）
- 结点 i 的右孩子为 2i+1（2i+1 ≤ n 时）
- 结点 i 所在的层次为 ⌊log₂i⌋ + 1

## 5.3 二叉树的存储结构

### 5.3.1 顺序存储结构

用一组地址连续的存储单元依次自上而下、自左而右存储完全二叉树上的结点元素。

```c
#define MaxSize 100
typedef struct {
    ElemType data[MaxSize];  // 从下标1开始存储
    int length;              // 实际存储的元素个数
} SqBiTree;
```

⚠️ **适用范围：** 完全二叉树和满二叉树。一般的二叉树用顺序存储会造成空间浪费。

⚠️ **注意：** 下标从 1 开始存储，利用完全二叉树的性质方便求父子关系。

### 5.3.2 链式存储结构 ⚠️

```c
typedef struct BiTNode {
    ElemType data;
    struct BiTNode *lchild, *rchild;  // 左右孩子指针
} BiTNode, *BiTree;
```

⚠️ **n 个结点的二叉链表中有 n+1 个空链域。**

推导：n 个结点共 2n 个指针域，n-1 个非空指针域（每个非根结点有且仅有一个双亲指向它），所以空链域 = 2n - (n-1) = n+1。

#### 三叉链表（带双亲指针）

```c
typedef struct BiTNode {
    ElemType data;
    struct BiTNode *lchild, *rchild, *parent;  // 左右孩子及双亲指针
} BiTNode, *BiTree;
```

## 5.4 二叉树的遍历 ⚠️（核心考点）

### 5.4.1 先序遍历（根→左→右）

#### 递归实现

```c
void PreOrder(BiTree T) {
    if (T != NULL) {
        visit(T);            // 访问根结点
        PreOrder(T->lchild); // 递归遍历左子树
        PreOrder(T->rchild); // 递归遍历右子树
    }
}
```

#### 非递归实现（利用栈）⭐

```c
void PreOrder2(BiTree T) {
    InitStack(S);
    BiTree p = T;
    while (p || !StackEmpty(S)) {
        if (p) {
            visit(p);           // 访问根结点
            Push(S, p);         // 根指针入栈
            p = p->lchild;      // 遍历左子树
        } else {
            Pop(S, p);          // 根指针出栈
            p = p->rchild;      // 转向右子树
        }
    }
}
```

**记忆口诀：** 入栈时访问，先左后右。

### 5.4.2 中序遍历（左→根→右）

#### 递归实现

```c
void InOrder(BiTree T) {
    if (T != NULL) {
        InOrder(T->lchild);  // 递归遍历左子树
        visit(T);            // 访问根结点
        InOrder(T->rchild);  // 递归遍历右子树
    }
}
```

#### 非递归实现（利用栈）⭐

```c
void InOrder2(BiTree T) {
    InitStack(S);
    BiTree p = T;
    while (p || !StackEmpty(S)) {
        if (p) {
            Push(S, p);         // 根指针入栈
            p = p->lchild;      // 遍历左子树
        } else {
            Pop(S, p);          // 根指针出栈
            visit(p);           // 访问根结点
            p = p->rchild;      // 转向右子树
        }
    }
}
```

**记忆口诀：** 出栈时访问，先左后右。

### 5.4.3 后序遍历（左→右→根）

#### 递归实现

```c
void PostOrder(BiTree T) {
    if (T != NULL) {
        PostOrder(T->lchild);
        PostOrder(T->rchild);
        visit(T);
    }
}
```

#### 非递归实现（利用栈）⭐

```c
void PostOrder2(BiTree T) {
    InitStack(S);
    BiTree p = T, r = NULL;  // r记录最近访问过的结点
    while (p || !StackEmpty(S)) {
        if (p) {
            Push(S, p);
            p = p->lchild;
        } else {
            GetTop(S, p);
            if (p->rchild && p->rchild != r) {  // 右子树存在且未被访问
                p = p->rchild;
                Push(S, p);
                p = p->lchild;
            } else {
                Pop(S, p);
                visit(p);
                r = p;         // 记录最近访问的结点
                p = NULL;      // 重置p，防止重复入栈
            }
        }
    }
}
```

### 5.4.4 层次遍历（利用队列）⭐

```c
void LevelOrder(BiTree T) {
    LinkQueue Q;
    InitQueue(Q);
    BiTree p;
    EnQueue(Q, T);
    while (!QueueEmpty(Q)) {
        DeQueue(Q, p);
        visit(p);
        if (p->lchild != NULL)
            EnQueue(Q, p->lchild);
        if (p->rchild != NULL)
            EnQueue(Q, p->rchild);
    }
}
```

### 5.4.5 遍历序列的性质

⚠️ **关键结论：**
- **只有先序+中序** 或 **后序+中序** 才能**唯一确定**一棵二叉树
- **先序+后序不能唯一确定**一棵二叉树
- 由先序和中序确定二叉树的过程：先序序列的第一个元素是根，在中序序列中找到该根的位置，左边是左子树的中序序列，右边是右子树的中序序列，再递归处理
- 由后序和中序确定二叉树的过程：后序序列的最后一个元素是根，在中序序列中找到该根的位置

### 5.4.6 线索二叉树 ⚠️（重要考点）

#### 基本概念

在含 n 个结点的二叉树中，有 n+1 个空指针域。利用这些空指针域来存放指向其前驱或后继的指针，这样的二叉树称为**线索二叉树**。

#### 存储结构

```c
typedef struct ThreadNode {
    ElemType data;
    struct ThreadNode *lchild, *rchild;
    int ltag, rtag;  // 0表示指向孩子，1表示指向线索
} ThreadNode, *ThreadTree;
```

- `ltag = 0`：lchild 指向左孩子
- `ltag = 1`：lchild 指向前驱
- `rtag = 0`：rchild 指向右孩子
- `rtag = 1`：rchild 指向后继

#### 中序线索化

```c
ThreadNode *pre = NULL;  // 全局变量，指向当前访问结点的前驱

void InThread(ThreadTree T) {
    if (T != NULL) {
        InThread(T->lchild);  // 线索化左子树
        // 处理当前结点
        if (T->lchild == NULL) {
            T->lchild = pre;
            T->ltag = 1;
        }
        if (pre != NULL && pre->rchild == NULL) {
            pre->rchild = T;
            pre->rtag = 1;
        }
        pre = T;
        InThread(T->rchild);  // 线索化右子树
    }
}

void CreateInThread(ThreadTree T) {
    pre = NULL;
    if (T != NULL) {
        InThread(T);
        if (pre->rchild == NULL)  // 处理最后一个结点
            pre->rtag = 1;
    }
}
```

#### 中序线索二叉树中找中序后继

```c
// 在中序线索二叉树中找到p的中序后继
ThreadNode *InNextNode(ThreadNode *p) {
    if (p->rtag == 1)
        return p->rchild;  // 右标志为1，rchild直接就是后继
    else {
        // 右子树中最左下的结点
        ThreadNode *q = p->rchild;
        while (q->ltag == 0)
            q = q->lchild;
        return q;
    }
}
```

⚠️ **遍历线索二叉树的时间复杂度：O(n)，空间复杂度：O(1)**

## 5.5 树与森林

### 5.5.1 树的存储结构

#### （1）双亲表示法

用一组连续空间存储每个结点，同时在每个结点中增设一个伪指针，指示其双亲结点在数组中的位置。

```c
#define MAX_TREE_SIZE 100
typedef struct {
    ElemType data;
    int parent;  // 双亲位置域
} PTNode;

typedef struct {
    PTNode nodes[MAX_TREE_SIZE];
    int n;  // 结点数
} PTree;
```

**优点：** 求双亲结点的时间复杂度 O(1)；求孩子结点需要遍历整个数组。

#### （2）孩子表示法

将每个结点的孩子结点用单链表链接起来。

```c
struct CTNode {
    int child;              // 孩子结点在数组中的位置
    struct CTNode *next;    // 下一个孩子
};

typedef struct {
    ElemType data;
    struct CTNode *firstChild;  // 第一个孩子
} CTBox;

typedef struct {
    CTBox nodes[MAX_TREE_SIZE];
    int n, r;  // 结点数和根的位置
} CTree;
```

**优点：** 便于查找孩子；求双亲需要遍历。

#### （3）孩子兄弟表示法（二叉树表示法）⭐

每个结点包含：数据域、指向第一个孩子的指针、指向下一个兄弟的指针。

```c
typedef struct CSNode {
    ElemType data;
    struct CSNode *firstchild, *nextsibling;  // 第一个孩子和右兄弟指针
} CSNode, *CSTree;
```

⚠️ **重要：** 孩子兄弟表示法可以将一棵树转换为唯一的二叉树（森林 → 二叉树的转换基础）。

### 5.5.2 树、森林与二叉树的转换 ⚠️

#### 树 → 二叉树

1. 在兄弟结点之间加一条连线
2. 对每个结点，只保留它与第一个孩子的连线，删除与其他孩子的连线
3. 以树根为轴心，顺时针旋转 45°

转换结果：**左孩子右兄弟**

#### 森林 → 二叉树

1. 先将森林中的每棵树转换为二叉树
2. 从第二棵二叉树开始，依次把后一棵二叉树的根结点作为前一棵二叉树的根结点的右孩子

#### 二叉树 → 树

1. 若某结点是其双亲的左孩子，则把该结点的右孩子、右孩子的右孩子……都与该结点的双亲用线连起来
2. 删去原二叉树中所有双亲到右孩子的连线
3. 调整成树的形状

#### 二叉树 → 森林

1. 先把每个结点与其右孩子的连线删除，得到若干棵二叉树
2. 将每棵二叉树转换为树

### 5.5.3 树和森林的遍历

#### 树的遍历

| 遍历方式 | 说明 | 与二叉树遍历的对应 |
|---------|------|-------------------|
| **先根遍历** | 先访问根结点，再依次先根遍历各棵子树 | 对应二叉树的先序遍历 |
| **后根遍历** | 先依次后根遍历各棵子树，再访问根结点 | 对应二叉树的中序遍历 |
| **层次遍历** | 用队列逐层访问 | 同二叉树的层次遍历 |

#### 森林的遍历

| 遍历方式 | 说明 | 与二叉树遍历的对应 |
|---------|------|-------------------|
| **先序遍历** | 先访问森林中第一棵树的根结点，先序遍历第一棵树的子树森林，再先序遍历其余树构成的森林 | 对应二叉树的先序遍历 |
| **中序遍历** | 中序遍历第一棵树的根结点的子树森林，再访问第一棵树的根结点，再中序遍历其余树构成的森林 | 对应二叉树的中序遍历 |

⚠️ **重要结论：**
- 树的先根遍历 = 对应二叉树的先序遍历
- 树的后根遍历 = 对应二叉树的中序遍历
- 森林的先序遍历 = 对应二叉树的先序遍历
- 森林的中序遍历 = 对应二叉树的中序遍历

## 5.6 树与二叉树的应用

### 5.6.1 哈夫曼树（最优二叉树）⭐

#### 定义

给定 n 个权值作为 n 个叶子结点，构造一棵二叉树，若该树的**带权路径长度（WPL）** 达到最小，则称这样的二叉树为最优二叉树，也称为哈夫曼树。

- **路径长度：** 从树中一个结点到另一个结点之间的分支数目
- **树的路径长度：** 从树根到每一个结点的路径长度之和
- **结点的带权路径长度：** 从该结点到树根之间的路径长度与结点上权值的乘积
- **树的带权路径长度（WPL）：** 树中所有叶子结点的带权路径长度之和

$$WPL = \sum_{i=1}^{n} w_i l_i$$

#### 哈夫曼树的构造 ⚠️

**算法步骤：**
1. 将 n 个权值分别看成只有根结点的 n 棵二叉树，构成森林 F
2. 在 F 中选取两棵根结点权值最小的树作为左右子树，构造一棵新的二叉树，新二叉树的根结点权值为左右子树根结点权值之和
3. 在 F 中删除这两棵树，同时将新得到的二叉树加入 F
4. 重复步骤 2 和 3，直到 F 中只剩一棵树为止

⚠️ **哈夫曼树的特点：**
- 权值越大的叶子结点离根结点越近
- 每次都是选择两个权值最小的结点合并
- 哈夫曼树中**不存在度为 1 的结点**
- n 个叶子结点的哈夫曼树共有 **2n - 1** 个结点

#### 哈夫曼编码 ⚠️

利用哈夫曼树设计的二进制编码，用于数据压缩。

**前缀编码：** 没有任何一个编码是另一个编码的前缀。

**哈夫曼编码的构造：**
1. 根据字符出现的频率（权值）构造哈夫曼树
2. 从根到每个叶子结点的路径上，左分支记为 0，右分支记为 1
3. 每个叶子结点对应的编码就是从根到该叶子的路径

⚠️ **哈夫曼编码是不等长的前缀编码，频率高的字符编码短，频率低的字符编码长。**

**例：** 设有字符集 {A, B, C, D, E}，权值（出现频率）分别为 {5, 2, 1, 3, 4}。

构造过程：
1. 选 C(1) 和 B(2) 合并，得 CB(3)
2. 选 D(3) 和 CB(3) 合并，得 DCB(6)
3. 选 E(4) 和 A(5) 合并，得 EA(9)
4. 选 DCB(6) 和 EA(9) 合并，得 DCBEA(15)

```
         15
        /  \
      6     9
     / \   / \
    3   3 4   5
   / \      (A)
  1   2    (E)
 (C) (B)
   (D)
```

编码结果：
- A: 11（权值大，编码短）
- E: 10
- D: 00
- B: 011
- C: 010

**WPL = 5×2 + 4×2 + 3×2 + 2×3 + 1×3 = 10 + 8 + 6 + 6 + 3 = 33**

### 例题1：二叉树性质（2009年真题改编）

一棵完全二叉树上有 1001 个结点，则叶子结点的个数是多少？

**解析：**
- 完全二叉树结点数 n = 1001
- 高度 h = ⌊log₂1001⌋ + 1 = 9 + 1 = 10
- 前 9 层有 2⁹ - 1 = 511 个结点
- 第 10 层有 1001 - 511 = 490 个结点
- 第 10 层的 490 个结点都是叶子
- 第 9 层中，有 ⌈490/2⌉ = 245 个是双亲（有孩子），2⁸ - 245 = 11 个是叶子
- 叶子结点总数 = 490 + 11 = 501

**另一种方法：**
- n₁ = 0 或 1（完全二叉树中度为1的结点最多1个）
- n = 1001 为奇数 → n₁ = 1
- n = n₀ + n₁ + n₂ = n₀ + 1 + n₂ = 1001
- n₀ = n₂ + 1
- 代入：n₂ + 1 + 1 + n₂ = 1001，2n₂ = 999
- ⚠️ n₂ 不是整数，说明 n₁ = 1 不对

**重新计算：**
- n = 1001 为奇数 → 完全二叉树中若 n 为奇数则 n₁ = 0
- n₀ + n₂ = 1001
- n₀ = n₂ + 1
- 2n₂ + 1 = 1001，n₂ = 500，n₀ = 501

**答案：501**

### 例题4：哈夫曼编码
