# 第5章 I/O管理

## 5.1 I/O管理概述

### 5.1.1 I/O设备的分类

| 分类标准 | 类型 |
|---------|------|
| 按使用特性 | 存储设备、人机交互设备、网络通信设备 |
| 按传输速率 | 低速（键盘、鼠标）、中速（打印机）、高速（磁盘） |
| 按信息交换单位 | 块设备（磁盘）、字符设备（键盘） |
| 按共享属性 | 独占设备、共享设备、虚拟设备 |

### 5.1.2 I/O管理目标和功能

| 目标 | 说明 |
|------|------|
| **设备独立性** | 用户程序与具体物理设备无关 |
| **提高设备利用率** | 使设备和 CPU 并行工作 |
| **方便用户使用** | 提供统一的设备访问接口 |

### 5.1.3 I/O软件层次结构 ⭐

```
用户层I/O软件（库函数：printf, scanf等）
    ↓
设备独立性软件（统一接口、缓冲、错误处理、设备分配与回收）
    ↓
设备驱动程序（将抽象I/O请求转换为具体设备操作）
    ↓
中断处理程序（处理设备完成I/O后的中断）
    ↓
硬件（设备控制器和设备本身）
```

⚠️ **设备独立性软件（设备无关软件）的功能：**
- 提供统一接口
- 缓冲管理
- 差错控制
- 设备分配与回收
- 逻辑设备名到物理设备名的映射

### 5.1.4 应用程序I/O接口

| 接口类型 | 说明 |
|---------|------|
| **字符设备接口** | get/put 操作，一次一个字符 |
| **块设备接口** | read/write 操作，以块为单位 |
| **网络设备接口** | socket 接口 |

## 5.2 设备独立性软件 ⭐

### 5.2.1 设备独立性（设备无关性）

用户程序使用**逻辑设备名**请求 I/O，操作系统通过逻辑设备名到物理设备名的映射表将逻辑设备名转换为物理设备名。

**优点：**
- 增加设备分配的灵活性
- 易于实现 I/O 重定向

### 5.2.2 缓冲区管理 ⭐⭐

#### 缓冲的作用

1. 缓和 CPU 与 I/O 设备之间速度不匹配的矛盾
2. 减少对 CPU 的中断频率
3. 提高 CPU 与 I/O 设备之间的并行性

#### 缓冲的类型

**（1）单缓冲**

系统在主存中为设备分配一个缓冲区。

**处理时间：** max(C, T) + M

其中 C 为 CPU 处理时间，T 为输入时间，M 为传送时间。

**（2）双缓冲（缓冲对换）⭐**

系统分配两个缓冲区，CPU 和设备可以并行工作。

**处理时间：** max(C+M, T)

**（3）循环缓冲**

多个缓冲区首尾相连形成环形。

**（4）缓冲池 ⭐**

将多个缓冲区组成一个公共缓冲池，分为三类：
- **空缓冲队列 emq**
- **输入队列 inq**（装满输入数据的缓冲区）
- **输出队列 outq**（装满输出数据的缓冲区）

四种工作缓冲区：
- 收容输入 hin
- 提取输入 sin
- 收容输出 hout
- 提取输出 sout

### 5.2.3 设备分配与回收

#### 设备分配的数据结构

| 数据结构 | 说明 |
|---------|------|
| **设备控制表（DCT）** | 每个设备一张，记录设备状态、类型、等待队列等 |
| **控制器控制表（COCT）** | 每个控制器一张 |
| **通道控制表（CHCT）** | 每个通道一张 |
| **系统设备表（SDT）** | 记录系统中所有设备的信息 |

#### 设备分配的策略

| 考虑因素 | 说明 |
|---------|------|
| **设备固有属性** | 独占设备、共享设备、虚拟设备 |
| **设备分配算法** | FCFS、优先级高者优先 |
| **设备分配安全性** | 避免死锁 |

#### SPOOLing 技术（假脱机技术）⭐⭐⭐

利用一道程序模拟脱机输入/输出，将独占设备改造为共享设备。

**SPOOLing 系统的组成：**

| 组成部分 | 功能 |
|---------|------|
| **输入井/输出井** | 模拟脱机输入/输出的磁盘区域 |
| **输入进程/输出进程** | 模拟脱机输入/输出的外围控制机 |
| **输入缓冲区/输出缓冲区** | 缓和内外设速度差异 |

⚠️ **SPOOLing 的核心应用：共享打印机**

- 多个进程的打印请求被 SPOOLing 系统放入输出井排队
- 打印机被虚拟为多台"虚拟打印机"
- 用户感觉独占打印机，实际上是共享的

⚠️ **SPOOLing = 虚拟设备技术 = 将独占设备虚拟为共享设备**

### 5.2.4 磁盘高速缓存（Disk Cache）

在内存中开辟一块区域，暂存磁盘中的数据。

| 对比 | 缓冲区（Buffer） | 高速缓存（Cache） |
|------|----------------|-----------------|
| 目的 | 解决速度不匹配 | 减少磁盘访问次数 |
| 数据 | 可能被立即丢弃 | 希望被再次使用 |

## 5.3 磁盘和固态硬盘

### 5.3.1 磁盘管理

- **磁盘格式化：** 低级格式化（物理格式化）→ 分区 → 逻辑格式化（建立文件系统）
- **引导块：** 包含引导程序（bootstrap）
- **坏块管理：** 扇区备用（扇区预留）、扇区重映射

### 5.3.2 固态硬盘（SSD）

基于闪存技术，没有机械部件。

| 特性 | 说明 |
|------|------|
| 读写速度 | 随机读取快，顺序写入受限 |
| 写入放大 | 实际写入量 > 用户写入量 |
| 磨损均衡 | 保证各闪存块均匀磨损 |

⚠️ **SSD 的读写单位是页（Page），擦除单位是块（Block）。**

⚠️ **写入放大问题：** SSD 写入前需要先擦除整个块，即使只修改一页。因此需要垃圾回收和磨损均衡策略。

## 5.4 本章例题

### 例题1：缓冲区计算

在双缓冲系统中，CPU 处理一个缓冲区数据的时间为 40μs，从缓冲区传送数据到用户区的时间为 5μs，外设向一个缓冲区输入数据的时间为 100μs。求处理一批数据的平均时间。

**解析：**

双缓冲下，CPU 处理和外设输入可以并行。

CPU 处理一个缓冲区的时间 = 40 + 5 = 45μs
外设输入一个缓冲区的时间 = 100μs

max(45, 100) = 100μs

**平均每 100μs 处理一个缓冲区的数据。**

### 例题2：SPOOLing 技术

SPOOLing 技术如何将独占设备变为共享设备？以打印机为例说明。

**答案：**

1. 当用户进程请求打印时，SPOOLing 系统并不真正将打印机分配给它
2. SPOOLing 系统在磁盘的输出井中为该进程申请一个空闲盘块，将打印数据送入输出井
3. SPOOLing 系统为用户进程申请一张空白的用户请求打印表，填写打印说明信息，挂到打印请求队列上
4. SPOOLing 打印进程依次从打印请求队列中取出请求，将输出井中的数据送到打印机打印
5. 用户进程感觉独占了打印机，但实际上是 SPOOLing 进程在管理，多个进程的打印请求在输出井中排队

### 例题3：设备独立性

什么是设备独立性？如何实现？

**答案：**

设备独立性是指用户程序中使用逻辑设备名来请求使用某类设备，而系统在实际执行时使用物理设备名。

实现方法：
- 设置逻辑设备表（LUT），建立逻辑设备名到物理设备名的映射
- 用户使用逻辑设备名请求 I/O
- 系统查找 LUT，分配物理设备
- 通过逻辑设备名实现 I/O 重定向
