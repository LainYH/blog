# 第4章 文件管理

## 4.1 文件系统基础

### 4.1.1 文件的概念

**文件：** 是以计算机硬盘为载体的存储在计算机上的信息集合。

### 4.1.2 文件的逻辑结构 ⭐

#### 按是否有结构分类

| 类型 | 说明 | 例子 |
|------|------|------|
| **有结构文件（记录式文件）** | 由一组相似记录组成 | 数据库文件 |
| **无结构文件（流式文件）** | 无结构，以字节为单位 | 源程序文件、可执行文件 |

#### 有结构文件的分类

| 类型 | 说明 |
|------|------|
| **顺序文件** | 记录按关键字排列，顺序存储和访问 |
| **索引文件** | 建立索引表，支持随机访问 |
| **索引顺序文件** | 将记录分组，组间索引，组内顺序 |

### 4.1.3 文件的物理结构（文件分配方式）⭐⭐

#### （1）连续分配 ⭐

文件占用磁盘上一组连续的块。

| 优点 | 缺点 |
|------|------|
| 支持顺序访问和随机访问 | 文件长度不便于动态增长 |
| 顺序访问速度快 | 产生外部碎片 |
| 寻道时间短 | 需要事先确定文件大小 |

**FCB 中只需记录起始块号和长度。**

#### （2）链接分配 ⭐

**① 隐式链接**

每个磁盘块包含指向下一个块的指针。

| 优点 | 缺点 |
|------|------|
| 消除外部碎片 | 只支持顺序访问，不支持随机访问 |
| 文件可动态增长 | 可靠性差（指针损坏则文件丢失） |

**② 显式链接（FAT）**

将所有磁盘块的链接指针集中存放在一张**文件分配表（FAT）** 中。

| 优点 | 缺点 |
|------|------|
| 支持随机访问（查 FAT） | FAT 需要占用内存空间 |
| 比隐式链接可靠 | |

⚠️ **FAT 在系统启动时读入内存。**

#### （3）索引分配 ⭐⭐

每个文件有一个**索引块**（索引表），记录文件的所有磁盘块号。

| 优点 | 缺点 |
|------|------|
| 支持随机访问 | 索引块占用额外空间 |
| 文件可动态增长 | 大文件需要多级索引 |
| 无外部碎片 | |

**大文件的索引方案：**

| 方案 | 说明 |
|------|------|
| **链接方案** | 将多个索引块链接起来 |
| **多层索引** | 索引块指向索引块（类似多级页表） |
| **混合索引（UNIX 方案）** ⭐ | 直接块 + 一级间接 + 二级间接 + 三级间接 |

⚠️ **UNIX 混合索引方案：**
- 假设一个块 4KB，块号占 4B，一个索引块可存 1000 个块号
- 直接块：10 个（前 10 个块直接索引）
- 一级间接：1 个索引块 → 1000 个块
- 二级间接：1 个索引块 → 1000 个索引块 → 1000×1000 个块
- 三级间接：1 个索引块 → 1000×1000×1000 个块
- 最大文件 = (10 + 1000 + 10⁶ + 10⁹) × 4KB

#### 三种分配方式的对比

| 对比 | 连续分配 | 链接分配 | 索引分配 |
|------|---------|---------|---------|
| 顺序访问 | 快 | 快 | 快 |
| 随机访问 | 支持 | 不支持（FAT支持） | 支持 |
| 碎片 | 外部碎片 | 无 | 无（索引块有内部碎片） |
| 文件增长 | 困难 | 容易 | 容易 |
| 可靠性 | 中 | 差（隐式） | 好 |

### 4.1.4 目录结构 ⭐⭐

#### FCB（文件控制块）

FCB 是用于描述和控制文件的数据结构，包含：
- 基本信息（文件名、物理位置、逻辑结构、物理结构等）
- 存取控制信息
- 使用信息

⚠️ **文件目录 = FCB 的有序集合（文件目录本身也是文件）。**

#### 目录结构类型

| 类型 | 说明 |
|------|------|
| **单级目录** | 整个系统一个目录表；不支持重名，不便于文件共享 |
| **两级目录** | 主文件目录（MFD）+ 用户文件目录（UFD）；解决了重名问题 |
| **树形目录** ⭐ | 多级层次结构；现代OS的标准 |
| **无环图目录** | 在树形基础上增加共享（硬链接/符号链接）；可能产生环 |

### 4.1.5 文件共享 ⭐

| 方式 | 说明 | 特点 |
|------|------|------|
| **基于索引结点的共享（硬链接）** | 多个目录项指向同一个索引结点（i-node） | 删除一个链接不影响其他链接；引用计数 |
| **基于符号链的共享（软链接/符号链接）** | 创建一个包含目标文件路径的链接文件 | 链接文件有自己的 inode；删除原文件后链接失效 |

⚠️ **硬链接不能跨文件系统，软链接可以。**

### 4.1.6 文件保护

| 保护方式 | 说明 |
|---------|------|
| **访问控制** | 按用户身份设置访问权限（读/写/执行） |
| **口令保护** | 文件设置口令，访问时验证 |
| **加密保护** | 文件内容加密存储，读取时解密 |

## 4.2 文件系统实现 ⭐

### 4.2.1 文件系统层次结构

```
用户接口（文件操作的系统调用）
    ↓
文件目录系统（目录管理）
    ↓
存取控制模块（权限检查）
    ↓
逻辑文件系统与文件信息缓冲区（逻辑→物理映射）
    ↓
物理文件系统（管理磁盘块的分配与回收）
    ↓
辅助分配模块（空闲空间管理）
    ↓
设备管理模块（设备驱动与缓冲管理）
```

### 4.2.2 目录实现

| 方式 | 说明 |
|------|------|
| **线性列表** | 用线性表存储目录项，查找需要遍历 |
| **哈希表** | 用哈希函数定位目录项，查找快 |

### 4.2.3 文件系统在外存的组织

#### 磁盘分区与文件系统

**引导块：** 存放引导程序。
**超级块：** 存放文件系统的元数据（块大小、块数目、空闲块信息等）。
**i-node 区：** 存放所有 i-node。
**数据区：** 存放文件数据。

#### 空闲空间管理 ⭐

| 方法 | 说明 | 特点 |
|------|------|------|
| **空闲表法** | 用表记录所有空闲区 | 适合连续分配 |
| **空闲链表法** | 将空闲块链接起来 | 简单 |
| **位示图法** ⭐ | 用二进制位表示每个块的状态（0空闲/1占用） | 直观，常用于分页管理 |
| **成组链接法** | 将空闲块分组，每组第一个块记录下一组的信息 | UNIX 使用 |

⚠️ **位示图中第 i 个字、第 j 位对应的块号 = i × 字长 + j。**

## 4.3 磁盘组织与管理 ⭐

### 4.3.1 磁盘的结构

```
磁盘 = 多个盘片（Platter）
盘片 = 两个盘面（Surface）
盘面 = 多个磁道（Track）
磁道 = 多个扇区（Sector）——最小读写单位
```

⚠️ **磁盘地址 = 柱面号（磁道号）+ 盘面号 + 扇区号**

⚠️ **柱面（Cylinder）：** 所有盘面上同一半径的磁道构成一个柱面。

### 4.3.2 磁盘调度算法 ⭐⭐

磁盘访问时间 = **寻道时间** + **旋转延迟** + **传输时间**

#### （1）先来先服务（FCFS）

按请求到达的先后顺序服务。

**优点：** 公平。
**缺点：** 寻道时间长。

#### （2）最短寻道时间优先（SSTF）⭐

选择离当前磁头位置最近的磁道。

**优点：** 寻道时间短。
**缺点：** 可能导致**饥饿**（远端磁道长期得不到服务）。

#### （3）扫描算法（SCAN/电梯算法）⭐⭐

磁头沿一个方向移动，依次服务途经的请求，到达磁盘末端后反向。

**优点：** 兼顾效率和公平。
**缺点：** 两端磁道的请求等待时间较长。

#### （4）循环扫描算法（C-SCAN）⭐

磁头只沿一个方向扫描，到达末端后直接返回起始端（返回途中不服务）。

**优点：** 各磁道的响应时间更均匀。

#### （5）LOOK 和 C-LOOK ⭐

- **LOOK：** SCAN 的改进，磁头不必到达最远端，只需到达最远的请求位置就反向
- **C-LOOK：** C-SCAN 的改进，同上

⚠️ **实际系统中常用 LOOK/C-LOOK 算法。**

#### 磁盘调度算法比较

| 算法 | 寻道时间 | 饥饿 | 公平性 |
|------|---------|------|--------|
| FCFS | 长 | 无 | 好 |
| SSTF | 短 | **有** | 差 |
| SCAN | 中 | 无 | 较好 |
| C-SCAN | 中 | 无 | 好 |

### 4.3.3 减少磁盘延迟的方法

**交替编号：** 逻辑上相邻的扇区在物理上不相邻（间隔编号），以弥补旋转延迟。

**错位命名：** 不同盘面的同一柱面上，扇区号错开一个位置。

### 4.3.4 磁盘的管理

- **磁盘初始化（低级格式化）：** 将磁盘分成扇区
- **分区（逻辑格式化）：** 创建文件系统
- **引导块：** 存放引导程序
- **坏块管理：** 标记坏块，避免使用

## 4.4 本章例题

### 例题1：混合索引计算

设 UNIX 文件系统中，每个盘块 1KB，每个盘块号占 4B。一个 inode 有 12 个直接块、1 个一级间接、1 个二级间接、1 个三级间接。

求：（1）最大文件大小；（2）访问文件第 15000B 处的数据需要几次磁盘读取？

**解析：**

每个盘块可存 1KB/4B = 256 个块号。

（1）直接：12 × 1KB = 12KB
一级间接：256 × 1KB = 256KB
二级间接：256 × 256 × 1KB = 65536KB = 64MB
三级间接：256 × 256 × 256 × 1KB = 16777216KB = **16GB**
最大文件 ≈ 16GB + 64MB + 256KB + 12KB

（2）15000B / 1024 = 14.65，即第 15 个块（块号 14），在直接块范围内（0~11 号直接块）。
第 12~267 号块在一级间接中，14 ≥ 12，需要读：
- 1 次一级间接索引块
- 1 次数据块
共 **2 次磁盘读取**。

### 例题2：磁盘调度

磁头当前位置 53，磁道请求序列：98, 183, 37, 122, 14, 124, 65, 67。

分别用 FCFS、SSTF、SCAN（向磁道号增大方向）计算总寻道长度。

**FCFS：**
53→98→183→37→122→14→124→65→67
距离：45+85+146+85+108+110+59+2 = **640**

**SSTF：**
53→37→14→65→67→98→122→124→183
距离：16+23+51+2+31+24+2+59 = **208**

**SCAN（向增大方向，假设磁盘200磁道）：**
53→65→67→98→122→124→183→200→37→14
距离：12+2+31+24+2+59+17+163+23 = **333**

（注：SCAN 需要到达最远端后反向。若用 LOOK 则：53→65→67→98→122→124→183→37→14，距离 12+2+31+24+2+59+146+23 = 299）

### 例题3：位示图

某磁盘有 2048 个块，用位示图管理空闲块，位示图中字长 32 位。

求：（1）位示图需要多少个字？（2）第 100 号块对应位示图的哪个字的哪一位？

**解析：**
（1）字数 = 2048 / 32 = **64 个字**
（2）100 / 32 = 3 余 4，即第 **3 号字的第 4 位**（从 0 开始编号）
