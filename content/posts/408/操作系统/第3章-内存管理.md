# 第3章 内存管理

## 3.1 内存管理基础

### 3.1.1 内存管理的功能

| 功能 | 说明 |
|------|------|
| **内存空间的分配与回收** | 记录内存使用情况，按策略分配和回收 |
| **地址转换** | 将逻辑地址转换为物理地址（重定位） |
| **内存空间的扩充** | 利用虚拟存储技术，从逻辑上扩充内存 |
| **内存共享** | 允许多个进程共享同一内存区域 |
| **存储保护** | 保护各进程互不干扰 |

### 3.1.2 程序的链接与装入 ⭐

#### 程序的装入（地址重定位）

| 方式 | 说明 | 时机 |
|------|------|------|
| **绝对装入** | 编译时产生绝对地址 | 编译时 |
| **可重定位装入（静态重定位）** | 装入时完成地址转换 | 装入时 |
| **动态运行时装入（动态重定位）** | 运行时才进行地址转换（需重定位寄存器支持） | 运行时 |

⚠️ **动态重定位的优点：** 内存中的程序可以移动，便于内存紧凑。

#### 程序的链接

| 方式 | 说明 |
|------|------|
| **静态链接** | 装入前链接成完整模块 |
| **装入时动态链接** | 边装入边链接 |
| **运行时动态链接** | 运行时才链接（如 DLL） |

### 3.1.3 逻辑地址与物理地址 ⭐

- **逻辑地址（相对地址/虚拟地址）：** 程序员视角的地址，从 0 开始编址
- **物理地址（绝对地址/实地址）：** 内存中实际的地址

⚠️ **CPU 执行指令时使用逻辑地址，经过地址转换得到物理地址后访问内存。**

### 3.1.4 内存保护

| 方法 | 说明 |
|------|------|
| **上下限寄存器** | 每个进程设置上下限寄存器，检查地址是否在范围内 |
| **重定位寄存器 + 界地址寄存器** ⭐ | 物理地址 = 逻辑地址 + 重定位寄存器值；检查逻辑地址 < 界地址 |

## 3.2 连续分配管理方式 ⭐

### 3.2.1 单一连续分配

内存分为系统区和用户区，用户区只有一个用户程序。

**特点：** 简单、无外部碎片、有内部碎片、只适合单用户单任务。

### 3.2.2 固定分区分配

将内存划分为若干固定大小的分区，每个分区装入一个作业。

**特点：** 可多道程序、有内部碎片、无外部碎片。

### 3.2.3 动态分区分配 ⭐

根据进程需要动态分配内存分区。

**数据结构：**
- 空闲分区表
- 空闲分区链

#### 分配算法 ⭐

| 算法 | 说明 | 特点 |
|------|------|------|
| **首次适应（FF）** | 从头找第一个足够大的空闲分区 | 简单，低地址碎片多 |
| **最佳适应（BF）** | 找最小的足够大的空闲分区 | 产生最多的外部碎片 |
| **最坏适应（WF）** | 找最大的空闲分区 | 大分区很快用完 |
| **邻近适应（NF）** | 从上次分配位置继续找 | 高地址区域容易保留大分区 |
| **下次适应（NF/循环首次适应）** | 从上次找到的位置继续 | 分配均匀 |

⚠️ **首次适应通常实际效果最好。**

#### 内存回收

回收分区时需要考虑与相邻空闲分区的合并。

#### 碎片问题 ⭐

| 类型 | 定义 | 产生于 |
|------|------|--------|
| **内部碎片** | 分配给进程的内存块中未被利用的部分 | 固定分区 |
| **外部碎片** | 内存中所有空闲分区总和足够，但不连续 | 动态分区 |

⚠️ **解决外部碎片的方法：** **紧凑（拼接/内存紧缩）**——将已分配的内存块移到一起。需要动态重定位的支持。

## 3.3 非连续分配管理方式 ⭐⭐

### 3.3.1 基本分页存储管理 ⭐⭐

#### 基本概念

| 概念 | 说明 |
|------|------|
| **页/页面** | 将进程的逻辑地址空间划分为等大小的块 |
| **页框/页帧/物理块/内存块** | 将物理内存划分为与页面大小相同块 |
| **页表** | 实现页号到物理块号的映射 |

⚠️ **页面大小为 2 的幂次（如 4KB）。**

#### 地址结构 ⭐

```
逻辑地址 = | 页号(P) | 页内偏移(W) |
```

$$\text{页号} = \text{逻辑地址} / \text{页面大小}$$
$$\text{页内偏移} = \text{逻辑地址} \mod \text{页面大小}$$

⚠️ **若页面大小为 L（2^n），则逻辑地址的低 n 位为页内偏移，高位为页号。**

#### 页表

每个进程有一个页表，存放在内存中。

页表项包含：**物理块号** + 其他控制位（有效位、访问位、修改位等）。

⚠️ **页表基址寄存器（PTBR）** 存放页表在内存中的起始地址。

#### 基本地址变换机构 ⭐⭐

```
逻辑地址 → 页号 → 查页表得物理块号 → 物理地址 = 物理块号 × 页面大小 + 页内偏移
```

⚠️ **访存次数：** 基本分页存储管理需要 **2 次访存**（1 次查页表 + 1 次访问目标数据）。

#### 具有快表（TLB）的地址变换机构

$$\text{有效访存时间} = h \times (t_{TLB} + t_{mem}) + (1-h) \times (t_{TLB} + 2t_{mem})$$

其中 h 为 TLB 命中率，t_TLB 为 TLB 访问时间，t_mem 为内存访问时间。

⚠️ **TLB 命中时只需 1 次访存；TLB 未命中需要 2 次访存。**

#### 两级页表和多级页表 ⭐

当页表太大时，将页表再分页。

**两级页表的逻辑地址：**
```
| 外层页号 | 内层页号 | 页内偏移 |
```

⚠️ **两级页表需要 3 次访存**（查外层页表 + 查内层页表 + 访问数据）。

#### 页大小的影响

| 页大 | 页小 |
|------|------|
| 内部碎片多 | 内部碎片少 |
| 页表小 | 页表大 |
| TLB 利用率高 | TLB 利用率低 |

### 3.3.2 基本分段存储管理 ⭐⭐

#### 基本概念

按照程序的逻辑结构将地址空间划分为若干段，每段有自己的段名和长度。

#### 地址结构

```
逻辑地址 = | 段号(S) | 段内偏移(W) |
```

#### 段表

每个进程有一个段表，段表项包含：**段长** + **段基址（物理起始地址）**。

#### 地址变换机构

```
逻辑地址 → 段号 → 查段表得段基址 → 物理地址 = 段基址 + 段内偏移
```

⚠️ **检查段内偏移 < 段长（越界检查）。**

⚠️ **访存次数：** 2 次（1 次查段表 + 1 次访问目标数据）。

### 3.3.3 段页式管理 ⭐⭐

将程序分段，每段再分页。

#### 地址结构

```
逻辑地址 = | 段号 | 页号 | 页内偏移 |
```

#### 地址变换

1. 由段号查段表 → 得到该段的页表起始地址
2. 由页号查页表 → 得到物理块号
3. 物理地址 = 物理块号 × 页面大小 + 页内偏移

⚠️ **段页式管理需要 3 次访存**（查段表 + 查页表 + 访问数据）。**TLB 尤为重要。**

### 3.3.4 分页 vs 分段 ⭐

| 对比维度 | 分页 | 分段 |
|---------|------|------|
| 目的 | 提高内存利用率 | 满足用户需要（逻辑划分） |
| 划分依据 | 物理划分，大小固定 | 逻辑划分，大小不固定 |
| 地址空间 | 一维 | 二维（段号+段内偏移） |
| 内部碎片 | 有 | 无 |
| 外部碎片 | 无 | 有 |
| 信息共享和保护 | 不方便（以页为单位） | 方便（以段为单位） |

## 3.4 虚拟内存管理 ⭐⭐⭐

### 3.4.1 虚拟存储器的基本概念

虚拟存储器是将内存和外存相结合，为用户提供一个比实际内存大得多的虚拟内存空间。

#### 虚拟存储器的特征 ⭐

| 特征 | 说明 |
|------|------|
| **多次性** | 作业不必一次性全部装入内存，可分多次调入 |
| **对换性** | 作业运行时不需要的部分可以换出外存 |
| **虚拟性** | 从逻辑上扩充内存容量 |

#### 虚拟存储器的实现基础 ⭐

1. **一定容量的内存和外存**
2. **页表（或段表）中的有效位/存在位**：标记是否在内存中
3. **缺页/缺段中断机构**
4. **地址变换机构**

### 3.4.2 请求分页存储管理 ⭐⭐

#### 页表项的扩充

| 字段 | 说明 |
|------|------|
| **有效位（存在位）P** | 1 表示在内存中，0 表示不在 |
| **物理块号** | 有效位 = 1 时给出物理块号 |
| **外存地址** | 有效位 = 0 时给出在外存的位置 |
| **访问位 A** | 用于页面置换算法 |
| **修改位 M（脏位）** | 页面是否被修改过（修改过的页面换出时需写回外存） |

#### 缺页中断 ⭐⭐

当 CPU 访问的页面不在内存中时，产生**缺页中断**：
1. 保护 CPU 现场
2. 分析中断原因（缺页）
3. 从外存找到缺页
4. 若内存有空闲页框 → 调入页面
5. 若内存已满 → 执行页面置换算法
6. 修改页表
7. 恢复现场，重新执行被中断的指令

⚠️ **缺页中断是内中断（异常/陷入），每条指令最多产生一次缺页中断。**

⚠️ **缺页中断处理后，需要重新执行被中断的指令（不是下一条）。**

#### 请求分页的地址变换过程 ⭐

1. 查 TLB → 命中则直接得到物理块号
2. TLB 未命中 → 查页表
3. 页表项有效位 = 1 → 得到物理块号
4. 有效位 = 0 → **缺页中断**
5. 通过物理块号 + 页内偏移 → 物理地址

### 3.4.3 页面置换算法 ⭐⭐⭐

当内存已满且需要装入新页面时，需要选择一个页面换出。

#### （1）最佳置换算法（OPT）⭐

选择**未来最长时间内不再被访问**的页面换出。

⚠️ **无法实现**（需要预知未来），但作为理论最优标准。

#### （2）先进先出置换算法（FIFO）⭐

选择最早进入内存的页面换出。

⚠️ **FIFO 可能出现 Belady 异常：** 分配的物理块数增加，缺页次数反而增加。

#### （3）最近最久未使用算法（LRU）⭐⭐

选择**最近最长时间未被访问**的页面换出。

**实现方法：** 每个页面设置一个访问时间字段，每次访问时更新。

⚠️ **LRU 不会出现 Belady 异常。** 硬件开销较大。

#### （4）时钟置换算法（CLOCK/NRU）⭐

将页面组织成循环链表，每个页面有一个访问位（A）。

**简单 CLOCK：**
1. 指针扫描页面，若 A = 0，替换该页面；若 A = 1，置 A = 0，继续扫描

**改进型 CLOCK（考虑修改位 M）⭐：**

按优先级选择：
1. (A=0, M=0)：最近未访问且未修改 → 最佳淘汰页
2. (A=0, M=1)：最近未访问但已修改
3. (A=1, M=0)：最近访问但未修改
4. (A=1, M=1)：最近访问且已修改 → 最后淘汰

⚠️ **改进型 CLOCK 算法最多需要 4 轮扫描。**

#### 页面置换算法比较

| 算法 | 是否最优 | 是否Belady异常 | 硬件开销 |
|------|---------|--------------|---------|
| OPT | 是 | 不会 | 不可实现 |
| FIFO | 否 | **会** | 小 |
| LRU | 近似最优 | 不会 | 大 |
| CLOCK | 近似 | 不会 | 中 |

### 3.4.4 页面分配策略

| 策略 | 说明 |
|------|------|
| **固定分配局部置换** | 每个进程固定分配若干页框，缺页时在自己的页框中置换 |
| **可变分配全局置换** | 系统维护一个空闲页框池，缺页时分配空闲页框或从全局选择置换 |
| **可变分配局部置换** | 进程的页框数可调整，但只能在自己的页框中置换 |

### 3.4.5 抖动（颠簸）⭐

**抖动：** 刚刚换出的页面马上又要换入，频繁进行页面置换，导致 CPU 利用率急剧下降。

**主要原因：** 进程分配的物理块不足（多道程序度过高）。

**解决方法：**
- 采用工作集模型
- 控制多道程序度
- 挂起若干进程

### 3.4.6 工作集 ⭐

**工作集：** 某进程在某段时间间隔内访问的页面集合。

⚠️ **给进程分配的物理块数应 ≥ 其工作集的大小。**

## 3.5 本章例题

### 例题1：地址转换

某系统页面大小 4KB，逻辑地址为 32 位，页表中逻辑页号 5 对应物理块号 100。求逻辑地址 0x00005A3B 的物理地址。

**解析：**
- 页号 = 0x00005A3B / 0x1000 = 5
- 页内偏移 = 0x00005A3B % 0x1000 = 0xA3B
- 物理块号 = 100 = 0x64
- 物理地址 = 0x64 × 0x1000 + 0xA3B = **0x00064A3B**

某系统有 3 个物理块，页面访问序列：7, 0, 1, 2, 0, 3, 0, 4, 2, 3, 0, 3, 2, 1, 2, 0, 1, 7, 0, 1

分别用 FIFO 和 LRU 计算缺页次数。

**FIFO 结果：**

|------|-----|-----|-----|-------|
| 7 | 7 | - | - | √ |
| 0 | 7 | 0 | - | √ |
| 1 | 7 | 0 | 1 | √ |
| 2 | 2 | 0 | 1 | √ |
| 0 | 2 | 0 | 1 | |
| 3 | 2 | 3 | 1 | √ |
| 0 | 2 | 3 | 0 | √ |
| 4 | 4 | 3 | 0 | √ |
| 2 | 4 | 2 | 0 | √ |
| 3 | 4 | 2 | 3 | √ |
| 0 | 0 | 2 | 3 | √ |
| 3 | 0 | 2 | 3 | |
| 2 | 0 | 2 | 3 | |
| 1 | 0 | 1 | 3 | √ |
| 2 | 0 | 1 | 2 | √ |

| 1 | 0 | 1 | 2 | |
| 7 | 7 | 1 | 2 | √ |

**FIFO 缺页次数：15**

**LRU 结果：**（略，详细计算后）**缺页次数：12**

### 例题3：有效访问时间

某系统页面大小 4KB，TLB 命中率为 98%，TLB 访问时间 10ns，内存访问时间 100ns。求有效访问时间。

**解析：**

EAT = 0.98 × (10 + 100) + 0.02 × (10 + 100 + 100) = 0.98 × 110 + 0.02 × 210 = 107.8 + 4.2 = **112ns**
