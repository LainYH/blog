# 第1章 操作系统概述

## 1.1 操作系统的概念、特征、功能和提供的服务

### 1.1.1 操作系统的概念

操作系统（Operating System, OS）是指控制和管理整个计算机系统的硬件和软件资源，合理地组织、调度计算机的工作与资源的分配，进而为用户和其他软件提供方便接口与环境的程序集合。

⚠️ **操作系统是系统软件，是最基本的系统软件。**

### 1.1.2 操作系统的特征 ⭐

| 特征 | 说明 |
|------|------|
| **并发（Concurrency）** | 两个或多个事件在同一时间间隔内发生（宏观上同时，微观上交替） |
| **共享（Sharing）** | 系统中的资源可供多个并发执行的进程共同使用 |
| **虚拟（Virtual）** | 把一个物理实体变为若干逻辑上的对应物 |
| **异步（Asynchronism）** | 进程的执行以不可预知的速度向前推进 |

⚠️ **并发和共享是操作系统最基本的两个特征，二者互为存在条件。**

⚠️ **并发 vs 并行：**
- **并发：** 同一时间间隔内发生（宏观同时，微观交替）
- **并行：** 同一时刻同时发生（需要多处理器/多核硬件支持）

⚠️ **虚拟的典型例子：**
- **时分复用：** 虚拟处理器（多道程序设计）、虚拟设备（SPOOLing）
- **空分复用：** 虚拟存储器、虚拟磁盘

### 1.1.3 操作系统的功能 ⭐

| 功能 | 说明 |
|------|------|
| **处理机管理** | 进程控制、进程同步、进程通信、调度 |
| **存储器管理** | 内存分配与回收、地址映射、内存保护、内存扩充（虚拟内存） |
| **文件管理** | 文件存储空间管理、目录管理、文件读写管理和保护 |
| **设备管理** | 缓冲管理、设备分配、设备处理（设备驱动） |
| **提供用户接口** | 命令接口（联机/脱机）、程序接口（系统调用）、图形接口 |

### 1.1.4 操作系统提供的服务

- 程序执行
- I/O 操作
- 文件系统操作
- 通信
- 错误检测
- 资源分配
- 统计
- 保护和安全

## 1.2 操作系统的发展和分类

### 1.2.1 发展历程

| 阶段 | 特点 |
|------|------|
| **手工操作阶段** | 人工操作，用户独占全机，CPU 利用率极低 |
| **单道批处理系统** | 自动性、顺序性、单道性；CPU 仍大量空闲（I/O 时 CPU 等待） |
| **多道批处理系统** ⭐ | 多道程序并发执行，CPU 利用率高；无交互能力 |
| **分时操作系统** ⭐ | 多路性、独立性、及时性、交互性；每个用户感觉独占系统 |
| **实时操作系统** ⭐ | 及时性和可靠性；硬实时/软实时 |
| **网络操作系统** | 实现网络中各台计算机之间的通信和资源共享 |
| **分布式操作系统** | 若干台计算机相互协作完成同一任务 |
| **个人计算机操作系统** | 单用户多任务 |

### 1.2.2 各类操作系统的特点

#### 批处理系统 vs 分时系统 vs 实时系统 ⭐

| 特征 | 批处理 | 分时 | 实时 |
|------|--------|------|------|
| 设计目标 | 提高资源利用率 | 用户交互 | 及时响应 |
| 多路性 | 作业级 | 用户级 | 过程控制级 |
| 独立性 | 作业间独立 | 用户间独立 | 任务间独立 |
| 及时性 | 较差 | 用户可接受 | 严格时间约束 |
| 交互性 | 无 | 强 | 有限 |
| 可靠性 | 一般 | 一般 | 高 |

## 1.3 操作系统的运行环境 ⭐

### 1.3.1 内核态和用户态

**内核态（管态/核心态）：** 操作系统内核程序运行时所处的状态，具有最高权限，可以执行一切指令，访问所有寄存器和存储区。

**用户态（目态）：** 用户程序运行时所处的状态，权限受限，不能执行特权指令。

⚠️ **特权指令只能在内核态下执行：**
- 输入输出指令
- 置中断指令
- 存取用于内存保护的寄存器
- 送 PSW 到程序状态字寄存器的指令

⚠️ **内核态 ↔ 用户态的切换：**
- **用户态 → 内核态：** 通过**中断、异常或系统调用**（陷入/trap）
- **内核态 → 用户态：** 通过设置 PSW（程序状态字）为用户态

### 1.3.2 中断和异常 ⭐

| 类型 | 来源 | 例子 |
|------|------|------|
| **中断（外中断）** | CPU 外部 | I/O 中断、时钟中断 |
| **异常（内中断/陷入）** | CPU 内部 | 除零异常、缺页异常、非法指令 |

⚠️ **中断和异常是用户态进入内核态的唯一途径！**

### 1.3.3 系统调用 ⭐⭐

系统调用是操作系统提供给应用程序（程序员）使用的接口，是应用程序获取操作系统服务的唯一途径。

| 系统调用类型 | 功能 |
|------------|------|
| 设备管理 | 请求/释放设备 |
| 文件管理 | 创建/删除/读/写文件 |
| 进程控制 | 创建/终止进程、获得/设置进程属性 |
| 进程通信 | 进程间消息传递、信号量操作 |
| 内存管理 | 分配/释放内存 |

⚠️ **系统调用的执行过程：**
1. 用户程序调用系统调用（陷入指令/trap指令/int指令）
2. CPU 由用户态切换到内核态
3. 通过中断向量找到系统调用入口
4. 执行操作系统内核程序
5. 执行完毕，返回用户态

⚠️ **系统调用与库函数的区别：**
- 系统调用是操作系统的一部分，运行在内核态
- 库函数是用户态的函数，可能调用系统调用，也可能不调用

### 1.3.4 大内核和微内核

| 类型 | 说明 | 优缺点 |
|------|------|--------|
| **大内核** | 将主要功能模块都作为系统内核 | 性能高，但内核庞大复杂 |
| **微内核** | 只把最基本的功能保留在内核中 | 内核小、可靠、易扩展，但性能较低 |

## 1.4 本章例题

### 例题1：操作系统的特征

下列关于操作系统的叙述中，正确的是（  ）。

A. 批处理系统不具有交互性
B. 分时系统中用户感觉独占计算机
C. 实时系统必须在规定时间内完成任务
D. 以上都正确

**答案：D**

### 例题2：用户态和内核态

下列操作中，必须在内核态下执行的是（  ）。

A. 关中断  B. 读取时钟  C. 系统调用  D. A和B

**答案：D**

解析：关中断和读取时钟都是特权指令。系统调用是用户程序发起的（用户态），但执行时会切换到内核态。

### 例题3：中断和系统调用

系统调用是通过什么机制实现的？

**答案：** 通过**中断/陷入（trap）机制**实现。用户程序执行陷入指令，触发软中断，CPU 由用户态切换到内核态，进入操作系统内核执行系统调用服务。
