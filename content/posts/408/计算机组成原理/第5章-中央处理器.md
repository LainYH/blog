# 第5章 中央处理器

## 5.1 CPU的功能和基本结构

### 5.1.1 CPU的功能

| 功能 | 说明 |
|------|------|
| **指令控制** | 取指令、分析指令、执行指令（完成指令序列的控制） |
| **操作控制** | 产生完成每条指令所需的控制信号 |
| **时间控制** | 对各种操作实施时间上的定时 |
| **数据加工** | 对数据进行算术和逻辑运算 |
| **中断处理** | 处理各种中断和异常 |

### 5.1.2 CPU的基本结构 ⭐

```
CPU = 运算器 + 控制器

运算器 = ALU + 暂存寄存器 + ACC（累加器）+ 通用寄存器组 + PSW（程序状态字寄存器）+ 移位器 + 计数器

控制器 = PC（程序计数器）+ IR（指令寄存器）+ MAR + MDR + 指令译码器 + 时序系统 + 微操作信号发生器
```

#### CPU 中的主要寄存器 ⭐

| 寄存器 | 功能 |
|--------|------|
| **PC（程序计数器）** | 存放当前欲执行指令的地址，自动 +1 指向下一条指令 |
| **IR（指令寄存器）** | 存放当前正在执行的指令 |
| **MAR（存储器地址寄存器）** | 存放访存地址 |
| **MDR（存储器数据寄存器）** | 存放从主存读出或欲写入主存的数据 |
| **ACC（累加器）** | 暂时存放 ALU 运算的一个操作数或结果 |
| **PSW（程序状态字寄存器/标志寄存器）** | 保存运算结果的状态标志（ZF、SF、OF、CF 等） |

⚠️ **PC 的修改：**
- 顺序执行：PC + "1"（"1" = 一条指令长度）
- 转移执行：PC ← 目标地址
- **在取指周期结束后、执行指令之前**，PC 已经 + "1"

⚠️ **PC 和 MAR 都是地址寄存器，但功能不同：**
- PC 存放的是指令地址（在取指阶段使用）
- MAR 存放的是数据或指令的访存地址

## 5.2 指令执行过程 ⭐

### 5.2.1 指令周期

**指令周期：** CPU 从主存中取出并执行一条指令所需的全部时间。

一个指令周期 = 若干个**机器周期（CPU周期）**

一个机器周期 = 若干个**时钟周期（节拍/T周期）**

```
指令周期 = 取指周期 + 间址周期 + 执行周期 + 中断周期
```

| 周期 | 说明 |
|------|------|
| **取指周期** | 从主存取指令（每个指令都有） |
| **间址周期** | 取操作数地址（间接寻址才有） |
| **执行周期** | 执行指令操作 |
| **中断周期** | 处理中断请求（有中断才有） |

### 5.2.2 指令执行的数据流

#### 取指周期 ⭐

1. PC → MAR（将指令地址送 MAR）
2. MAR → 地址总线 → 主存
3. 1 → R（读命令）
4. 主存 → 数据总线 → MDR
5. MDR → IR（将指令送 IR）
6. PC + "1" → PC

#### 间址周期

1. IR 中的地址码 → MAR
2. MAR → 地址总线 → 主存
3. 1 → R
4. 主存 → 数据总线 → MDR
5. MDR → IR 的地址码字段（有效地址替换形式地址）

#### 执行周期

根据指令类型不同而不同。

#### 中断周期

1. 保存断点（PC 的值入栈）
2. 形成中断服务程序入口地址 → PC
3. 关中断

## 5.3 数据通路 ⭐

### 5.3.1 数据通路的基本结构

| 类型 | 说明 |
|------|------|
| **CPU 内部总线** | 连接 CPU 内部各寄存器和 ALU |
| **三总线结构** | AB（地址总线）、DB（数据总线）、CB（控制总线） |

### 5.3.2 数据传送

**寄存器之间通过内部总线的数据传送：**

例：实现 (R₀) → R₁

1. R₀ → Bus（R₀out 有效）
2. Bus → R₁（R₁in 有效）

⚠️ **同一时刻只能有一个寄存器向总线输出数据（out），但可以有多个寄存器从总线接收数据（in）。**

## 5.4 控制器的功能和工作原理 ⭐⭐

### 5.4.1 控制器的分类

| 类型 | 说明 |
|------|------|
| **硬布线控制器** | 用组合逻辑电路产生控制信号 |
| **微程序控制器** | 用微程序（微指令序列）产生控制信号 |

### 5.4.2 硬布线控制器

**基本思想：** 控制信号由组合逻辑电路（门电路）直接产生。

控制信号 = f（指令操作码, 时钟周期, 状态标志, 外部信号）

**特点：**
- 速度快（纯组合逻辑）
- 设计复杂、不灵活、难以扩展
- 用于 RISC 处理器

### 5.4.3 微程序控制器 ⭐⭐⭐（核心考点）

#### 基本概念

| 术语 | 定义 |
|------|------|
| **微命令（微操作控制信号）** | 控制部件通过控制线向执行部件发出的控制信号 |
| **微操作** | 微命令的执行过程 |
| **微指令** | 若干微命令的组合，一条微指令对应一个时钟周期内的所有微操作 |
| **微程序** | 微指令序列，**一条机器指令对应一个微程序** |
| **控制存储器（CM）** | 存放微程序的存储器，用 ROM 实现 |

⚠️ **核心对应关系：**
```
机器指令 → 微程序 → 多条微指令 → 每条微指令包含多个微命令
```

#### 微程序控制器的结构

```
控制存储器(CM) → 微指令寄存器(μIR) → 微命令形成部件 → 控制信号
         ↑                                        |
      微地址形成部件 ← 微地址寄存器(μMAR) ← 下址字段
```

#### 微指令的格式 ⭐

**水平型微指令：**
```
| 操作控制字段 | 顺序控制字段 |
```
- 操作控制字段：发出微命令
- 顺序控制字段：指明下一条微指令的地址

**垂直型微指令：**
```
| 微操作码 | 目的地址 | 源地址 |
```

| 对比 | 水平型微指令 | 垂直型微指令 |
|------|------------|------------|
| 微指令长度 | 长 | 短 |
| 并行性 | 高（可同时发出多个微命令） | 低（一条微指令通常只发一个微命令） |
| 微程序长度 | 短 | 长 |
| 编程 | 复杂 | 简单 |

#### 微指令的编码方式 ⭐

| 编码方式 | 说明 | 特点 |
|---------|------|------|
| **直接编码（直接控制）** | 每个微命令占一位 | 最快，但微指令最长 |
| **字段直接编码** ⭐ | 将微命令分组，每组编码 | 折中方案 |
| **字段间接编码** | 某些字段的含义由另一字段决定 | 进一步缩短，但更慢 |

⚠️ **字段直接编码的分组原则：**
1. 互斥性微命令分在同一组（不会同时出现）
2. 相容性微命令分在不同组（可以同时出现）
3. 每组留一个编码表示不操作

#### 微指令的地址形成方式 ⭐

| 方式 | 说明 |
|------|------|
| **下址字段法** | 微指令的下址字段直接给出下一条微指令的地址 |
| **增量计数法** | (μMAR) + 1 → μMAR |
| **断定方式（下址 + 判断）** | 根据测试条件确定下址 |
| **由机器指令的操作码形成** | 机器指令的操作码 → 微程序入口地址 |

#### 微程序设计

**取指微程序：** 公共的微程序，每条机器指令执行前都要先执行。
**执行微程序：** 不同机器指令的执行微程序不同。

⚠️ **微程序入口地址：** 由机器指令的操作码通过**地址映射**得到。

## 5.5 指令流水线 ⭐⭐⭐（核心考点）

### 5.5.1 指令流水线的基本概念

将一条指令的执行过程分为若干个阶段，每个阶段由不同的硬件完成，多条指令的不同阶段可以同时进行。

**五段流水线：**
```
取指(IF) → 译码(ID) → 执行(EX) → 访存(MEM) → 写回(WB)
```

### 5.5.2 流水线的性能指标 ⭐⭐

#### 吞吐率

$$TP = \frac{n}{T_k} = \frac{n}{(k + n - 1)\Delta t}$$

其中：
- n 为任务数
- k 为流水线段数
- Δt 为时钟周期
- T_k 为完成 n 个任务的总时间

**最大吞吐率：**
$$TP_{max} = \frac{1}{\Delta t}$$

#### 加速比

$$S = \frac{nk\Delta t}{(k+n-1)\Delta t} = \frac{nk}{k+n-1}$$

其中 nkΔt 为顺序执行的时间。

**最大加速比：**
$$S_{max} = k$$

#### 效率

$$\eta = \frac{n}{k+n-1} = TP \times \Delta t$$

### 5.5.3 流水线的冒险（冲突）⚠️

#### （1）结构冒险（资源冲突）

多条指令在同一时刻争用同一资源。

**解决方法：**
- 增加资源（如将指令存储器和数据存储器分开——**哈佛结构**）
- 流水线暂停（插入气泡）

#### （2）数据冒险（数据冲突）⭐⭐

后续指令需要用到前面指令尚未产生或尚未写入的结果。

| 类型 | 说明 | 示例 |
|------|------|------|
| **RAW（写后读）** | 后面的指令读取前面指令要写的数据 | ADD R₁, R₂, R₃; SUB R₄, R₁, R₅ |
| **WAR（读后写）** | 后面的指令写前面指令要读的数据 | 在乱序执行中可能出现 |
| **WAW（写后写）** | 两条指令写同一个位置 | 在乱序执行中可能出现 |

⚠️ **在顺序流水线中，主要关注 RAW 冲突。**

**解决方法：**
1. **数据旁路（转发/旁路 bypassing）** ⭐：将 ALU 的计算结果直接转发给下一条指令，不等写入寄存器
2. **插入流水线气泡（stall）**：暂停流水线若干周期
3. **指令调度**：编译器调整指令顺序

#### （3）控制冒险（控制冲突）

遇到转移、调用、返回等指令导致 PC 改变。

**解决方法：**
1. **分支预测**（静态预测 / 动态预测）
2. **预取转移目标**
3. **延迟转移（延迟槽）**

### 5.5.4 流水线的多发技术

| 技术 | 说明 |
|------|------|
| **超标量流水线** | 每个时钟周期内并发执行多条指令 |
| **超流水线** | 将流水线的各段再细分，每个时钟周期内分时发射多条指令 |
| **超长指令字（VLIW）** | 由编译器将多条能并行执行的指令组合成一条超长指令 |

## 5.6 中断系统 ⭐⭐

### 5.6.1 中断的基本概念

**中断：** CPU 在执行程序的过程中，出现了某些异常情况或特殊请求，CPU 暂时中止现行程序的执行，转去处理这些异常或请求，处理完毕后再返回继续执行原程序。

### 5.6.2 中断的分类

| 分类 | 类型 |
|------|------|
| 按来源 | **内中断（异常）**：CPU 内部产生的；**外中断**：外部设备产生的 |
| 按是否可屏蔽 | **不可屏蔽中断（NMI）**：必须响应；**可屏蔽中断（INTR）**：可被屏蔽 |
| 按优先级 | 硬件故障 > 非屏蔽中断 > 可屏蔽中断 |

#### 常见的中断和异常

| 类型 | 说明 |
|------|------|
| **故障（Fault）** | 可修复的异常，返回后重新执行引起故障的指令（如缺页） |
| **陷阱（Trap）** | 有意的异常，执行完后继续执行下一条指令（如系统调用） |
| **终止（Abort）** | 不可恢复的致命错误，程序终止（如硬件故障） |

### 5.6.3 中断处理过程 ⭐

1. **关中断**
2. **保存断点**（PC 入栈）
3. **引出中断服务程序**（中断向量 → PC）
4. **保存现场**（PSW、通用寄存器入栈）
5. **开中断**
6. **执行中断服务**
7. **关中断**
8. **恢复现场**（出栈）
9. **开中断**
10. **中断返回**

⚠️ **中断响应在指令执行结束后进行。**

⚠️ **中断向量：** 中断服务程序的入口地址。
⚠️ **中断向量表：** 存放所有中断向量的表格，按中断类型号索引。

### 5.6.4 多重中断和中断屏蔽

**多重中断：** 在处理一个中断的过程中，又出现了新的中断请求。

**中断屏蔽技术：** 通过设置中断屏蔽字，可以动态改变中断的优先级。

⚠️ **中断屏蔽字：** 每个中断源对应一个屏蔽位，为 1 表示屏蔽该中断源。

## 5.7 本章例题

### 例题1：流水线性能计算

一个 4 段流水线，各段执行时间分别为 100ns, 80ns, 100ns, 80ns。连续执行 100 条指令。

求：吞吐率、加速比、效率。

**解析：**
- 时钟周期 = max(100, 80, 100, 80) = 100ns
- 总时间 = (4 + 100 - 1) × 100 = 10300ns
- 顺序执行时间 = 100 × (100+80+100+80) = 36000ns
- 吞吐率 = 100 / 10300ns = 9.71 × 10⁶ 条/s
- 加速比 = 36000 / 10300 = 3.50
- 效率 = (100 × 360) / (4 × 10300) = 36000/41200 = 87.4%

### 例题2：数据冲突分析

以下指令序列在 5 段流水线（IF-ID-EX-MEM-WB）中执行，是否存在数据冲突？如何解决？

```
I1: ADD R1, R2, R3    // R1 = R2 + R3
I2: SUB R4, R1, R5    // R4 = R1 - R5
I3: AND R6, R1, R7    // R6 = R1 AND R7
```

**解析：**
- I1 写 R1（在 WB 阶段），I2 读 R1（在 ID 阶段需要读操作数）
- I2 的 ID 阶段在 I1 的 EX 阶段之后，此时 ALU 结果已产生
- **解决方案：数据旁路（forwarding）**——将 I1 的 EX 阶段结果直接转发给 I2 的 EX 阶段

若没有旁路，则需插入 2 个 stall（等到 I1 写回后再读）。

### 例题3：中断优先级

设有 4 个中断源 D₁、D₂、D₃、D₄，优先级 D₁ > D₂ > D₃ > D₄。它们的中断屏蔽字分别为：

| 中断源 | 屏蔽字 |
|--------|--------|
| D₁ | 1 1 1 1 |
| D₂ | 0 1 1 1 |
| D₃ | 0 0 1 1 |
| D₄ | 0 0 0 1 |

若 CPU 正在执行 D₂ 的中断服务程序时，D₁、D₃、D₄ 同时发出请求，处理顺序如何？

**答案：** D₂ 的屏蔽字为 0111，表示 D₂ 只屏蔽 D₂、D₃、D₄（自己和低优先级）。D₁ 未被屏蔽，D₁ 可以打断 D₂；D₃、D₄ 被屏蔽。

处理顺序：**先响应 D₁**，D₁ 执行完后回到 D₂，D₂ 执行完后响应 D₃（或 D₄，看同时到达的排队机制）。
