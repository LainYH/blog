# 第1章 计算机系统概述

## 1.1 计算机发展历程

### 1.1.1 计算机硬件的发展

| 代数 | 时间 | 主要元件 | 特点 |
|------|------|---------|------|
| 第一代 | 1946-1958 | 电子管 | 体积大、耗电多、速度低 |
| 第二代 | 1958-1964 | 晶体管 | 体积减小、可靠性提高 |
| 第三代 | 1964-1971 | 中小规模集成电路 | 体积进一步减小 |
| 第四代 | 1971至今 | 大规模/超大规模集成电路 | 微处理器诞生 |

⚠️ **摩尔定律：** 集成电路上可容纳的晶体管数目约每隔 18 个月便会增加一倍。

### 1.1.2 软件的发展

- 机器语言 → 汇编语言 → 高级语言
- 操作系统的发展：无操作系统 → 批处理 → 分时系统 → 现代操作系统

---

## 1.2 计算机系统层次结构

### 1.2.1 计算机系统的组成

计算机系统 = **硬件** + **软件**

⚠️ **易混淆：** 软件和硬件在逻辑上是等效的。

### 1.2.2 计算机硬件的基本组成

冯·诺依曼机的特点：
1. 采用**存储程序**的工作方式
2. 计算机硬件系统由**运算器、存储器、控制器、输入设备、输出设备**五大部件组成
3. 指令和数据以**同等地位**存储在存储器中，可按地址寻访
4. 指令和数据均用**二进制代码**表示
5. 指令由**操作码**和**地址码**组成

⚠️ **冯·诺依曼计算机的核心思想：存储程序。**

⚠️ **现代计算机的结构：** 以**存储器**为中心（早期以运算器为中心）。

#### 各部件功能

| 部件 | 功能 |
|------|------|
| **运算器（ALU）** | 进行算术运算和逻辑运算 |
| **存储器** | 存放数据和程序 |
| **控制器（CU）** | 指挥和控制各部件协调工作，完成取指令、分析指令和执行指令的操作 |
| **输入设备** | 将信息转换为机器能识别的形式 |
| **输出设备** | 将结果转换为人们熟悉的形式 |

⚠️ **CPU = 运算器 + 控制器（+ 寄存器 + 中断系统）**

⚠️ **主机 = CPU + 主存储器**

### 1.2.3 计算机软件的分类

| 分类 | 包含 |
|------|------|
| **系统软件** | 操作系统、数据库管理系统、语言处理程序、分布式软件系统、网络软件系统、标准库程序、服务性程序 |
| **应用软件** | 各种面向用户的程序 |

⚠️ **三个级别的语言：**
1. **机器语言：** 二进制代码，计算机可直接执行
2. **汇编语言：** 助记符表示，需汇编程序翻译
3. **高级语言：** 接近自然语言，需编译程序或解释程序翻译

⚠️ **翻译程序的分类：**
- **汇编程序：** 汇编语言 → 机器语言
- **编译程序：** 高级语言 → 汇编/机器语言（整体翻译）
- **解释程序：** 高级语言逐句翻译执行（不生成目标代码）

### 1.2.4 计算机系统的层次结构

| 层次 | 名称 | 说明 |
|------|------|------|
| 第5层 | 高级语言级 | 面向用户 |
| 第4层 | 汇编语言级 | 汇编程序 |
| 第3层 | 操作系统级 | 操作系统虚拟机 |
| 第2层 | 一般机器级（传统机器级） | 微程序 |
| 第1层 | 微程序机器级 | 硬件直接执行微指令 |

⚠️ **硬件和软件的界面：** 一般机器级和操作系统级之间的界面。不同计算机中此界面位置不同。

## 1.3 计算机的性能指标

### 1.3.1 主要性能指标

#### （1）机器字长 ⭐

CPU 一次能处理数据的位数，通常等于 CPU 内部**通用寄存器**的位数。

⚠️ 机器字长决定了计算机的运算精度。字长越长，运算精度越高。

#### （2）数据通路带宽 ⭐

数据总线一次所能并行传送信息的位数。

⚠️ **注意区分机器字长和数据通路带宽：** 两者可以不同。

#### （3）主存容量 ⭐

主存储器所能存储信息的最大容量。

- MAR（存储器地址寄存器）的位数反映了存储单元的**个数**
- MDR（存储器数据寄存器）的位数反映了存储单元的**字长**

⚠️ **MAR 的位数决定了可寻址的最大范围：** 若 MAR 为 n 位，则最大可寻址空间为 2^n 个存储单元。

#### （4）运算速度

**① 吞吐量和响应时间**
- **吞吐量：** 系统在单位时间内处理请求的数量
- **响应时间：** 从用户向计算机发送请求到系统对该请求做出响应并获得所需结果的等待时间

**② CPU 时钟周期 ⭐**

机器内部主时钟的频率（主频）的倒数。

$$\text{CPU时钟周期} = \frac{1}{\text{主频}}$$

**③ CPI（Cycles Per Instruction）⭐**

执行一条指令所需的时钟周期数。

$$\text{CPI} = \frac{\text{总时钟周期数}}{\text{指令条数}}$$

**④ CPU 执行时间 ⭐**

$$\text{CPU执行时间} = \frac{\text{CPU时钟周期数}}{\text{主频}} = \frac{\text{指令条数} \times \text{CPI}}{\text{主频}}$$

**⑤ MIPS（Million Instructions Per Second）⭐**

$$\text{MIPS} = \frac{\text{指令条数}}{\text{执行时间} \times 10^6} = \frac{\text{主频}}{\text{CPI} \times 10^6}$$

**⑥ MFLOPS、GFLOPS、TFLOPS**

$$\text{MFLOPS} = \frac{\text{浮点操作次数}}{\text{执行时间} \times 10^6}$$

#### （5）基准程序（Benchmark）

用来测量计算机处理性能的一组标准化测试程序。

## 1.4 本章例题

### 例题1：性能计算（2013年真题改编）

某计算机的主频为 1.2GHz，各类指令的 CPI 如下：A 类指令占 60%，CPI 为 3；B 类指令占 25%，CPI 为 4；C 类指令占 10%，CPI 为 5；D 类指令占 5%，CPI 为 6。

求该计算机的 MIPS。

**解析：**

加权 CPI = 0.6×3 + 0.25×4 + 0.1×5 + 0.05×6 = 1.8 + 1.0 + 0.5 + 0.3 = 3.6

MIPS = 主频 / (CPI × 10⁶) = 1.2×10⁹ / (3.6 × 10⁶) = 1200 / 3.6 ≈ **333.3**

### 例题2：存储容量计算

设 MAR 为 16 位，MDR 为 32 位，求存储器的容量和可寻址单元数。

**解析：**

- MAR = 16 位 → 可寻址单元数 = 2^16 = **65536 = 64K**
- MDR = 32 位 → 每个存储单元 32 位 = 4 字节
- 存储器容量 = 2^16 × 32bit = 2^16 × 4B = **256KB**

### 例题3：层次结构

判断以下说法是否正确：
（1）高级语言程序可以直接在机器上运行
（2）编译程序和汇编程序都属于系统软件
（3）汇编语言编写的程序必须经过汇编程序翻译后才能执行

**答案：**
（1）错误——高级语言需要经过编译或解释才能执行
（2）正确——都是翻译程序，属于系统软件
（3）正确——汇编语言不能直接被机器理解
