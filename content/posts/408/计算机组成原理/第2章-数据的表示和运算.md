# 第2章 数据的表示和运算

## 2.1 数制与编码

### 2.1.1 进位计数制及其转换

#### 常用进制

| 进制 | 基数 | 数码 |
|------|------|------|
| 二进制 | 2 | 0, 1 |
| 八进制 | 8 | 0-7 |
| 十进制 | 10 | 0-9 |
| 十六进制 | 16 | 0-9, A-F |

#### 任意进制 → 十进制

按权展开相加法：

$$N = \sum_{i=m}^{n-1} D_i \times r^i$$

例：(1011.101)₂ = 1×2³ + 0×2² + 1×2¹ + 1×2⁰ + 1×2⁻¹ + 0×2⁻² + 1×2⁻³ = 8+2+1+0.5+0.125 = 11.625

#### 十进制 → 任意进制

**整数部分：除 r 取余法**（从下往上读取）

例：将 75 转换为二进制：
75 ÷ 2 = 37 余 1
37 ÷ 2 = 18 余 1
18 ÷ 2 = 9 余 0
9 ÷ 2 = 4 余 1
4 ÷ 2 = 2 余 0
2 ÷ 2 = 1 余 0
1 ÷ 2 = 0 余 1

结果：(75)₁₀ = (1001011)₂

**小数部分：乘 r 取整法**（从上往下读取）

例：将 0.6875 转换为二进制：
0.6875 × 2 = 1.375 → 取 1
0.375 × 2 = 0.75 → 取 0
0.75 × 2 = 1.5 → 取 1
0.5 × 2 = 1.0 → 取 1

结果：(0.6875)₁₀ = (0.1011)₂

⚠️ **注意：** 并非所有十进制小数都能精确转换为有限位二进制小数。

#### 二进制 ↔ 八进制/十六进制

- **二 → 八：** 每 3 位二进制数对应 1 位八进制数
- **二 → 十六：** 每 4 位二进制数对应 1 位十六进制数

### 2.1.2 BCD码（Binary-Coded Decimal）

用 4 位二进制数来表示 1 位十进制数。

| BCD码 | 编码方式 | 特点 |
|-------|---------|------|
| **8421码** | 有权码，权值8,4,2,1 | 最常用，若两个数相加结果不在0-9范围内需+6修正 |
| **余3码** | 8421码 + 0011 | 无权码，+6修正的自然结果 |
| **2421码** | 有权码，权值2,4,2,1 | 编码不唯一，规定≥5的编码首位为1 |

⚠️ **8421码的加法修正规则：** 若两个 8421 码相加结果 > 9（即结果不在 0000-1001 范围内），需加 6（0110）修正，并向高位进位。

例：用 8421 码计算 5 + 8：
- 5 的 8421 码：0101
- 8 的 8421 码：1000
- 相加：0101 + 1000 = 1101（> 9，需修正）
- 修正：1101 + 0110 = 10011
- 结果：10011（低4位 0011 = 3，进位 1，即 13）

### 2.1.3 字符与字符串

#### ASCII码

- 7 位二进制编码，共 128 个字符
- 数字 '0'-'9' 的 ASCII 码为 48-57（30H-39H）
- 大写字母 'A'-'Z' 的 ASCII 码为 65-90（41H-5AH）
- 小写字母 'a'-'z' 的 ASCII 码为 97-122（61H-7AH）

#### 字符串的存储

**大端模式（Big-Endian）：** 高字节存放在低地址（符合人类阅读顺序）
**小端模式（Little-Endian）：** 低字节存放在低地址

⚠️ **408考试中常用大端模式。**

### 2.1.4 奇偶校验码

在数据最高位前增加一位**校验位**，使得整个编码中"1"的个数为奇数（奇校验）或偶数（偶校验）。

⚠️ **奇偶校验只能检测奇数位错误，不能检测偶数位错误，也不能纠正错误。**

## 2.2 定点数的表示和运算 ⭐⭐（重点）

### 2.2.1 定点数的表示

定点数：小数点的位置固定的数。

- **定点小数：** 小数点固定在最高数值位之前，纯小数
- **定点整数：** 小数点固定在最低数值位之后，纯整数

#### 无符号数

所有位都表示数值，没有符号位。

n 位无符号数的表示范围：0 ~ 2^n - 1

#### 有符号数的表示

| 表示方法 | 说明 |
|---------|------|
| **原码** | 符位 + 数值的绝对值 |
| **反码** | 正数同原码；负数：符号位不变，数值位按位取反 |
| **补码** ⭐ | 正数同原码；负数：反码 + 1（或：从右往左找到第一个 1，其左侧各位取反） |
| **移码** | 补码的符号位取反 |

⚠️ **补码是计算机中表示有符号整数的标准方式！**

#### 各种码的表示范围（n位，含1位符号位）

| 表示方法 | 最小值 | 最大值 | 特点 |
|---------|--------|--------|------|
| **原码** | -(2^(n-1)-1) | 2^(n-1)-1 | +0和-0两个零 |
| **反码** | -(2^(n-1)-1) | 2^(n-1)-1 | +0和-0两个零 |
| **补码** | -2^(n-1) | 2^(n-1)-1 | **唯一零**，多表示一个负数 |
| **移码** | -2^(n-1) | 2^(n-1)-1 | 唯一零 |

⚠️ **重要结论：**
- n 位补码表示范围：-2^(n-1) ~ 2^(n-1) - 1
- 补码比原码多表示一个 -2^(n-1)
- **真值 0 的补码表示是唯一的**：[+0]补 = [-0]补 = 0000...000

#### 补码的几个重要性质 ⚠️

1. **[x]补 按位取反再加 1 = [-x]补**（求相反数的补码）
2. **补码的符号位可以参与运算**
3. **[x+y]补 = [x]补 + [y]补**（模运算，自然丢失溢出位）
4. **[x-y]补 = [x]补 + [-y]补**

### 2.2.2 定点数的运算

#### （1）移位运算

| 类型 | 正数 | 负数 |
|------|------|------|
| **算术左移** | 低位补 0 | 低位补 0（补码） |
| **算术右移** | 高位补 0 | 高位补 1（补码） |

⚠️ **算术移位 vs 逻辑移位：**
- **算术移位：** 考虑符号位，用于有符号数
- **逻辑移位：** 不考虑符号位，用于无符号数
- 逻辑左移 = 算术左移
- 逻辑右移：高位补 0

⚠️ **溢出判断：** 算术左移时，若符号位发生变化，则溢出。

#### （2）加减法运算 ⭐⭐

**补码加减法的规则：**
$$[A+B]补 = [A]补 + [B]补 \pmod{2^n}$$
$$[A-B]补 = [A]补 + [-B]补 \pmod{2^n}$$

⚠️ **[-B]补 的求法：[B]补 连同符号位按位取反，末位加 1。**

**溢出判断 ⚠️（高频考点）：**

| 方法 | 判断条件 |
|------|---------|
| **一位符号位判断** | 参加操作的两个数符号相同，结果符号与之不同则溢出 |
| **双符号位（变形补码）** | 结果的两个符号位不同则溢出：00 正数无溢出，01 正溢出，10 负溢出，11 负数无溢出 |
| **一位符号位 + 进位** | 最高位进位 ⊕ 符号位进位 = 1 时溢出 |

⚠️ **正溢出：** 两个正数相加结果为负
⚠️ **负溢出：** 两个负数相加结果为正

**例：** 用 8 位补码计算 (+15) + (+20)：
- [+15]补 = 00001111
- [+20]补 = 00010100
- 相加 = 00100011 = [+35]补 ✓

**例：** 用 8 位补码计算 (+100) + (+50)：
- [+100]补 = 01100100
- [+50]补 = 00110010
- 相加 = 10010110 = [-106]补（发生了正溢出！）

#### （3）乘法运算 ⭐

**原码一位乘法：**

| 步骤 | 操作 |
|------|------|
| 初始化 | 部分积 ACC = 0，乘数在 MQ 中 |
| 判断乘数最低位 | 若为 1，ACC += 被乘数；若为 0，不操作 |
| 右移 | ACC 和 MQ 整体右移 1 位（ACC 的最低位移入 MQ 的最高位） |
| 重复 | 重复 n 次（n 为数值位数） |

⚠️ **符号位单独处理：** 结果符号 = 被乘数符号 ⊕ 乘数符号。

**补码一位乘法（Booth算法）⭐⭐（重要）：**

设 [X]补 和 [Y]补 为两个补码。

1. 在乘数末位补一个附加位 y(n+1) = 0
2. 根据 [yᵢ, yᵢ₊₁] 判断操作：
   - 00 或 11：部分积加 0
   - 01：部分积加 [X]补
   - 10：部分积加 [-X]补
3. 算术右移 1 位
4. 重复 n 次
5. 最后一步不移位

⚠️ **Booth 算法的优点：** 无论被乘数和乘数的正负，都可以直接用补码进行运算，无需单独处理符号位。

#### （4）除法运算

**原码恢复余数法：**
1. 符号位单独处理（同号为正，异号为负）
2. 用被除数减去除数（绝对值），若余数 ≥ 0 商 1，余数 < 0 商 0 并**恢复余数**
3. 左移后重复上述过程

**原码不恢复余数法（加减交替法）⭐：**
- 余数 > 0：商 1，余数左移 1 位，减除数
- 余数 < 0：商 0，余数左移 1 位，加除数

⚠️ **补码除法：** 比较余数和除数是否同号：
- 同号：商 1，余数左移后减除数
- 异号：商 0，余数左移后加除数

## 2.3 浮点数的表示和运算 ⭐⭐（高频考点）

### 2.3.1 浮点数的表示

浮点数的表示形式：

$$N = (-1)^S \times M \times R^E$$

其中：
- **S：符号位**（0 为正，1 为负）
- **M：尾数（mantissa）**，定点小数，表示精度
- **R：基数（radix）**，通常为 2
- **E：阶码（exponent）**，定点整数，表示范围

### 2.3.2 浮点数的格式

```
| 1位 | m位 | 1位 | n位 |
| 阶符 | 阶码 | 数符 | 尾数 |
```

### 2.3.3 浮点数的规格化 ⚠️

**规格化的目的：** 尾数的最高有效位为有效数字，使精度最大化。

| 基数 | 规格化条件 |
|------|-----------|
| 基数为 2（原码） | 1/2 ≤ |M| < 1，即尾数最高位为 1 |
| 基数为 2（补码） | 正数：1/2 ≤ M < 1（0.1xxx...）；负数：-1 ≤ M < -1/2（1.0xxx...）|
| 基数为 4 | 尾数最高 2 位不全为 0 |
| 基数为 8 | 尾数最高 3 位不全为 0 |

⚠️ **补码规格化的判断方法：** 数符位与尾数最高位不同。

⚠️ **左规和右规：**
- **左规：** 尾数左移 1 位，阶码减 1（处理非规格化数）
- **右规：** 尾数右移 1 位，阶码加 1（处理尾数溢出）

### 2.3.4 IEEE 754 标准 ⭐⭐（常考）

#### 单精度浮点数（32位）

```
| 1位 | 8位  | 23位 |
| S   | 阶码 | 尾数 |
```

- 阶码采用**移码**表示，偏置值 = 127（2^7 - 1）
- 尾数采用**原码**表示，隐藏最高位 1（隐含的 1.M）

$$\text{真值} = (-1)^S \times 1.M \times 2^{E-127}$$

#### 双精度浮点数（64位）

```
| 1位 | 11位 | 52位 |
| S   | 阶码 | 尾数 |
```

- 阶码偏置值 = 1023（2^10 - 1）
$$\text{真值} = (-1)^S \times 1.M \times 2^{E-1023}$$

#### IEEE 754 特殊值 ⚠️

| 阶码 | 尾数 | 含义 |
|------|------|------|
| 全 0 | 全 0 | ±0 |
| 全 0 | 非 0 | 非规格化数（无隐藏位，0.M） |
| 全 1 | 全 0 | ±∞ |
| 全 1 | 非 0 | NaN（非数） |
| 1~254 | 任意 | 规格化数 |

⚠️ **单精度表示范围：**
- 最小正规格化数：2^(-126)（阶码 = 1，E-127 = -126）
- 最大正规格化数：(2-2^(-23)) × 2^127

**例：** 将 IEEE 754 单精度数 (BF800000)₁₆ 转换为十进制。

- 十六进制 → 二进制：1011 1111 1000 0000 0000 0000 0000 0000
- S = 1（负数）
- 阶码 = 0111 1111 = 127，E = 127-127 = 0
- 尾数 = 000...0，1.M = 1.0
- 结果：(-1)^1 × 1.0 × 2^0 = **-1.0**

**例：** 将十进制数 -13.625 转换为 IEEE 754 单精度表示。

- 13.625 的二进制：1101.101
- 规格化：1.101101 × 2^3
- S = 1（负数）
- E = 3 + 127 = 130 = 10000010
- M = 10110100000000000000000（去掉隐含的1，后面补0）
- IEEE 754：1 10000010 10110100000000000000000
- 十六进制：C15A0000

### 2.3.5 浮点数的加减运算 ⭐⭐

**运算步骤：**

1. **对阶（求阶差）：** 小阶向大阶看齐，阶差为多少尾数右移多少位
2. **尾数求和：** 按定点数加减法运算
3. **规格化：**
   - 若结果为 ±1x.xxx（补码中符号位和最高数值位相同），需**右规**
   - 若结果为 ±0.00...01xxx（补码中符号位和最高数值位之间有多个0），需**左规**
4. **舍入：** 右规时会丢失低位，需要舍入
5. **溢出判断：** 判断阶码是否溢出

⚠️ **舍入方法：**
- **0 舍 1 入：** 右移时移出的最高位为 0 则舍去，为 1 则在末位加 1
- **恒置 1：** 右移后末位恒置为 1

⚠️ **溢出判断：**
- **上溢：** 阶码 > 最大阶码，结果为 ∞
- **下溢：** 阶码 < 最小阶码，结果为 0（当作机器零处理）

## 2.4 算术逻辑单元（ALU）

### 2.4.1 ALU的功能

ALU 是组合逻辑电路，能进行多种算术运算和运算操作。

### 2.4.2 加法器 ⭐

#### 一位全加器

$$S_i = A_i \oplus B_i \oplus C_{i-1}$$
$$C_i = A_iB_i + (A_i \oplus B_i)C_{i-1}$$

#### 串行进位加法器（行波进位）

进位信号逐位传递，速度慢。

**延迟：** O(n)，n 为位数。

#### 并行进位加法器（先行进位/超前进位）⭐

利用**进位生成函数 Gᵢ = AᵢBᵢ** 和**进位传递函数 Pᵢ = Aᵢ ⊕ Bᵢ**，使得各位的进位信号同时产生。

$$C_i = G_i + P_i C_{i-1}$$

**优点：** 速度快，延迟 O(1)
**缺点：** 电路复杂

#### 带标志位的加法器

ALU 除输出结果外，还产生状态标志位：

| 标志位 | 名称 | 含义 |
|--------|------|------|
| **ZF** | 零标志 | 结果为 0 时置 1 |
| **SF** | 符号标志 | 结果为负时置 1（等于结果的符号位） |
| **OF** | 溢出标志 | 有符号运算溢出时置 1 |
| **CF** | 进位/借位标志 | 无符号运算溢出时置 1 |

⚠️ **OF 和 CF 的区别：**
- **OF（有符号溢出）：** 最高位进位 ⊕ 次高位进位 = 1
- **CF（无符号溢出）：** 加法时最高位进位；减法时最高位借位

## 2.5 本章例题

### 例题1：补码加法溢出判断

用 8 位补码计算 86 + 43，判断是否溢出。

**解析：**
- [+86]补 = 01010110
- [+43]补 = 00101011
- 相加 = 01111101 = [+125]补
- 符号位为 0，两个操作数符号位都为 0，结果符号位为 0，**无溢出**
- 结果：125

### 例题2：IEEE 754 单精度转换

IEEE 754 单精度数 C0A00000H 的十进制值是多少？

**解析：**
- 二进制：1100 0000 1010 0000 0000 0000 0000 0000
- S = 1（负数）
- 阶码 = 10000001 = 129，E = 129 - 127 = 2
- 尾数 = 01000000000000000000000，1.M = 1.01 = 1.25
- 结果 = -1.25 × 2^2 = **-5.0**

### 例题3：Booth 算法

用 Booth 算法计算 (-5) × 3，写出运算过程（4 位补码）。

**解析：**
- [-5]补 = 1011，[+3]补 = 0011
- 被乘数 [X]补 = 1011，[-X]补 = 0101
- 乘数 = 0011，附加位 = 0

| 步骤 | 部分积 | 乘数及附加位 | 操作 |
|------|--------|------------|------|
| 初始 | 0000 | 00110 | |
| 1 | 0000 | 00110 | y₁y₂=10，+[-X]补=+0101 → 0101 |
| | 0010 | 10011 | 右移 |
| 2 | 0010 | 10011 | y₁y₂=11，+0 → 0010 |
| | 0001 | 01001 | 右移 |
| 3 | 0001 | 01001 | y₁y₂=01，+[X]补=+1011 → 1100 |
| | 1110 | 00100 | 右移 |
| 4 | 1110 | 00100 | y₁y₂=00，+0 → 1110 |
| | 1110 | 00100 | 最后一步不移位 |

结果 = 11100010（8位）= -15 ✓（-5 × 3 = -15）

### 例题4：浮点数加法

用两个数 X = 2^(-01) × 0.1101 和 Y = 2^(-10) × (-0.1010) 进行加法运算（补码，4位尾数）。

**解析：**
1. 对阶：X 的阶为 -1，Y 的阶为 -2，Y 的阶小，Y 右规 1 位：Y = 2^(-01) × (-0.0101)
2. 尾数相加：0.1101 + (-0.0101) = 0.1000
3. 结果 = 2^(-01) × 0.1000，已规格化
4. 结果 = 2^(-1) × 0.5 = **0.25**
