# 第7章 输入输出系统

## 7.1 I/O系统基本概念

### 7.1.1 I/O系统的发展

| 阶段 | 特点 |
|------|------|
| 早期 | I/O 设备与主机分散连接，CPU 直接控制 I/O |
| 接口模块阶段 | I/O 设备通过接口与总线相连 |
| 通道阶段 | 通道是专门管理 I/O 的处理器 |
| I/O 处理机阶段 | 独立的 I/O 处理机 |

### 7.1.2 I/O接口的功能和结构

#### I/O接口的功能 ⭐

1. **地址译码与设备选择**
2. **数据缓冲**
3. **信号格式转换**（串/并转换、电平转换等）
4. **传送控制命令和状态信息**
5. **中断请求管理**
6. **数据检错和纠错**

#### I/O接口的基本结构

```
              数据总线
                ↑↓
    ┌──────────────────────┐
    │  数据缓冲寄存器(DBR)  │
    │  设备状态寄存器       │
    │  设备控制寄存器       │
    │  地址译码与控制逻辑   │
    └──────────────────────┘
         ↑     ↑      ↑
       地址线  控制线  数据线 → I/O设备
```

⚠️ **接口（Interface）和端口（Port）：**
- **接口：** 两个系统或部件之间的交接部分
- **端口：** 接口中可由 CPU 进行读/写操作的寄存器

### 7.1.3 I/O端口的编址方式 ⭐

| 方式 | 说明 | 特点 |
|------|------|------|
| **统一编址（存储器映射方式）** | I/O 端口和主存统一编址，用访存指令访问 I/O 端口 | 不需要专门的 I/O 指令，但占用了主存地址空间 |
| **独立编址（I/O映射方式）** | I/O 端口有独立的地址空间，需专用 I/O 指令（如 IN、OUT） | 不占主存地址空间，但需要专用指令和控制信号 |

## 7.2 I/O方式 ⭐⭐⭐（核心考点）

### 7.2.1 程序查询方式 ⭐

**基本思想：** CPU 不断地查询 I/O 设备的状态，当设备准备好时才进行数据传送。

```
CPU → 查询状态 → 未就绪 → 继续查询
                 ↓ 就绪
              传送数据
```

| 优点 | 缺点 |
|------|------|
| 硬件简单 | CPU 利用率极低 |
| 易于理解 | CPU 完全被 I/O 操作占用 |
| | 不能实现多设备并行工作 |

⚠️ **CPU 和 I/O 串行工作。**

### 7.2.2 程序中断方式 ⭐⭐

**基本思想：** I/O 设备准备好后主动向 CPU 发送中断请求，CPU 响应中断后执行中断服务程序完成数据传送。

```
CPU执行主程序 → I/O设备准备好 → 发中断请求 → CPU响应中断
    ↓                                              ↓
继续执行                                      执行中断服务程序
    ↓                                              ↓
（中断返回后继续）                            传送一个数据
```

| 优点 | 缺点 |
|------|------|
| CPU 不需要一直等待 | 每次中断只能传送一个数据 |
| 支持多设备并行 | 中断处理有开销（保存/恢复现场） |
| CPU 利用率提高 | 不适合大量数据的高速传送 |

⚠️ **中断处理过程：**
1. 设备准备好，发出中断请求
2. CPU 在每条指令执行结束后检查中断
3. 若允许中断，CPU 响应中断
4. 保存断点和现场
5. 执行中断服务程序（传送数据）
6. 恢复现场，返回断点

⚠️ **中断响应条件：**
1. 中断源有中断请求
2. CPU 允许中断（开中断）
3. 一条指令执行完毕

### 7.2.3 DMA 方式 ⭐⭐⭐（重中之重）

**DMA（Direct Memory Access）：** 直接存储器访问，I/O 设备和主存之间直接进行数据传送，不需要 CPU 干预。

#### DMA 的基本概念

- DMA 控制器（DMAC）接管总线控制权
- 数据在 I/O 设备和主存之间直接传送
- CPU 不参与数据传送过程，只需在传送前后进行初始化设置

#### DMA 控制器的组成 ⭐

```
DMA控制器：
├── 主存地址寄存器（AR）→ 存放主存起始地址
├── 外设地址寄存器（DAR）→ 存放设备号/端口
├── 数据计数器（WC）→ 存放传送数据的个数
├── 控制/状态逻辑
├── 数据缓冲寄存器（BR）
└── 中断机构 → 传送完毕后向CPU发中断
```

#### DMA 的传送过程 ⭐

1. **预处理：** CPU 设置 DMA 参数（主存地址、传送字节数、传送方向等），启动 DMA
2. **数据传送：** DMA 控制器接管总线控制，完成数据传送（不需要 CPU 干预）
3. **后处理：** 传送完毕后 DMA 向 CPU 发中断，CPU 处理中断（检查传送是否正确等）

#### DMA 与主存的交互方式 ⭐⭐

| 方式 | 说明 | CPU 访存 | 适用场景 |
|------|------|---------|---------|
| **停止 CPU 访存** | DMA 传送时 CPU 不能访问主存 | 被停止 | 数据量大、速率高 |
| **周期挪用（周期窃取）** ⭐ | DMA 在 CPU 不访存时插入一个存取周期 | DMA 优先 | I/O 设备速率不太高 |
| **DMA 和 CPU 交替访存** | 将 CPU 周期分为 C1（DMA用）和 C2（CPU用） | 交替进行 | CPU 工作周期足够长 |

⚠️ **周期挪用的关键：** 当 DMA 和 CPU 同时请求访存时，**DMA 优先**（因为 I/O 设备的数据若不及时传送就会丢失）。

#### DMA 的优先级 ⚠️

**DMA 请求 > 中断请求 > 普通指令**

因为 DMA 不及时响应会导致数据丢失。

#### DMA 与中断的区别 ⭐⭐

| 对比维度 | DMA | 中断 |
|---------|-----|------|
| 数据传送 | I/O 和主存直接传送 | 需要 CPU 参与传送（执行中断服务程序） |
| CPU 介入 | 只在传送前后介入 | 每次传送都需要介入 |
| 适用场景 | 大批量数据高速传送 | 少量数据或异常处理 |
| 响应时机 | 每个存取周期结束时 | 每条指令结束时 |
| 优先级 | 高于中断 | 低于 DMA |
| 对 CPU 的影响 | 不中断现行程序（仅窃取周期） | 中断现行程序 |

## 7.3 本章例题

### 例题1：I/O 编址方式

若主存地址空间为 64KB，I/O 空间为 256B，采用统一编址和独立编址分别需要多少位地址？

**解析：**
- 统一编址：64KB + 256B 共用地址空间，需要 log₂(64K + 256) ≈ log₂(65792) ≈ **17 位**（实际按 2^17 = 128KB 计算）
- 独立编址：主存 16 位，I/O 端口 8 位，分别编址

### 例题2：DMA 周期挪用

设 CPU 主频为 100MHz，每机器周期 3 个时钟周期，主存存取周期为 1 个时钟周期。某 I/O 设备每秒向主存传送 100KB 数据，采用周期挪用方式。

问：每秒 CPU 被占用多少个时钟周期？

**解析：**
- 每秒传送 100KB = 100 × 1024 B = 102400 B
- 每次传送 1B，需挪用 1 个存取周期 = 1 个时钟周期
- 每秒被占用的时钟周期 = **102400 个**
- 占 CPU 时钟周期的比例 = 102400 / 100M = **0.1024%**

### 例题3：中断与 DMA 综合

某系统有 4 个中断源和 1 个 DMA 通道。画出优先级排列，并说明为什么 DMA 的优先级高于中断。

**答案：**

优先级：**DMA > 中断源1 > 中断源2 > 中断源3 > 中断源4**

原因：
- DMA 传输的数据来自外设，若不及时读取，外设的数据缓冲区会被新数据覆盖，导致数据丢失
- 中断请求延迟响应一般不会导致数据丢失（外设有缓冲）
- 因此 DMA 请求的紧迫性高于中断请求

### 例题4：三种 I/O 方式的比较

比较程序查询、程序中断和 DMA 三种 I/O 方式在 CPU 利用率、数据传送速率和适用场景方面的差异。

**答案：**

| 方式 | CPU 利用率 | 数据传送速率 | 适用场景 |
|------|-----------|------------|---------|
| 程序查询 | 极低 | 低 | 低速设备，简单系统 |
| 程序中断 | 较高 | 中 | 低速/中速设备 |
| DMA | 最高 | 高 | 高速批量数据传送 |
