# 第6章 总线

## 6.1 总线概述

### 6.1.1 总线的基本概念

**总线：** 是一组能为多个部件**分时共享**的公共信息传送线路。

⚠️ **总线的特点：**
- **分时：** 同一时刻只允许一个部件向总线发送信息
- **共享：** 总线上可以挂接多个部件，各部件之间通过总线交换信息

### 6.1.2 总线的分类

| 分类方式 | 类型 | 说明 |
|---------|------|------|
| 按功能 | **片内总线** | CPU 内部寄存器之间、寄存器与 ALU 之间的总线 |
| | **系统总线** | CPU、主存、I/O 接口之间的总线 |
| | **通信总线（外部总线）** | 计算机系统之间或与其他系统之间的总线 |
| 按数据传输方式 | **并行总线** | 一次传输多位数据 |
| | **串行总线** | 一次传输一位数据 |
| 按时序控制方式 | **同步总线** | 由统一的时钟信号控制 |
| | **异步总线** | 采用应答方式控制 |

#### 系统总线的分类 ⭐

| 类型 | 传输内容 |
|------|---------|
| **数据总线（DB）** | 传输数据，双向，宽度与机器字长相关 |
| **地址总线（AB）** | 传输地址，单向（CPU → 内存/I/O），宽度决定寻址空间 |
| **控制总线（CB）** | 传输控制信号和状态信号，每根线单向，整体双向 |

### 6.1.3 系统总线的结构

| 结构 | 特点 |
|------|------|
| **单总线结构** | 所有设备挂在同一组总线上，简单但速度慢 |
| **双总线结构** | 主存总线 + I/O 总线，通过通道连接 |
| **三总线结构** | 主存总线 + I/O 总线 + DMA 总线 |

## 6.2 总线仲裁 ⭐

### 6.2.1 集中仲裁方式

#### （1）链式查询方式 ⭐

各设备通过**总线请求线**发出请求，**总线忙线**检查总线状态，**BG（总线允许）** 信号沿菊花链逐个传递。

**特点：**
- 优先级固定（离仲裁器近的优先级高）
- 硬件简单
- 对电路故障敏感
- 优先级不能改变

#### （2）计数器定时查询方式 ⭐

总线控制器收到请求后，从计数器当前值开始计数，按计数值选择设备。

**特点：**
- 优先级可通过计数器初值设置
- 响应速度较快
- 需要 ⌈log₂n⌉ 根设备地址线

#### （3）独立请求方式 ⭐

每个设备有独立的总线请求线和总线允许线，总线控制器根据优先级策略决定。

**特点：**
- 响应速度最快
- 优先级灵活
- 硬件复杂（需 2n 根线）

### 6.2.2 分布式仲裁方式

各设备自己决定优先级，不需要中央仲裁器。

## 6.3 总线定时 ⭐

### 6.3.1 同步定时方式

所有设备使用**统一的时钟信号**，总线操作在固定的时间点完成。

| 优点 | 缺点 |
|------|------|
| 传输速度快，逻辑简单 | 不能满足速度差异大的设备 |
| 适合总线长度短、各设备速度相近的情况 | 总线效率可能较低 |

### 6.3.2 异步定时方式

不采用统一的时钟，设备之间通过**握手信号**（应答方式）实现定时。

| 类型 | 握手方式 | 特点 |
|------|---------|------|
| **不互锁** | 主模块发出请求后不必等待从模块回答 | 最快，但可靠性低 |
| **半互锁** | 主模块等待从模块的回答，但从模块的回答有时间限制 | 折中 |
| **全互锁** | 主模块等从模块回答，从模块等主模块撤销请求 | 最可靠，但最慢 |

### 6.3.3 半同步定时方式

在同步定时的基础上增加**等待信号（WAIT）**，适应速度差异。

### 6.3.4 分离式定时方式（分离事务）

将一个完整的总线事务分解为两个独立的子过程，提高总线利用率。

## 6.4 总线标准 ⚠️

| 总线标准 | 说明 |
|---------|------|
| **ISA** | 工业标准体系结构，早期 PC 总线 |
| **EISA** | 扩展 ISA |
| **PCI** | 外设部件互连，32/64 位，即插即用 |
| **PCI-E** | PCI Express，串行高速总线 |
| **AGP** | 加速图形端口，专用于显卡 |
| **USB** | 通用串行总线，即插即用、热插拔 |
| **RS-232C** | 串行通信接口标准 |

## 6.5 本章例题

### 例题1：总线仲裁方式比较

链式查询、计数器查询、独立请求三种仲裁方式的比较。

**答案：**

| 比较维度 | 链式查询 | 计数器查询 | 独立请求 |
|---------|---------|-----------|---------|
| 优先级 | 固定 | 可变 | 灵活 |
| 线数 | 3根 | 3+⌈log₂n⌉根 | 2n+2根 |
| 可靠性 | 低 | 中 | 高 |
| 速度 | 慢 | 中 | 快 |

### 例题2：总线带宽计算

某总线时钟频率为 66MHz，数据总线宽度为 32 位，每时钟周期传输一次数据。求总线最大数据传输率。

**解析：**
- 每秒传输次数 = 66 × 10⁶
- 每次传输 32 位 = 4 字节
- 最大数据传输率 = 66 × 10⁶ × 4 = **264 MB/s**
