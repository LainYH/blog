# 第3章 存储系统

## 3.1 存储器的层次结构

### 3.1.1 存储器的分类

#### 按存储介质

| 类型 | 介质 | 特点 |
|------|------|------|
| 半导体存储器 | 半导体器件 | 速度快、集成度高 |
| 磁表面存储器 | 磁性材料 | 容量大、非易失 |
| 光存储器 | 光学材料 | 容量大、便于携带 |

#### 按存取方式

| 类型 | 特点 | 典型 |
|------|------|------|
| **随机存取存储器（RAM）** | 存取时间与存储单元的物理位置无关 | 主存 |
| **顺序存取存储器（SAM）** | 存取时间与物理位置有关 | 磁带 |
| **直接存取存储器（DAM）** | 先直接找到存储区域，再顺序查找 | 磁盘 |

#### 按可保存性

| 类型 | 特点 | 典型 |
|------|------|------|
| **易失性存储器** | 断电后信息消失 | RAM（SRAM、DRAM） |
| **非易失性存储器** | 断电后信息不消失 | ROM、磁盘、闪存 |

### 3.1.2 存储器的层次结构 ⭐

```
    CPU
    ↓↑
  寄存器
    ↓↑
  Cache（高速缓存）
    ↓↑
  主存储器（内存）
    ↓↑
  辅助存储器（外存）
```

⚠️ **从上到下：容量递增、速度递减、价格递减、位价递减。**

⚠️ **主存-Cache层次：** 由硬件自动管理，对程序员透明。
⚠️ **主存-辅存层次：** 由操作系统和硬件共同管理，虚拟存储器。

## 3.2 主存储器 ⭐⭐

### 3.2.1 SRAM 和 DRAM

| 对比维度 | SRAM（静态RAM） | DRAM（动态RAM） |
|---------|----------------|----------------|
| 存储元件 | 触发器 | 电容 |
| 速度 | 快 | 慢 |
| 集成度 | 低 | 高 |
| 功耗 | 大 | 小 |
| 是否需要刷新 | 不需要 | 需要定期刷新 |
| 价格 | 贵 | 便宜 |
| 用途 | Cache | 主存 |

#### DRAM 的刷新 ⚠️

| 刷新方式 | 说明 |
|---------|------|
| **集中刷新** | 在一个刷新周期内，利用一段固定时间对所有行逐一刷新（存在"死时间"） |
| **分散刷新** | 每个存取周期后插入一个刷新周期（不存在死时间，但降低了存储器速度） |
| **异步刷新** | 将刷新周期分散安排（刷新间隔 = 刷新周期 / 行数，每行每隔一个刷新间隔刷新一次） |

⚠️ **刷新对 CPU 透明。** 刷新单位是**行**。

### 3.2.2 ROM（只读存储器）

| 类型 | 可改写 | 特点 |
|------|--------|------|
| **MROM（掩模式ROM）** | 不可 | 出厂时写入，不可修改 |
| **PROM** | 一次性 | 可编程一次 |
| **EPROM** | 可多次 | 可用紫外线擦除重写 |
| **EEPROM** | 可多次 | 可用电擦除重写，可按字节擦除 |
| **Flash Memory（闪存）** | 可多次 | U盘、SSD，按块擦除 |
| **SSD（固态硬盘）** | 可多次 | 基于闪存，速度快 |

### 3.2.3 存储器的连接和控制

#### 存储器芯片的结构

$$\text{存储器芯片} = \text{存储体} + \text{地址译码器} + \text{驱动器} + \text{读写电路} + \text{控制电路}$$

#### 存储器容量扩展 ⭐

| 扩展方式 | 方法 | 说明 |
|---------|------|------|
| **位扩展** | 多个芯片的数据线并联 | 增加存储字长 |
| **字扩展** | 多个芯片共用数据线，用高位地址线译码选片 | 增加存储单元数 |
| **字位同时扩展** | 以上两者结合 | 既增加字长又增加单元数 |

#### 存储器与 CPU 的连接 ⭐

**步骤：**
1. 确定所需芯片数目和扩展方式
2. 地址线连接：低位地址线直接连芯片的地址引脚，高位地址线用于译码选片
3. 数据线连接
4. 读/写控制线连接
5. 片选信号连接

⚠️ **译码器（74138）：** 3位输入 → 8位输出（低电平有效）

## 3.3 Cache ⭐⭐⭐（核心考点）

### 3.3.1 Cache 的基本原理

**局部性原理 ⭐：**
- **时间局部性：** 最近被访问的信息很可能在不久后再次被访问
- **空间局部性：** 最近被访问的信息附近的信息很可能不久后被访问

⚠️ **Cache 就是利用局部性原理，在 CPU 和主存之间增设的速度快、容量小的存储器。**

#### Cache 的命中率

$$h = \frac{N_c}{N_c + N_m}$$

其中 N_c 为 Cache 命中次数，N_m 为 Cache 未命中次数。

#### 平均访问时间

$$t_a = h \times t_c + (1-h) \times (t_c + t_m)$$

其中 t_c 为 Cache 访问时间，t_m 为主存访问时间。

若采用**同时访问** Cache 和主存的方式（未命中时需要等主存）：
$$t_a = h \times t_c + (1-h) \times t_m$$

### 3.3.2 主存与 Cache 的映射方式 ⭐⭐

#### （1）直接映射 ⭐

每个主存块只能映射到 Cache 中**唯一的一个**位置。

**映射规则：** 主存块号 mod Cache 行数

```
| 标记(Tag) | Cache行号 | 块内地址 |
```

| 优点 | 缺点 |
|------|------|
| 硬件简单，判断快 | 冲突率高，Cache 利用率低 |

**判断命中：** 比较 Tag。

#### （2）全相联映射 ⭐

主存中任何一个块可以映射到 Cache 中的**任何一个**位置。

```
| 标记(Tag) | 块内地址 |
```

| 优点 | 缺点 |
|------|------|
| 冲突率低，Cache 利用率高 | 需要比较所有行的 Tag，硬件复杂，速度慢 |

#### （3）组相联映射 ⭐⭐

将 Cache 分成若干组，主存的每一块可以映射到**特定组内的任意一行**。

**映射规则：** 主存块号 mod Cache 组数

```
| 标记(Tag) | 组号 | 块内地址 |
```

⚠️ **直接映射是组相联映射的特殊情况（每组 1 行）。**
⚠️ **全相联映射是组相联映射的特殊情况（只有 1 组）。**

### 3.3.3 Cache 中主存块的替换算法 ⚠️

当 Cache 已满且需要装入新的主存块时，需要选择一个已有的块被替换。

| 算法 | 说明 | 硬件复杂度 |
|------|------|-----------|
| **随机算法（RAND）** | 随机选择一个块替换 | 低 |
| **先进先出（FIFO）** | 替换最早进入 Cache 的块 | 中 |
| **近期最少使用（LRU）** ⭐ | 替换最近最少被访问的块 | 中 |
| **最不经常使用（LFU）** | 替换被访问次数最少的块 | 高 |

⚠️ **LRU 算法的实现：** 每个 Cache 行设置一个计数器，命中时该行计数器清零，其他行计数器 +1；替换时选择计数器值最大的行。

⚠️ **直接映射不需要替换算法**（只有一个候选位置）。

### 3.3.4 Cache 写策略 ⚠️

| 写策略 | 说明 |
|--------|------|
| **全写法（写直达，Write-Through）** | 写 Cache 的同时写主存（常配合**写缓冲**使用） |
| **回写法（写回，Write-Back）** | 只写 Cache，当该块被替换时才写回主存（需设置**脏位/修改位**） |

⚠️ **全写法 + 写缓冲：** 数据先写入写缓冲，由写缓冲在适当时机写入主存，减少 CPU 等待时间。

### 3.3.5 Cache 的性能分析 ⭐

**例：** CPU 执行某程序时，访问 Cache 命中 2000 次，未命中 50 次。Cache 存取周期为 20ns，主存存取周期为 100ns。求 Cache-主存系统的平均访问时间和效率。

- 命中率 h = 2000/2050 = 97.56%
- 平均访问时间 t_a = 0.9756 × 20 + 0.0244 × (20+100) = 19.512 + 2.928 = 22.44ns
- 访问效率 e = t_c / t_a = 20/22.44 = 89.1%

## 3.4 虚拟存储器 ⭐⭐

### 3.4.1 虚拟存储器的基本概念

虚拟存储器将主存和辅存统一编址，形成一个比实际主存容量大得多的虚拟地址空间。

⚠️ **虚拟存储器由"主存-辅存"层次实现，对应用程序员透明。**

### 3.4.2 页式虚拟存储器 ⭐

#### 基本原理

- 虚拟地址空间划分为大小相等的**虚拟页（虚页）**
- 主存划分为大小相等的**物理页（实页/页框）**
- 虚页和物理页大小相同

**虚拟地址 = 虚拟页号 + 页内偏移**

#### 页表

页表实现虚页号到物理页号的映射。

⚠️ **页表项（PTE）的内容：**
- 物理页号
- **有效位（装入位）：** 1 表示该虚页已调入主存，0 表示不在主存
- 脏位（修改位）
- 引用位（使用位）

#### 地址转换过程 ⭐

1. CPU 发出虚拟地址
2. 用虚拟页号查**快表（TLB）**
3. TLB 命中 → 直接获得物理页号
4. TLB 未命中 → 查**页表**（在主存中）
5. 页表中有效位 = 1 → 物理地址
6. 有效位 = 0 → **缺页中断**，操作系统将该页从辅存调入主存

⚠️ **缺页中断处理后，需要重新执行被中断的指令。**

### 3.4.3 快表（TLB）⭐⭐

TLB（Translation Lookaside Buffer）是一个高速缓存，用于存放近期常用的页表项。

⚠️ **TLB 是 Cache，页表在主存中。**

#### 带 TLB 的地址转换时间

$$t = h_{TLB} \times (t_{TLB} + t_{cache} + t_{mem}) + (1-h_{TLB}) \times (t_{TLB} + t_{page} + t_{cache} + t_{mem})$$

简化（忽略 Cache 未命中的情况）：
$$t = t_{TLB} + t_{cache} + t_{mem} \quad \text{（TLB命中）}$$
$$t = t_{TLB} + t_{mem\_page} + t_{cache} + t_{mem} \quad \text{（TLB未命中）}$$

⚠️ **TLB 命中则无需访问主存中的页表，大大提高了地址转换速度。**

### 3.4.4 段式虚拟存储器

按程序的逻辑结构划分段，每段有自己的段名和长度。

**虚拟地址 = 段号 + 段内偏移**

⚠️ **段表项：** 段起点 + 段长 + 其他控制位。

### 3.4.5 段页式虚拟存储器

程序按段划分，每段再按页划分。

**虚拟地址 = 段号 + 虚拟页号 + 页内偏移**

⚠️ 需要先查段表（得到该段的页表起始地址），再查页表（得到物理页号），最后合成物理地址。需要访问两次主存（段表 + 页表），因此 TLB 更为重要。

## 3.5 本章例题

### 例题1：Cache 命中率和平均访问时间

某系统有 Cache 和主存，Cache 容量 8KB，主存容量 128KB，Cache 和主存均按 64B 分块。设 Cache 命中率为 95%，Cache 访问时间为 20ns，主存访问时间为 200ns。

求：（1）Cache-主存系统的平均访问时间；（2）访问效率。

**解析：**

（1）t_a = h × t_c + (1-h) × (t_c + t_m) = 0.95 × 20 + 0.05 × (20+200) = 19 + 11 = **30ns**

（2）e = t_c / t_a = 20/30 = **66.7%**

### 例题2：Cache 地址映射计算

假设主存容量 1MB，Cache 容量 16KB，块大小 64B。分别计算直接映射、2 路组相联和全相联映射下各字段的位数。

**解析：**

- 主存地址 = log₂(1MB) = **20 位**
- 块内地址 = log₂(64B) = **6 位**
- Cache 行数 = 16KB / 64B = **256 行**

**直接映射：**
- Cache 行号 = log₂(256) = **8 位**
- Tag = 20 - 8 - 6 = **6 位**

**2 路组相联：**
- 组数 = 256 / 2 = **128 组**
- 组号 = log₂(128) = **7 位**
- Tag = 20 - 7 - 6 = **7 位**

**全相联映射：**
- Tag = 20 - 6 = **14 位**

### 例题3：虚拟地址转换

某系统虚拟地址 32 位，页面大小 4KB，页表项 4B，TLB 为 4 组 2 路组相联。计算：

（1）虚拟页号的位数
（2）页表的大小
（3）TLB 的地址划分

**解析：**

（1）页面大小 4KB = 2^12 B → 页内偏移 12 位
虚拟页号 = 32 - 12 = **20 位**

（2）虚页数 = 2^20，每项 4B
页表大小 = 2^20 × 4B = **4MB**

（3）TLB 4 组 2 路：
- 组号 = log₂(4) = **2 位**
- Tag = 20 - 2 = **18 位**
