# 第1章 计算机网络体系结构

## 1.1 计算机网络概述

### 1.1.1 计算机网络的概念

计算机网络是一个将分散的、具有独立功能的计算机系统，通过通信设备与线路连接起来，由功能完善的软件实现**资源共享**和**信息传递**的系统。

### 1.1.2 计算机网络的组成

| 组成 | 说明 |
|------|------|
| **硬件** | 主机、通信设备（路由器、交换机）、通信链路 |
| **软件** | 协议、应用软件 |
| **协议** ⭐ | 为进行网络中的数据交换而建立的规则、标准或约定 |

### 1.1.3 计算机网络的功能

| 功能 | 说明 |
|------|------|
| **数据通信** | 最基本的功能 |
| **资源共享** | 硬件、软件、数据共享 |
| **分布式处理** | 将任务分配给多台计算机处理 |
| **提高可靠性** | 替代机、负载均衡 |
| **负载均衡** | 各计算机之间更均衡地分配工作任务 |

### 1.1.4 计算机网络的分类

| 分类方式 | 类型 |
|---------|------|
| 按分布范围 | 广域网（WAN）、城域网（MAN）、局域网（LAN）、个人区域网（PAN） |
| 按传输技术 | 广播式网络、点对点网络 |
| 按拓扑结构 | 总线型、星型、环型、网状型 |
| 按使用者 | 公用网、专用网 |
| 按交换方式 | 电路交换、报文交换、分组交换 |

### 1.1.5 计算机网络的性能指标 ⭐

| 指标 | 定义 | 单位 |
|------|------|------|
| **速率（数据率/比特率）** | 数据的传送速率 | b/s, kb/s, Mb/s, Gb/s |
| **带宽** | 信号的频带宽度/网络的最大数据率 | Hz / b/s |
| **吞吐量** | 单位时间内通过某个网络的数据量 | b/s |
| **时延** | 数据从网络一端传送到另一端所需的时间 | s |
| **时延带宽积** | 传播时延 × 带宽 | bit |
| **往返时延（RTT）** | 从发送方发送数据到收到确认的总时间 | s |
| **信道利用率** | 某信道有百分之几的时间被利用 | % |

#### 时延的组成 ⭐

| 时延类型 | 计算方式 | 说明 |
|---------|---------|------|
| **发送时延（传输时延）** | 数据长度 / 发送速率 | 发送器将数据推向信道所需时间 |
| **传播时延** | 信道长度 / 电磁波传播速率 | 信号在信道中传播所需时间 |
| **处理时延** | — | 主机或路由器处理分组所需时间 |
| **排队时延** | — | 分组在路由器输入/输出队列中排队等待时间 |

⚠️ **易混淆：发送时延 vs 传播时延**
- 发送时延与**数据长度和发送速率**有关
- 传播时延与**信道长度和信号传播速率**有关
- "高速链路"提高的是**发送速率**，减小的是**发送时延**，不是传播时延

## 1.2 计算机网络体系结构与参考模型 ⭐⭐

### 1.2.1 协议、接口、服务

| 概念 | 定义 |
|------|------|
| **协议** | 对等实体之间通信的规则集合（语法、语义、同步） |
| **接口（服务访问点 SAP）** | 同一结点内相邻两层之间交换信息的连接点 |
| **服务** | 下层为上层提供的功能调用 |

⚠️ **协议是"水平的"（对等实体之间），服务是"垂直的"（上下层之间）。**

### 1.2.2 OSI 参考模型 ⭐⭐

**7 层模型：**

```
应用层        ← 用户与网络的接口
表示层        ← 数据格式转换、加密、压缩
会话层        ← 建立、管理和终止会话
传输层        ← 端到端的可靠/不可靠传输
网络层        ← 路由选择、分组转发
数据链路层     ← 成帧、差错控制、流量控制
物理层        ← 透明传输比特流
```

#### 各层功能 ⭐

| 层 | 功能 | 数据单位 |
|----|------|---------|
| **物理层** | 定义接口特性（机械、电气、功能、规程）、比特传输、传输介质 | 比特 |
| **数据链路层** | 成帧、物理寻址（MAC地址）、差错控制、流量控制、访问控制 | 帧 |
| **网络层** | 逻辑寻址（IP地址）、路由选择、分组转发、拥塞控制 | 数据报/分组 |
| **传输层** | 端到端通信（进程到进程）、可靠传输/不可靠传输、差错控制、流量控制 | 报文段/用户数据报 |
| **会话层** | 会话管理、同步 | 报文 |
| **表示层** | 数据格式转换、加密解密、压缩 | 报文 |
| **应用层** | 为用户的应用进程提供服务 | 报文 |

#### OSI 模型中的数据传输

```
发送方：应用层 → 表示层 → 会话层 → 传输层 → 网络层 → 数据链路层 → 物理层
                ↓ (每层添加首部/尾部——封装)
接收方：物理层 → 数据链路层 → 网络层 → 传输层 → 会话层 → 表示层 → 应用层
                ↓ (每层剥去首部/尾部——解封装)
```

### 1.2.3 TCP/IP 模型 ⭐⭐

**4 层模型：**

```
应用层        ← HTTP, FTP, SMTP, DNS, ...
传输层        ← TCP, UDP
网际层（网络层）← IP, ICMP, IGMP, ARP, ...
网络接口层     ← 各种底层网络技术
```

### 1.2.4 OSI 与 TCP/IP 的对比 ⭐

| 对比维度 | OSI | TCP/IP |
|---------|-----|--------|
| 层次数 | 7 层 | 4 层 |
| 网络层服务 | 提供面向连接和无连接 | 只提供无连接（IP） |
| 传输层服务 | 只提供面向连接 | 提供面向连接（TCP）和无连接（UDP） |
| 产生时机 | 先有模型，后有协议 | 先有协议，后有模型 |
| 适用性 | 理论完善但未普及 | 广泛应用 |

⚠️ **TCP/IP 的网络层（IP层）只提供无连接的不可靠服务。可靠传输由传输层的 TCP 负责。**

### 1.2.5 五层参考模型（408常用）

```
应用层        ← HTTP, FTP, DNS, SMTP, POP3, ...
传输层        ← TCP, UDP
网络层        ← IP, ICMP, ARP, IGMP, 路由协议, ...
数据链路层     ← 以太网, PPP, HDLC, ...
物理层        ← 传输介质, 编码, ...
```

## 1.3 本章例题

### 例题1：时延计算

设数据长度为 1000B，发送速率为 100kb/s，信道长度为 1000km，电磁波传播速率为 2×10⁸ m/s。求发送时延和传播时延。

**解析：**
- 发送时延 = 1000×8 / (100×10³) = 80ms
- 传播时延 = 1000×10³ / (2×10⁸) = 5ms

### 例题2：体系结构

在 OSI 参考模型中，提供流量控制功能的层有哪些？

**答案：** 数据链路层、网络层、传输层

解析：数据链路层在相邻结点间提供流量控制，网络层提供全局的流量控制，传输层提供端到端的流量控制。

### 例题3：封装与解封装

发送方发送数据时，数据在各层如何变化？

**答案：**

发送时各层添加首部（尾部）：
- 传输层：添加 TCP/UDP 首部（段）
- 网络层：添加 IP 首部（数据报/分组）
- 数据链路层：添加帧头和帧尾（帧）
- 物理层：转换为比特流

接收时各层剥去相应的首部（尾部），完成解封装。
